## 为什么

JM 来源当前在用户切换来源后把默认空白关键词操作替换为随机漫画，导致用户无法像访问 JM 首页那样浏览稳定、可理解的内容入口。JM 首页已经提供连载更新、汉化组、韩漫、推荐和最新漫画等分栏，搜索页应在空白关键词场景保留这些栏目语义，同时把随机浏览留给显式的随机按钮。

## 变更内容

- 将 JM 的“关键词模式 + 空白关键词 + 第 1 页”定义为首页栏目请求，而不是普通空查询或随机请求。
- 从 JM 首页解析包含标准 `/album/{id}` 卡片的栏目，排除小说、书库等不兼容现有漫画下载模型的内容。
- 扩展搜索结果契约，以去重漫画集合配合栏目标题和漫画 ID 引用表达分组，避免同一本漫画在多个栏目中重复传输或重复进入批量操作。
- 在搜索页按栏目渲染 JM 首页结果，并让栏目结构参与过滤、高亮、批量选择和搜索缓存恢复；普通关键词、排行、标签及其他来源继续使用现有平铺结果。
- 用户主动切换到已认证 JM 时执行空白关键词搜索，不再自动触发随机；显式随机按钮保持原行为。
- JM 首页请求复用现有系统代理、Session、认证和结构化人机验证恢复链路。

## 功能 (Capabilities)

### 新增功能

- `jm-home-sections`: 定义 JM 空白关键词首页请求、栏目解析、分组搜索结果契约及搜索页栏目展示行为。

### 修改功能

<!-- 无。现有搜索、人机验证、加载反馈和 IPC 安全需求保持不变；新能力在其既有约束下工作。 -->

## 影响

- Python：`sources/jm/parser.py`、`sources/__init__.py`、`python/ipc/search_mixin.py` 及 JM/搜索 IPC 测试夹具。
- 共享契约：`shared/types.ts` 中的搜索结果与栏目描述类型。
- React：`src/pages/SearchPage.tsx`、搜索缓存 store、分组结果渲染与相关单元测试。
- Electron：沿用现有 `search` IPC 和 JM 挑战恢复编排，不新增网络通道。
- 网络：JM 首页请求继续使用解析器已注入系统代理的同一 Session，不引入新依赖或绕过代理的请求实现。
