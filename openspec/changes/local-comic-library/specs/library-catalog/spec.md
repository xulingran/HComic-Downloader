## ADDED Requirements

### 需求:以当前下载目录中的磁盘资产为漫画库真相源

系统必须仅将当前配置 `downloadDir` 内实际存在的合法漫画资产展示在漫画库中。`download_history` 必须只用于补全来源、漫画 ID、专辑和下载时间等信息，禁止仅因存在历史记录就在漫画库中展示已丢失文件。

#### 场景:历史记录对应文件已丢失
- **当** `download_history` 中存在记录但其输出文件在磁盘上不存在
- **那么** 该记录不作为漫画库资产展示，并在扫描结果中记为失效历史关联

#### 场景:磁盘资产没有下载历史
- **当** 用户手动复制一个合法 CBZ 到 `downloadDir` 且数据库没有对应记录
- **那么** 扫描器仍将其加入漫画库，并标记为未关联来源的本地资产

### 需求:扫描受支持的本地漫画格式

扫描器必须识别下载目录顶层的 `.cbz`、`.zip` 和包含受支持图片的文件夹，必须跳过 `temp_*`、隐藏目录、空目录、符号链接逃逸和不支持的文件。多章节文件夹必须作为一个专辑资产建模，并保留有序章节清单。

#### 场景:扫描混合格式目录
- **当** 下载目录同时包含 CBZ、ZIP、单本图片文件夹、多章节文件夹、文本文件和 `temp_*` 目录
- **那么** 漫画库只返回四类合法漫画资产，文本文件与临时目录不出现在目录中

#### 场景:识别多章节文件夹
- **当** 某顶层文件夹包含多个各自含图片的章节子目录
- **那么** 系统返回一个带章节清单的专辑资产，禁止把专辑根目录和各章节重复计为多本独立漫画

### 需求:持久化可重建的漫画库索引

系统必须维护独立于 `download_history` 的漫画库索引。索引必须存储稳定资产 ID、规范化路径、格式、大小、修改时间、页数、标题、作者、标签、来源、专辑信息、封面缓存键和扫描状态；索引必须可从磁盘完全重建，损坏或删除索引不得造成漫画文件丢失。

#### 场景:首次建立索引
- **当** 当前下载目录尚无漫画库索引
- **那么** 后端在后台扫描合法资产，为每项生成资产 ID 并持久化索引，然后分批返回可浏览结果

#### 场景:索引数据库损坏
- **当** 索引数据库无法打开或校验失败
- **那么** 系统隔离损坏索引并从磁盘重建，禁止修改或删除任何漫画资产

### 需求:增量扫描必须对账外部文件变化

扫描器必须使用规范化路径、格式、大小和修改时间识别未变化、新增、修改和删除资产。未变化资产禁止重复解析完整压缩包；外部新增、修改、重命名或删除必须在刷新后反映到索引和 UI。

#### 场景:重复刷新未变化的漫画库
- **当** 用户再次刷新且所有资产的路径、大小和修改时间均未变化
- **那么** 系统复用已有元数据与封面缓存，不重新读取每个压缩包的全部条目

#### 场景:用户在应用外删除文件
- **当** 某已索引资产被外部程序删除后用户刷新漫画库
- **那么** 系统从索引和当前列表移除该资产，并保留可诊断的失效历史关联信息

### 需求:按明确优先级合并资产元数据

CBZ 元数据必须优先来自有效 `ComicInfo.xml`；缺失字段依次由匹配的 `download_history` 和文件名解析补全。ZIP 与文件夹必须优先使用下载历史，其余字段回退文件名。无论元数据是否完整，合法资产都必须可见。

#### 场景:ComicInfo 与历史标题冲突
- **当** CBZ 的 `ComicInfo.xml` 标题与下载历史标题不同
- **那么** 漫画库显示 `ComicInfo.xml` 标题，并保留来源关联信息

#### 场景:资产没有任何可识别作者
- **当** 文件名、元数据和下载历史均无法提供作者
- **那么** 资产仍显示在漫画库中，作者展示为“未知作者”而非扫描失败

### 需求:本地封面按需提取并缓存

系统必须使用资产的第一张有效图片作为默认封面，并按资产 ID、修改时间和图片条目生成缓存键。封面提取必须按需或限流执行，禁止首次扫描时无上限并发解压所有资产；SFW 模式必须继续隐藏真实封面。

#### 场景:首次展示无缓存的 CBZ
- **当** 漫画卡进入可视区域且对应封面缓存不存在
- **那么** 后端只读取首张有效图片生成缩略图，前端在完成前显示骨架占位

#### 场景:漫画文件内容发生变化
- **当** 已缓存封面的资产修改时间发生变化
- **那么** 原封面缓存键失效，系统重新提取封面

### 需求:漫画库支持查询筛选排序和分页

漫画库查询必须支持标题/作者/标签关键字、来源、格式和健康状态筛选，支持按最近添加、最近阅读、标题、大小和修改时间排序，并使用稳定分页或游标避免一次向渲染进程发送整个大型目录。

#### 场景:组合筛选漫画
- **当** 用户输入作者关键字并选择 CBZ 格式与某个来源
- **那么** 返回结果同时满足全部条件，并返回总数及当前分页信息

#### 场景:相同排序值保持稳定
- **当** 多个资产具有相同修改时间
- **那么** 系统使用资产 ID 作为次级排序键，连续翻页不重复或遗漏资产

### 需求:后台扫描可观察且可取消

完整扫描必须在后台执行并发送阶段、当前数量、总量和当前标签等进度；同一漫画库同一时刻只允许一个完整扫描，用户必须能够取消，取消后已提交的有效索引更新可以保留但不得把未扫描资产误判为删除。

#### 场景:取消进行中的完整扫描
- **当** 用户在完整扫描未完成时点击取消
- **那么** 后端在安全边界停止后续解析，返回取消状态，禁止执行最终删除对账阶段
#### 场景:重复发起完整扫描
- **当** 一个完整扫描正在运行时再次请求完整扫描
- **那么** 后端复用当前扫描状态或返回“扫描进行中”，禁止并发启动第二个扫描器
