## 为什么

当前“下载管理”只能查看进程内的下载任务，用户无法在应用内浏览、阅读和管理已经落盘的 CBZ、ZIP 或图片文件夹；维护中心虽然已有磁盘扫描能力，但只提供健康与空间统计，尚未形成可日常使用的本地漫画库。将两者整合为一个漫画库工作区，可以让下载完成后的内容继续在应用内被发现、阅读和维护，同时保留现有下载队列控制能力。

## 变更内容

- 将侧边栏“下载管理”升级为“漫画库”，页面内部提供“漫画库”和“下载任务”两个子页签；漫画库为默认视图，下载任务完整保留现有暂停、恢复、重试、取消、专辑折叠和强制打包功能。
- 以当前配置的 `downloadDir` 为唯一漫画库根目录，扫描并索引顶层 CBZ、ZIP 和图片文件夹；磁盘存在性是资产真相源，`download_history` 仅用于补充来源和下载元数据。
- 提供漫画库网格/列表浏览、关键字搜索、来源/格式/健康状态筛选、排序、扫描进度、空状态和错误状态。
- 从本地资产生成并缓存封面，支持直接阅读本地 CBZ、ZIP、单本文件夹和多章节文件夹，并持久化本地阅读进度。
- 提供查看详情、在系统文件管理器中定位、重新扫描、单项健康检查、移入系统回收站、重命名和编辑受支持元数据等管理操作。
- 下载完成、目录迁移完成或用户手动刷新后增量更新漫画库索引；应用启动和外部文件变化场景可通过可取消的后台扫描进行对账。
- 所有本地文件访问均通过受限资产 ID 和根目录校验完成；渲染进程不得读取或删除任意绝对路径，正在下载、打包或维护中的资产不得执行破坏性操作。

## 功能 (Capabilities)

### 新增功能

- `library-workspace`: 漫画库与下载任务的统一页面结构、子页签导航、活跃任务提示和现有下载管理行为保留。
- `library-catalog`: 下载目录扫描、持久化索引、增量对账、封面、搜索、筛选、排序和详情展示。
- `local-comic-reader`: 通过受限本地图片交付链路阅读 CBZ、ZIP、图片文件夹及多章节资产，并保存阅读进度。
- `library-asset-management`: 安全定位、健康检查、移入回收站、重命名、元数据编辑及索引/下载历史同步。

### 修改功能

无。既有下载队列、下载目录迁移和维护中心规范保持原有行为；新能力通过复用并组合现有接口扩展用户体验。

## 影响

- 前端：`DownloadPage` 将拆分为漫画库工作区与下载任务视图，并新增漫画库 store、筛选/详情/管理组件以及本地阅读入口。
- 共享契约：新增漫画库资产、查询、扫描进度、阅读清单和管理结果类型及 IPC 通道。
- Python 后端：扩展维护扫描器，新增漫画库索引与阅读进度持久化、资产查询、封面提取、页清单和元数据更新服务。
- Electron：新增受限的本地漫画图片协议或等价流式通道，并使用系统文件管理器/回收站 API 执行定位与删除。
- 数据：在应用数据目录新增可重建的漫画库索引和本地阅读进度表；继续兼容现有 `download_history.db`。
- 测试：新增扫描/增量索引/路径安全/压缩包读取/管理原子性测试，以及 IPC 契约、页面交互、本地阅读和下载任务回归测试。
