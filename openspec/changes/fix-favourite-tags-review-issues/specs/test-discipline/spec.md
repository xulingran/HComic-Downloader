## 新增需求

### 需求:主干必须在干净检出状态下通过全套构建闸门，禁止以工作区未提交改动掩盖失败

仓库的完整验证流程（`pytest` / `npx tsc --noEmit` / `npm test` / `npm run lint:py` / `black --check .` / `npm run lint` / `npm run lint:test-quality`）必须在「干净工作区」（`git stash` 后或全新 clone，无未提交改动）下全部通过。禁止依赖开发者本地未提交的改动来让 `tsc` 或 Vitest 通过——此类部分状态会让远端 CI 或协作者检出后主干直接不可构建。本需求作为回归约束，防止「引用未提交 symbol」「mock 未提交」等问题再次进入主干。

#### 场景:审查或合并前必须验证干净主干

- **当** 准备合并或发布一个变更
- **那么** 验证流程必须在 `git stash` 清空工作区未提交改动后重新执行 `npx tsc --noEmit` 与 `npm test`
- **且** 二者必须全过，禁止仅凭带未提交改动的本地工作区结果放行

#### 场景:引用未提交 symbol 被闸门拦截

- **当** 一个已提交的源文件引用了仅存在于工作区未提交改动中的类型 / hook / API
- **那么** 干净主干的 `npx tsc --noEmit` 必须报错（`TS2305` / `TS2724` 等）并失败
- **且** 该失败必须被视为合并阻断，禁止通过「补一个本地未提交改动」绕过
