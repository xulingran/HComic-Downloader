## 上下文

共享 `ReaderShell` 已为遮罩和阅读器主体使用 `overlayPresenceVariants` 与 `readerPresenceVariants`，后者也包含向下滑出的 `exit` 状态。但当前 `ReaderShell` 在 `open=false` 时直接返回 `null`，外层没有 `AnimatePresence`，motion 节点也未声明退出状态。与此同时，在线 store 在关闭时立即清空 `readerComic`，本地 store 立即清空 `readerAsset`，使退场期间没有可渲染的数据。

关闭还承担进度 flush、预加载取消、缓存清理、设置复位和事件监听移除等职责。若简单延迟全部关闭逻辑，退场期间仍可能响应键盘、滚轮或异步回调；若只增加 AnimatePresence 而不保留内容，动画会渲染空壳或仍然立即消失。

## 目标 / 非目标

**目标：**

- 在线和本地阅读器在所有关闭入口下播放一致的主体下滑与遮罩淡出动画。
- 关闭请求发生后立即终止交互和会话活动，但保留退场所需的最后有效渲染快照。
- 在退出完成后可靠清理 store 内容、缓存及临时状态。
- 保留本地阅读进度的即时 flush 语义，并支持 reduced-motion 与 React StrictMode。
- 对快速重复关闭、关闭后立即打开另一漫画和过期动画回调提供确定行为。

**非目标：**

- 不调整阅读器打开动画的方向、曲线或时长。
- 不改变翻页、显示模式切换、图片预加载、IPC 或本地资产交付协议。
- 不为不同关闭入口设计不同动画。
- 不引入新的动画依赖或手写计时器模拟 Framer Motion 生命周期。

## 决策

### 1. 由共享 ReaderShell 的 AnimatePresence 负责视觉生命周期

`ReaderShell` 保持挂载，由其内部的条件渲染和 `AnimatePresence` 驱动整组遮罩与阅读器主体。遮罩和主体都显式声明 `initial`、`animate` 与 `exit`；退出完成通过 `onExitComplete` 上报拥有数据生命周期的上层。

这样在线和本地阅读器共享同一套视觉语义，也避免两个 modal 各自实现 mounted/closing 计时状态。替代方案是在每个 modal 内分别包裹 `AnimatePresence`，但容易造成动画和清理顺序漂移。

### 2. 区分可见性、渲染快照和最终清理

关闭请求立即把阅读器置为不可交互的 closing 状态，并把 `open` 置为 false 触发退出；最后一次有效 `comic` 或 `asset` 必须保留到 `onExitComplete`。退出完成后，再以会话身份校验的 finalize 动作清空内容引用和退场专用状态。

在线与本地 store 可以采用显式 `phase`/`isClosing` 加 finalize action，或等价的“open 与 payload 分离”模型，但禁止依赖固定毫秒数的 `setTimeout`。清理动作必须幂等。

### 3. 业务关闭在请求阶段完成，纯渲染数据在完成阶段释放

关闭请求阶段立即执行：停止键盘和指针输入、取消或失效异步预加载、关闭设置面板、flush 本地阅读进度并阻止新的进度写入。退出完成阶段释放最后渲染 payload、页面缓存和只为卸载准备的状态。

这避免为了动画而继续运行阅读会话，也确保退场画面仍有稳定内容。无法安全延迟的资源清理应通过不可变快照或已物化图片 URL 保证退出帧无需再次请求数据。

### 4. 使用会话标识防止过期退出回调关闭新阅读器

每次打开生成或推进会话标识。`onExitComplete` 只能 finalize 与本次退出对应的会话；如果用户在旧退出完成前打开新漫画，旧回调禁止清空新 payload 或改变新会话的 open 状态。重复关闭请求为幂等操作。

### 5. reduced-motion 沿用集中动画令牌

主体继续通过 `reduceSafe(readerPresenceVariants)` 去除 y 位移并保留透明度退场，遮罩使用现有 opacity variants。全局 reduced-motion CSS 仍作为兜底。不得为关闭路径增加独立硬编码时长。

### 6. 测试围绕可观察生命周期而非内部实现

组件测试验证关闭后节点在退出完成前仍存在、exit 完成后移除；交互测试覆盖关闭按钮、Escape 和遮罩入口；store/集成测试验证进度 flush、重复关闭及旧回调不会清除新会话。测试通过模拟 Framer Motion 完成事件或受控 presence stub，不使用真实等待时间。

## 风险 / 权衡

- [退场期间缓存被过早清理导致图片闪空] → 明确拆分会话停止与渲染快照释放，并以测试验证退出帧内容保持。
- [旧动画完成回调清空刚打开的新漫画] → finalize 携带并校验会话标识，过期回调无效。
- [本地进度因延迟清理而丢失] → 在关闭请求阶段立即 flush，而非等待动画结束。
- [退出期间仍响应输入或异步请求] → closing/open 状态立即关闭监听和输入门控，并失效旧异步任务。
- [StrictMode 或测试环境多次触发生命周期回调] → close/finalize 均保持幂等，不依赖一次性计时器。
- [共享外壳改动影响两类阅读器] → 增加在线、本地和 reduced-motion 的组件级回归测试，并运行现有阅读器测试集。

## 迁移计划

1. 先扩展阅读器状态契约，使 payload 能在 `open=false` 后保留到 finalize。
2. 再让共享外壳通过 AnimatePresence 驱动退出并上报完成事件。
3. 接入在线和本地阅读器的关闭请求、业务清理与最终清理。
4. 补齐回归测试并运行类型检查、前端测试及 lint。

该变更不涉及持久化数据迁移。若需回滚，可恢复关闭时同步清空 payload 和直接卸载行为，不影响已保存的阅读进度或后端数据。

## 待解决问题

无。具体状态字段命名可在实现中决定，但必须满足会话校验、幂等关闭和退出完成后清理的行为约束。
