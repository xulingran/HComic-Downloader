## 新增需求

### 需求: NH 入口子功能返回按钮持续可见

用户从 NH 入口页进入任一子功能（最近更新、热门排行、热门标签）后，「返回 NH 入口」按钮必须持续可见，直到用户通过显式退出动作（点击「返回 NH 入口」、切换到其他来源、点击随机）离开入口体系。在入口子功能内执行的后续操作——包括关键词搜索、翻页、排行下拉切换、标签增删——禁止隐藏该按钮。该按钮的显隐控制必须与 NH 入口页网格（`NhEntryGrid`）的渲染控制相互独立，禁止由单个状态同时承担两个职责，以避免事件处理器写入语义不一致导致的按钮误隐藏与入口页网格误重渲染。

#### 场景:进入热门排行后执行关键词搜索时按钮仍可见

- **当** 用户从 NH 入口页点击「热门排行」进入排行结果，且结果加载完成显示「返回 NH 入口」按钮
- **且** 用户在搜索框输入关键词并点击「搜索」触发关键词搜索
- **那么** 关键词搜索加载完成后，「返回 NH 入口」按钮必须仍然可见
- **且** 禁止因搜索模式为 keyword 而隐藏该按钮

#### 场景:进入最近更新后翻页时按钮仍可见

- **当** 用户从 NH 入口页点击「最近更新」进入结果列表，且结果加载完成显示「返回 NH 入口」按钮
- **且** 用户点击分页控件翻到下一页
- **那么** 翻页加载完成后，「返回 NH 入口」按钮必须仍然可见

#### 场景:在入口子功能内切换排行下拉时按钮仍可见

- **当** 用户已在 NH 热门排行结果中，「返回 NH 入口」按钮可见
- **且** 用户通过排行下拉切换到另一个排行选项
- **那么** 新排行结果加载完成后，「返回 NH 入口」按钮必须仍然可见

#### 场景:在入口子功能内增删标签时按钮仍可见

- **当** 用户已通过 NH 入口页热门标签进入标签搜索结果，「返回 NH 入口」按钮可见
- **且** 用户通过标签面板新增或移除标签触发新的标签搜索
- **那么** 新的标签搜索结果加载完成后，「返回 NH 入口」按钮必须仍然可见

#### 场景:点击返回入口页时按钮隐藏且入口页网格重现

- **当** 用户正在查看 NH 入口子功能结果，「返回 NH 入口」按钮可见
- **且** 用户点击「返回 NH 入口」按钮
- **那么** 系统必须清空当前漫画结果、分页和错误状态
- **且** 「返回 NH 入口」按钮必须隐藏
- **且** NH 入口页网格（最近更新 / 热门排行 / 热门标签）必须重新展示而不发起网络内容请求

#### 场景:切换到其他来源时按钮隐藏

- **当** 用户正在查看 NH 入口子功能结果，「返回 NH 入口」按钮可见
- **且** 用户通过来源下拉切换到非 NH 来源
- **那么** 「返回 NH 入口」按钮必须隐藏

#### 场景:点击随机时按钮隐藏

- **当** 用户正在查看 NH 入口子功能结果，「返回 NH 入口」按钮可见
- **且** 用户点击「随机」按钮（仅适用于支持随机的来源；NH 本身不支持随机，故此场景描述用户先切到支持随机的来源或本场景作为按钮隐藏语义的对称约束）
- **那么** 「返回 NH 入口」按钮必须隐藏

#### 场景:入口子功能内关键词搜索无果时不重渲染入口页网格

- **当** 用户从 NH 入口页进入热门排行或热门标签子功能，「返回 NH 入口」按钮可见
- **且** 用户进一步执行关键词搜索且无结果
- **那么** 系统必须显示「暂无搜索结果」空状态而非 NH 入口页网格
- **且** 「返回 NH 入口」按钮必须仍然可见，允许用户一键回到入口页
