## 为什么

NH 来源用户从入口页进入「最近更新 / 热门排行 / 热门标签」任一子功能后，左上角会显示「返回 NH 入口」按钮。但用户此时再点一次「搜索」（输入关键词触发关键词搜索），按钮会错误消失——`SearchPage.handleSearch` 在每次搜索时强行重算 `viewingNhEntry`，关键词模式（`mode === 'keyword'`）使其被重置为 `false`，按钮跟着隐藏。同时这个 state 还兼任 `NhEntryGrid` 入口页网格的渲染开关，重置后若搜索无果，入口页网格会错误重新出现，构成二次 bug。根因是单个 state 承担了「控制返回按钮显隐」与「控制入口页网格渲染」两个职责，事件处理器对它的写入语义不一致。

## 变更内容

- 拆分 `viewingNhEntry` 的两个职责：新增独立的「返回按钮可见」标志，与「入口页网格可见」解耦。
- 「返回 NH 入口」按钮的显隐改为：用户进入任一 NH 入口子功能（最近更新 / 热门排行 / 热门标签）后置 `true`，仅在显式退出入口体系（点击「返回 NH 入口」、切换来源、点击随机）时置 `false`。关键词搜索、翻页、排行下拉切换、标签切换等「仍在入口子功能内」的操作不得隐藏它。
- `NhEntryGrid` 入口页网格的渲染仍由原 state 控制，仅在用户真正回到入口页或切来源时为 `true`。
- 修正 `handleSearch` 中误重置按钮 state 的逻辑。
- 新增并修正回归测试，覆盖：进入热门排行后做关键词搜索时按钮仍可见；进入后翻页按钮仍可见；切来源 / 点返回 / 点随机时按钮正确隐藏；入口页网格不被错误重渲染。

## 功能 (Capabilities)

### 新增功能

无。

### 修改功能

- `nh-entry-page`: 「返回 NH 入口」按钮的显隐语义从「派生自当前 mode」改为「进入入口子功能后持续可见，直到用户显式退出入口体系」。补充按钮持续可见的需求与场景，并明确入口页网格渲染开关与按钮开关是两个独立关注点。
