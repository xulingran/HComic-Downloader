## 修改需求

### 需求:拖动生命周期必须可靠清理

阅读器必须通过 pointer capture 在指针离开轨道后继续接收拖动事件。正常释放时必须结束拖动态并对有效最终页执行一次拖动完成回调；`pointercancel` 或丢失 pointer capture 时必须幂等清除拖动态与页追踪冻结，禁止让阅读器永久停留在拖动状态。当承载进度条的组件在拖动进行中卸载时（如阅读器因 store 驱动在 keep-alive 下退场），拖拽 hook 必须主动释放 pointer capture、复位拖动态与页追踪冻结，禁止遗留捕获导致组件重挂载后新滑块元素无法接收 pointer 事件。

#### 场景:在轨道外释放指针

- **当** 用户从进度条开始拖动、把指针移出轨道后释放
- **那么** 阅读器必须完成到受限最终页的跳转、结束拖动视觉状态，并允许后续页面追踪恢复

#### 场景:系统取消指针序列

- **当** 拖动过程中发生 `pointercancel` 或组件丢失 pointer capture
- **那么** 阅读器必须清除拖动态和冻结状态，后续再次拖动必须能够正常开始

#### 场景:组件在拖动进行中卸载

- **当** 用户正在拖动进度条，且承载进度条的组件因阅读器关闭或 store 状态切换被卸载
- **那么** 拖拽 hook 必须释放已设置的 pointer capture 并复位 `isDragging` 状态，使该组件在 keep-alive 场景下重新挂载时新的滑块元素可正常开始下一次拖动
