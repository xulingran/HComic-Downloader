## 修改需求

### 需求:安全重命名漫画资产

系统必须允许用户修改资产显示名称并选择同步重命名磁盘文件。新名称必须经过既有文件名清理和保留名校验；当选择同步重命名文件时，目标路径必须经 `realpath` 解析并断言仍位于当前漫画库根目录内，禁止仅依赖输入字符清洗。目标路径与源路径解析为同一路径时必须拒绝，禁止执行无意义重命名。目标路径已存在时必须拒绝覆盖。成功后必须原子更新索引、下载历史输出路径和阅读进度关联，资产 ID 保持不变。重命名失败时必须回滚已执行的文件移动和历史关联，错误消息必须只返回固定语义文案，禁止把内部 OS 错误或磁盘路径透传给渲染进程。

#### 场景:重命名 CBZ

- **当** 用户将某 CBZ 重命名为一个合法且无冲突的新名称
- **那么** 文件在同一漫画库根目录内完成重命名，索引和下载历史指向新路径，阅读进度继续关联原资产 ID

#### 场景:目标名称已存在

- **当** 新名称解析到的目标路径已存在
- **那么** 系统拒绝重命名，源文件及所有数据库记录保持不变

#### 场景:清洗后路径仍尝试逃逸漫画库根目录

- **当** 清洗后的新名称经 `realpath` 解析后落在漫画库根目录之外
- **那么** 系统在执行任何磁盘操作前拒绝重命名，源文件保持不变，错误消息不包含磁盘路径

#### 场景:新名称解析为与源相同路径

- **当** 用户提交的新名称经清洗和 `realpath` 解析后等于源资产当前路径
- **那么** 系统拒绝重命名并返回「新名称与原名相同」语义，禁止误报为目标冲突

#### 场景:重命名过程中磁盘或历史更新失败

- **当** 文件已移动但下载历史关联更新抛出异常
- **那么** 系统回滚文件移动和历史关联，向渲染进程返回固定语义的失败消息，内部 OS 错误只写入日志

## 修改需求

### 需求:所有管理操作必须限制在当前漫画库根目录

定位、健康检查、删除、重命名和元数据编辑必须只接受资产 ID，并在操作时解析真实路径、验证根目录包含关系和拒绝符号链接逃逸。重命名操作的源路径和目标路径必须各自经 `realpath` 解析并断言位于漫画库根目录内，目标路径不得与源路径相同。切换下载目录后，旧根目录资产 ID 禁止继续执行文件操作。

#### 场景:使用旧漫画库资产 ID

- **当** 下载目录已迁移或切换后，渲染进程提交旧根目录中的资产 ID 请求删除
- **那么** 后端拒绝操作并要求刷新漫画库，旧目录文件不被修改

#### 场景:重命名目标逃逸根目录

- **当** 重命名目标路径经 `realpath` 解析落在漫画库根目录之外
- **那么** 后端拒绝执行磁盘重命名，源文件与索引保持不变
