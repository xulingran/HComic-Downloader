## 上下文

本项目有 5 个漫画来源，其中 `jmcomic`（JMComic）是核心来源之一。当前代码中内部分布式使用 `jmcomic` 作为来源标识符，但用户界面展示为 "JMComic"（首字母大写）。从 `jmcomic` 到 `jm` 的重命名是一个全栈重构，涉及 Python 后端（解析器、下载器、配置、IPC）、Electron 主进程、React 前端、测试和文档，总计 80+ 个文件。

本次重构无功能变更，纯粹是标识符统一。

## 目标 / 非目标

**目标：**
- 将 Python 后端的 `jmcomic` 标识符全部替换为 `jm`（目录、模块、类名、字符串字面量、配置键、IPC 方法名）
- 将 TypeScript/前端中的 `jmcomic` 标识符全部替换为 `jm`（类型字段、IPC 通道、配置键、变量名、Hook 名、CSS 类名）
- 将配置持久化中的 `jmcomicDomain` 键迁移为 `jmDomain`，并兼容旧键
- 更新所有测试文件
- 更新所有文档

**非目标：**
- 不改变 API 行为或数据格式
- 不改变 `ComicSource` 联合类型的 display label（但 `SOURCE_META.label` 和历史页映射中的标签 "JMComic" → "JM" 属于本次变更范围）
- 不引入新的功能、依赖或架构变更
- 不重构或优化任何逻辑

## 决策

| # | 决策 | 理由 | 替代方案 |
|---|------|------|---------|
| D1 | 目录 `sources/jmcomic/` → `sources/jm/` | 保持目录名与来源标识符一致；Python 模块路径与标识符一一映射 | 保留目录名不变（不彻底）；拆分为 `sources/jm/` 子包（过于复杂） |
| D2 | Python 类 `JmParser` 名称保持不变 | `JmParser` 已经是 `Jm` 前缀，不需要改成 `JmParser` 再改一次；类名与用户展示无关 | 改成 `JmParser2`（无意义） |
| D3 | 配置旧键兼容：读取时 fallback | 用户已持久化的配置包含 `jmcomicDomain`，直接忽略会导致设置丢失；读时 fallback + 写时新键，渐进迁移 | 一次性迁移脚本（增加复杂度）；硬性要求重新配置（用户体验差） |
| D4 | IPC 通道名同步变更，不保留旧通道别名 | 旧通道名不会被任何外部调用使用，前后端同时部署无兼容问题 | 保留旧通道别名（增加维护负担） |
| D5 | 持久化数据（下载历史、收藏夹等）中的 `jmcomic` 来源标识符暂不迁移 | 数据量大且格式已稳定；来源标识符在序列化中仅作为标签，不影响功能；后续可做离线数据迁移 | 立即扫描全部历史数据（性能开销大）；增加复杂性 |
| D6 | 测试覆盖：分阶段替换，优先 Python 后端子模块级测试 | 确保目录重命名 + 导入路径变更后所有测试通过，再推进到前端和集成测试 | 一次性全部替换（风险高，难以定位问题） |
| D8 | 用户展示标签 "JMComic" → "JM" | 与内部标识符统一成 `jm`，保持短名称风格；用户界面上与 "哔咔"、"拷贝漫画" 长度一致 | 保留 "JMComic"（不彻底） |
| D9 | 标签变更直接改 `SOURCE_META.label` 和历史页映射 | 仅 2 处集中定义，改动极小，无需迁移 | 加一层映射兼容（过度设计） |

## 风险 / 权衡

| 风险 | 缓解措施 |
|------|---------|
| 目录重命名后 git 丢失历史追踪 | 使用 `git mv`；在 commit message 中注明 `jmcomic` → `jm` 目录重命名，Git 会自动检测 rename |
| `jmcomic` 字符串作为数据值（而非标识符）被误替换 | 严格区分「标识符替换」（如配置键、IPC 通道）和「数据值保留」（如用户显示标签 "JMComic"、HTML 夹具中的内容）；对每个替换做 code review |
| 旧配置 `jmcomicDomain` 在 fallback 期间多次读写的竞态 | config 原子写入保证一致性；fallback 在读取时一次性转换，回写在新键位置 |
| 测试夹具（HTML 文件）中的 `jm_*.html` 文件名与实际来源标识符解耦 | 夹具文件名为 `jm_*.html` 已使用 `jm_` 前缀，无需修改；仅更新测试代码中的引用 |
