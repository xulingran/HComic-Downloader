## 修改需求

### 需求:持久化本地阅读进度

系统必须按资产 ID 和章节 ID 持久化当前页、总页数和最后阅读时间，并在用户切页时节流保存、关闭阅读器时立即保存。最近阅读排序必须使用该记录；资产被应用内重命名时进度必须保留。当资产存在有效阅读进度时，系统必须在打开阅读器前同时提供“继续阅读”和“从头开始”选择；继续阅读必须恢复保存的章节和页码，从头开始必须进入第一章第一页。选择从头开始禁止在阅读器成功加载前删除或覆盖原进度，后续进度更新必须继续使用既有保存机制。

#### 场景:继续阅读单本漫画
- **当** 用户在第 35 页关闭本地漫画，之后选择“继续阅读”
- **那么** 阅读器恢复到第 35 页，并更新最后阅读时间

#### 场景:从头阅读单本漫画
- **当** 单本漫画保存了第 35 页进度且用户选择“从头开始”
- **那么** 阅读器从第 1 页打开，成功加载并产生阅读进度后才允许用本次会话位置覆盖旧进度

#### 场景:继续阅读多章节漫画
- **当** 多章节漫画保存了第二章第 12 页进度且用户选择“继续阅读”
- **那么** 阅读器直接打开第二章第 12 页，禁止先进入章节选择器或第一章

#### 场景:从头阅读多章节漫画
- **当** 多章节漫画存在阅读进度且用户选择“从头开始”
- **那么** 阅读器直接打开第一章第 1 页，禁止沿用保存章节或要求用户再次选择章节

#### 场景:没有阅读进度
- **当** 用户查看从未阅读过的本地漫画
- **那么** 系统只提供“开始阅读”入口，并从单本第 1 页或多章节第一章第 1 页打开

#### 场景:保存章节已经失效
- **当** 资产重新扫描后已保存的章节 ID 不再属于该资产
- **那么** 系统禁止恢复到错误章节，并安全回退到章节选择器；用户仍可选择从头开始进入当前第一章第 1 页

#### 场景:外部修改导致页数缩短
- **当** 已保存进度为第 35 页但重新扫描后资产仅剩 20 页
- **那么** 继续阅读的恢复页被限制为第 20 页，禁止请求越界页面

#### 场景:从头开始加载失败
- **当** 用户选择“从头开始”但第一章页面清单或第一页加载失败
- **那么** 系统必须保留原有阅读进度，并显示可重试错误，禁止把进度提前重置为第 1 页
