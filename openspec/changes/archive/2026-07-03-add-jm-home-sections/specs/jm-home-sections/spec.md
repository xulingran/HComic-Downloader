## 新增需求

### 需求:JM 空白关键词搜索必须返回首页栏目

当来源为 `jm`、搜索模式为 `keyword`、关键词 trim 后为空且请求第 1 页时，系统必须请求并解析 JM 首页栏目，禁止把该操作隐式替换为随机漫画。非空关键词、排行模式以及其他来源必须保持既有搜索语义。

#### 场景:空白关键词进入 JM 首页
- **当** 已认证用户以 `source="jm"`、`mode="keyword"`、空白关键词和 `page=1` 发起搜索
- **那么** 后端请求当前 JM 域名根路径并返回首页栏目结果
- **且** 禁止请求 `/albums/random`

#### 场景:切换到 JM 不再自动随机
- **当** 用户在搜索页主动切换到已认证的 JM 来源
- **那么** 前端发起 JM 空白关键词第 1 页搜索
- **且** 禁止调用 random IPC

#### 场景:显式随机按钮保持随机语义
- **当** 用户在 JM 来源点击随机按钮
- **那么** 系统仍通过 random IPC 请求 `/albums/random`
- **且** 禁止把显式随机请求替换为首页栏目请求

#### 场景:普通 JM 搜索不受影响
- **当** 用户提交非空 JM 关键词或排行模式查询
- **那么** 系统继续使用既有关键词、ID 直搜或排行路径
- **且** 返回结果不携带首页栏目分组

### 需求:JM 首页必须只保留兼容漫画下载模型的栏目

系统必须按首页 DOM 中栏目标题与紧邻内容容器的关系解析栏目，并复用现有 JM 漫画卡片解析逻辑。默认栏目集合必须按首页顺序覆盖连载更新、禁漫汉化组、最新韩漫、推荐本本和最新漫画五类内容，每栏最多保留前 10 本；实际显示标题必须采用页面提供的标题，以兼容星期或届次变化。没有标准 `/album/{id}` 卡片的栏目必须被排除。

#### 场景:解析五个默认漫画栏目
- **当** 首页同时包含连载更新、禁漫汉化组、最新韩漫、推荐本本和最新漫画栏目
- **那么** 返回栏目保持上述首页出现顺序
- **且** 每个栏目最多包含前 10 个可解析漫画 ID

#### 场景:动态栏目标题原样保留
- **当** 首页标题因日期或活动变化为“周五連載更新”或“C107&推薦本本”等动态文本
- **那么** 返回栏目标题使用页面中的规范化可见文本
- **且** 栏目识别禁止仅依赖完整标题字符串相等

#### 场景:排除不兼容内容栏目
- **当** 首页包含禁漫书库、禁漫小说或其他没有 `/album/{id}` 漫画卡片的栏目
- **那么** 这些栏目不出现在搜索结果的栏目数组中
- **且** 其链接禁止被转换为可下载 `ComicInfo`

#### 场景:单个损坏卡片不影响其余栏目
- **当** 某栏目中个别卡片缺少漫画链接或无法解析
- **那么** 系统跳过该卡片并继续解析当前栏目和后续栏目
- **且** 仅在栏目没有任何有效漫画时省略整个栏目

### 需求:首页栏目搜索结果必须使用去重漫画与 ID 引用契约

共享 `SearchResult` 必须允许携带可选 `sections` 数组；每个栏目必须包含标题与有序 `comicIds`，所有被引用 ID 必须存在于顶层 `comics` 数组。顶层漫画必须按首次出现顺序去重，使过滤、批量选择和下载操作以唯一漫画集合运行；普通搜索结果必须保持向后兼容。

#### 场景:跨栏目重复漫画只进入顶层一次
- **当** 同一 JM 漫画同时出现在两个首页栏目
- **那么** 两个栏目的 `comicIds` 均可引用该漫画 ID
- **且** 顶层 `comics` 数组只包含该漫画一次

#### 场景:栏目引用不存在的漫画被禁止
- **当** 后端构造栏目搜索结果
- **那么** 每个 `comicIds` 元素都必须能在顶层 `comics` 中找到对应 JM 漫画
- **且** 禁止返回悬空、空白或重复的栏目内 ID

#### 场景:首页结果使用单页分页信息
- **当** JM 首页栏目解析成功
- **那么** 搜索结果分页必须为当前页 1、总页数 1
- **且** 搜索页禁止为首页栏目展示翻页控件或触发相邻页预加载

#### 场景:普通结果保持原有契约
- **当** 搜索结果不是 JM 首页栏目结果
- **那么** `sections` 必须省略或为空
- **且** 现有 `comics` 与 `pagination` 消费方无需栏目数据即可继续工作

### 需求:搜索页必须按栏目展示并完整恢复 JM 首页结果

搜索页收到非空 `sections` 时必须按栏目顺序分别展示标题和漫画卡片；过滤、推荐高亮、屏蔽占位、下载状态、阅读、批量选择必须复用现有卡片行为。栏目描述必须与漫画和分页一起进入搜索缓存，任何非栏目结果提交时必须清除旧栏目状态。

#### 场景:栏目结果分组渲染
- **当** JM 首页结果包含多个栏目
- **那么** 搜索页按返回顺序渲染每个栏目标题及其 `comicIds` 对应的卡片
- **且** 禁止把所有卡片退化为一个无标题平铺列表

#### 场景:跨栏目批量选择按唯一漫画计算
- **当** 同一本漫画在多个栏目中可见且用户选择该漫画
- **那么** 所有对应卡片反映同一选中状态
- **且** 批量下载集合只包含该漫画一次

#### 场景:栏目结果参与过滤和高亮
- **当** 首页漫画命中标签黑名单、推荐标签或活跃下载状态
- **那么** 栏目中的卡片必须呈现与普通搜索结果相同的屏蔽占位、推荐高亮和下载进度

#### 场景:缓存恢复栏目结构
- **当** 用户离开搜索页后返回已缓存的 JM 空白关键词上下文
- **那么** 系统即时恢复漫画、分页和栏目顺序
- **且** 禁止仅恢复顶层漫画而丢失栏目标题

#### 场景:普通搜索清除旧栏目
- **当** 用户从 JM 首页栏目提交普通关键词、排行、随机请求或切换到其他来源
- **那么** 新结果提交时必须清除上一结果的栏目状态
- **且** 禁止在普通结果上残留旧栏目标题或旧栏目引用

### 需求:JM 首页请求必须复用认证代理与挑战恢复边界

JM 首页网络请求必须复用 `JmParser` 的同一 Session、系统代理、Cookie、User-Agent、域名和现有结构化挑战检测。用户主动切换来源或提交空白搜索时允许显式启用交互恢复；启动自动加载、缓存刷新和预加载必须保持非交互，禁止自动弹出验证窗口。

#### 场景:首页请求使用同一 Session
- **当** 后端请求 JM 首页
- **那么** 请求必须由已调用 `apply_system_proxy_to_session()` 的 JM parser Session 发出
- **且** 携带当前运行期认证上下文和首页 Referer

#### 场景:首页挑战产生结构化错误
- **当** JM 首页响应包含 `cf-mitigated: challenge` 或稳定挑战页标记
- **那么** parser 必须抛出含首页 URL 的 `AntiBotChallengeError`
- **且** 禁止把挑战页解析为空栏目结果

#### 场景:用户主动空白搜索可交互恢复
- **当** 用户主动切换到 JM 或点击搜索提交空白关键词，并收到结构化挑战错误
- **那么** 搜索调用可以 `allowInteractiveChallenge=true` 进入现有 JM 搜索挑战恢复编排
- **且** 自动重试仍必须保持一次性和有界

#### 场景:后台首页加载禁止弹窗
- **当** JM 首页请求来自启动自动加载、缓存后台刷新或预加载
- **那么** 调用必须省略交互标志或传入 `allowInteractiveChallenge=false`
- **且** 即使收到挑战错误也禁止打开验证窗口
