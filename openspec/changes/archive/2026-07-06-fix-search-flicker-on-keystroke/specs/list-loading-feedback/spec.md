## 新增需求

### 需求:未提交的输入变化必须保持结果列表渲染稳定

当用户在搜索栏输入或删除文字但尚未提交搜索（未按 Enter、未点击搜索按钮、未触发任何会产生新结果集的动作）时，已加载的结果列表**必须**保持渲染稳定：结果容器的 React `key` **禁止**因实时输入值的变化而改变，已加载的卡片**禁止**发生整体卸载/重挂载，卡片进出场动画（enter/exit）**禁止**被重放。结果容器的 `key` **必须**派生自「当前已加载结果所属的查询上下文」（即最近一次成功完成搜索时记录的 query/mode/source/searchTags 组合），而**禁止**派生自实时输入值。

#### 场景:打字输入新字符时结果列表不闪烁

- **当** 用户在搜索栏已加载某查询上下文的结果列表后，连续输入若干字符修改 `query` 但未按 Enter、未点击搜索按钮
- **那么** 结果容器的 `key` 保持不变（与已加载上下文一致）
- **且** 已显示的卡片不发生卸载/重挂载
- **且** 卡片进出场动画不被重放（无闪烁）

#### 场景:删除搜索栏文字时结果列表不闪烁

- **当** 用户在搜索栏已加载某查询上下文的结果列表后，通过退格键删除若干字符修改 `query` 但未提交搜索
- **那么** 结果容器的 `key` 保持不变
- **且** 已显示的卡片不发生卸载/重挂载
- **且** 卡片进出场动画不被重放

#### 场景:提交搜索后容器 key 按预期切换

- **当** 用户在搜索栏修改 `query` 后按 Enter 或点击搜索按钮，新搜索成功完成并加载了新结果
- **那么** 结果容器的 `key` 切换为新的已加载查询上下文
- **且** 这一切换是预期的整页替换行为（与切换来源、新查询的现有契约一致）

#### 场景:首次进入页面无搜索时容器拥有稳定占位 key

- **当** 用户首次进入搜索页且尚未发起任何搜索（`loadedContextKey` 为空）
- **那么** 结果容器使用一个稳定的占位 `key`（不派生自实时输入）
- **且** 该占位 key 在用户输入未提交的 query 期间保持不变

#### 场景:翻页时容器 key 仍按页码切换

- **当** 用户在已加载某查询上下文的结果列表中翻页（同一上下文、不同页码）
- **那么** 结果容器的 `key` 因页码部分变化而切换（保留现有「翻页 remount 以规避 framer-motion 测量竞态」的设计）
- **且** 这一行为与「未提交输入不变 key」的契约不冲突（翻页是显式数据变更动作，不是实时输入）
