## 修改需求

### 需求:存储空间分析
系统必须能够扫描下载目录统计总体空间占用，并按来源、格式、作者等维度分组。`orphanFiles` 必须仅统计 `temp_*` 目录（与「孤儿临时目录清理」面板的孤儿定义一致）；非 `temp_*` 且不在 `download_history.output_path` 中的资产必须计入 `untrackedFiles` 字段，禁止将其误判为可清理孤儿。

来源识别必须采用多级回退策略，确保多章节专辑的根目录能正确继承其子章节记录的来源：优先用资产路径精确匹配 `download_history.output_path`；若未命中，必须再尝试用资产路径作为父目录，匹配任一以该路径为前缀的子章节历史记录；前两级均未命中时，才允许回退到路径名启发式（`_infer_source_site`）；启发式仍无法识别时，归入 `unknown`。

#### 场景:获取总体存储统计
- **当** 用户进入维护中心的存储分析页
- **那么** 系统返回 `totalSizeBytes`、`totalFiles`、`orphanFiles` 以及 `untrackedFiles` 概览

#### 场景:按格式分布
- **当** 下载目录中包含 `folder`、`cbz`、`zip` 三种格式
- **那么** 系统正确分别统计每种格式占用的字节数

#### 场景:按来源分布
- **当** 漫画资产的来源可被识别
- **那么** 系统按 `source_site` 分组统计；无法识别时归入 `unknown`

#### 场景:多章节专辑根目录继承子章节来源
- **当** 下载目录中存在一个 folder 格式的专辑根目录，且其下每个章节子目录在 `download_history.output_path` 中有对应记录（记录的 `source_site` 非空）
- **那么** 扫描器必须通过「资产路径作为父目录」的回退匹配命中任一子章节记录，并将该记录的 `source_site` 作为专辑根目录的来源，禁止将其归入 `unknown`

#### 场景:来源回退优先级
- **当** 某资产的路径既不等于任何 `output_path`，也不是任何 `output_path` 的父目录
- **那么** 系统必须依次尝试路径名启发式（`_infer_source_site`）；启发式仍失败时才归入 `unknown`，禁止跳过启发式直接归入 `unknown`

#### 场景:父目录回填避免覆盖精确匹配
- **当** 某资产路径能精确匹配到 `download_history.output_path`
- **那么** 必须优先使用精确匹配的记录元数据，禁止被父目录回退逻辑覆盖

#### 场景:按作者分布
- **当** 漫画资产的作者可从 `ComicInfo.xml` 或文件名模板中解析
- **那么** 系统按作者分组并返回占用空间 Top 20

#### 场景:识别 Top 大文件
- **当** 系统完成下载目录扫描
- **那么** 按单个资产大小排序，返回 Top 20 大文件及其路径、标题、作者、大小和页数

#### 场景:orphanFiles 仅计 temp 目录
- **当** 下载目录中存在 `temp_*` 目录
- **那么** 系统将其大小与数量计入 `orphanFiles`；非 `temp_*` 的资产禁止计入 `orphanFiles`

#### 场景:untrackedFiles 计未在历史的非临时资产
- **当** 某个非 `temp_*` 资产（folder / cbz / zip）不在 `download_history.output_path` 中
- **那么** 系统将其大小与数量计入 `untrackedFiles`，前端展示为"未在历史记录中"并提示"删除请谨慎"，禁止使用"孤儿"措辞

#### 场景:文件名解析剥离括号分组
- **当** 文件名为 `[Author] Title [1]` 或 `(Author) Title` 等带前导方/圆括号分组的形式
- **那么** 解析作者时必须先剥离前导 `[...]` / `(...)` 分组，再按 `-` 分隔作者与标题，禁止把整段含括号的字符串作为作者名
