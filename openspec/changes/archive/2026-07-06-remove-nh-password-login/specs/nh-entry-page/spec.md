## 修改需求

### 需求:NH 来源入口页

系统必须将 NH 入口页作为该来源的显式初始路由。用户切换到 NH，或应用冷启动且默认来源为 NH、同时没有可恢复的 NH 搜索缓存时，系统必须清空旧来源结果并展示入口页；禁止自动执行搜索、认证预校验或随机请求。入口页必须提供最近更新、热门排行和热门标签快捷入口。存在 keep-alive 搜索缓存时可以恢复用户之前的 NH 结果，但必须保留返回入口页的操作。NH 认证状态仅由 API Key 决定。

#### 场景:从其他来源切换到 NH

- **当** 用户在搜索页从任意其他来源选择 NH
- **那么** 系统必须清空旧来源漫画、分页、错误、查询词和已选标签
- **且** 必须展示含「最近更新」「热门排行」和热门标签区域的 NH 入口页
- **且** 禁止调用随机接口或自动加载最近更新

#### 场景:已配置 API Key 时切换到 NH

- **当** 用户已配置有效 NH API Key，并从其他来源切换到 NH
- **那么** 系统仍必须先展示 NH 入口页
- **且** 禁止因为认证有效而自动调用随机、搜索或收藏接口

#### 场景:未配置 API Key 时切换到 NH

- **当** 用户未配置 NH API Key 并切换到 NH
- **那么** 系统必须允许进入 NH 入口页并使用匿名浏览能力
- **且** 禁止显示全页登录阻断

#### 场景:冷启动默认来源为 NH

- **当** 应用冷启动读取到 `defaultSource="nh"` 且没有可恢复的搜索缓存
- **那么** 系统必须直接展示 NH 入口页
- **且** 禁止自动执行空关键词搜索或随机请求

#### 场景:恢复 NH 搜索缓存

- **当** SearchPage 通过 keep-alive 恢复到先前的 NH 最近更新、热门排行或标签结果
- **那么** 系统可以恢复缓存结果及其查询上下文
- **且** 必须提供“返回 NH 入口”操作

#### 场景:进入最近更新

- **当** 用户点击 NH 入口页的「最近更新」入口
- **那么** 系统必须以 NH 来源请求第一页最近更新结果
- **且** 请求必须使用空关键词的普通搜索语义
- **且** 结果加载后入口页必须隐藏并展示漫画列表

#### 场景:进入热门排行

- **当** 用户点击 NH 入口页的「热门排行」入口
- **那么** 系统必须以 NH 来源请求第一页 popular 排序结果
- **且** 前端搜索上下文必须能区分热门排行与最近更新，避免分页或缓存混用
- **且** 结果加载后入口页必须隐藏并展示漫画列表

#### 场景:返回入口页

- **当** 用户正在查看 NH 最近更新、热门排行或标签快捷入口结果并点击返回入口
- **那么** 系统必须清空当前漫画结果、分页和错误状态
- **且** 必须重新展示 NH 入口页而不发起网络内容请求
