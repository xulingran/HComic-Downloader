## 修改需求

### 需求:设置页必须回填已保存的账号密码

对于支持账号密码登录的来源（hcomic、moeimg、bika），设置页中的用户名和密码输入框必须显示当前配置中已保存的对应值，使用户无需重新输入即可再次点击登录。NH 不再属于账号密码登录来源，配置响应和设置页禁止提供 NH 用户名或密码回填；NH API Key 禁止明文回填。

#### 场景:配置异步加载后用户名正确回填

- **当** 用户打开设置页且后端配置中包含 `hcomicUsername`/`moeimgUsername`/`bikaUsername`
- **那么** 对应来源的用户名输入框必须在配置加载完成后显示保存的值

#### 场景:配置异步加载后密码正确回填

- **当** 用户打开设置页且后端配置中包含 `hcomicPassword`/`moeimgPassword`/`bikaPassword`
- **那么** 对应来源的密码输入框必须在配置加载完成后显示保存的值

#### 场景:登录失败后重新进入设置页凭据仍可见

- **当** 用户此前因网络异常或密码错误导致 hcomic、moeimg 或 bika 的账号密码登录失败，且凭据已被持久化
- **那么** 再次打开设置页时，对应账号和密码输入框必须显示上次提交的值

#### 场景:NH 不返回账号密码字段

- **当** 前端读取应用配置
- **那么** 配置契约中禁止包含 `nhUsername` 或 `nhPassword`
- **且** NH 设置区域禁止渲染对应输入框

#### 场景:NH API Key 不明文回填

- **当** 后端配置中已保存 NH API Key 且用户重新打开设置页
- **那么** 系统必须通过 NH 已认证状态和登录校验反映凭证已保存
- **且** 禁止将完整 API Key 明文填入 API Key 输入框
