# page-keep-alive 规范

## 新增需求

### 需求:已访问的 tab 页面必须保持组件实例存活，切走不卸载

系统必须在 tab 切换时保持已访问页面的 React 组件实例存活——切走一个页面时**禁止**卸载其组件树，**必须**改用 CSS `display: none` 将其移出渲染树；切回时**必须**复用原有实例（`display: block`），**禁止**重新挂载。这保证再次切回不发生 React mount、不重播列表 stagger 动画、不丢失页面内本地状态（如输入框内容、滚动位置、展开状态）。

#### 场景:切回已访问的搜索页时列表不重新飞入

- **当** 用户在搜索页看到卡片列表，切到下载页，再切回搜索页
- **那么** 搜索页组件实例被复用，卡片列表保持原样显示
- **且** 不触发 `AnimatedCardWrapper` 的进出场 stagger 动画（无卡片重新飞入）

#### 场景:切回已访问页面时本地状态保留

- **当** 用户在某个页面的输入框输入了内容、展开了某个折叠面板，切走再切回
- **那么** 输入框内容、折叠面板的展开状态完整保留
- **且** 滚动位置恢复到切走时的位置

#### 场景:切回搜索页不重复触发首屏搜索

- **当** 搜索页已有缓存数据（`useSearchCacheStore` 中存在当前 context），用户切走再切回
- **那么** 搜索页的 mount effect **禁止**重复触发（组件未重挂）
- **且** 列表直接从存活实例的状态渲染，不调用任何搜索 IPC

### 需求:未访问的页面必须懒创建，禁止预创建所有页面实例

系统**必须**采用懒创建策略——只有当用户首次访问某个 tab 时，才将该页面的组件实例加入存活集合并挂载；用户从未访问过的页面（如 Toolbox、About）**禁止**提前创建实例。存活集合初始**必须**只包含首屏页面（`search`）。

#### 场景:应用启动时只有搜索页实例存在

- **当** 应用完成启动、首屏渲染就绪
- **那么** 存活页面集合只包含 `search`，搜索页组件实例已挂载
- **且** 其他 7 个页面（downloads/favourites/history/settings/toolbox/maintenance/about）均未创建实例、未加载 chunk（lazy 页面）

#### 场景:首次访问设置页才创建实例并加载 chunk

- **当** 用户首次点击侧边栏「设置」
- **那么** 设置页加入存活集合，首次挂载其组件实例
- **且** 若设置页是 lazy 组件，触发其 chunk 加载
- **且** 此后再切走设置页，实例保持存活不卸载

### 需求:首次进入页面必须直接渲染真实内容，禁止显示骨架兜底

系统在页面**首次挂载**（懒创建触发）时，**必须**直接渲染真实页面内容，**禁止**在 tab 切换动画期间显示骨架兜底。首次挂载的 chunk 由 `react-code-splitting` 规范的空闲预热机制提前加载，数据走 store 缓存快路径渲染，mount 成本可控，无需骨架兜底（避免骨架闪现的视觉负担）。

**例外**：lazy 页面首次加载 chunk 时（仅未预热的低频页面，如 Toolbox/About），仍由 `<Suspense fallback={<PageSkeleton/>}>` 兜底——这是 React.lazy 的标准加载行为，非 deferred mount。

#### 场景:首次切到已预热的下载页直接渲染真实内容

- **当** 用户首次点击「下载」tab（下载页实例尚未创建，但 chunk 已由 idle prefetch 预热）
- **那么** tab 切换动画期间直接渲染真实的 `DownloadPage` 组件
- **且** 不显示 `PageSkeleton` 骨架
- **且** 真实内容随切换动画（slide + fade）淡入

#### 场景:首次切到未预热的低频页面由 Suspense 兜底

- **当** 用户首次点击未预热的低频 tab（如 Toolbox），chunk 尚未加载
- **那么** 在 chunk 加载期间显示 `Suspense fallback`（`PageSkeleton`）
- **且** chunk 加载完成后渲染真实页面
- **且** 这是 React.lazy 的标准加载行为，仅在未预热的低频页面出现一次

#### 场景:再次切回已访问页面无骨架、无重挂

- **当** 用户切回已访问过的页面（实例已存活）
- **那么** 不显示任何骨架
- **且** 直接通过 `display` 切换显示存活实例的已有内容
- **且** 不发生 React mount、不重播 stagger

### 需求:keep-alive 下切回页面必须支持轻量刷新钩子

系统在 keep-alive 模式下（页面不再重挂、mount effect 不重复触发），**必须**为需要同步后台状态的页面提供切回刷新能力。App 层**必须**向页面注入 `isActive` prop（指示该页是否为当前激活 tab）；需要刷新的页面**必须**在 `isActive` 从 false 变为 true 时触发一次轻量刷新。

**约束**：仅下载页需要主动刷新（同步后台下载任务状态）；其余页面**禁止**在切回时主动发起网络请求，**必须**依赖其 zustand store 缓存与后台订阅保证数据新鲜度。

#### 场景:切回下载页时重拉任务列表

- **当** 用户从其他 tab 切回下载页（`isActive` 变为 true）
- **那么** 下载页触发一次 `loadDownloads()` 刷新任务列表
- **且** 刷新期间显示已有数据（不显示骨架），刷新结果增量更新到列表

#### 场景:切回收藏页不发起网络请求

- **当** 用户从其他 tab 切回收藏页
- **那么** 收藏页**禁止**主动发起加载收藏列表的请求
- **且** 直接显示 `useFavouritesStore` 缓存的多页数据
- **且** `onDownloadProgress` 订阅在 keep-alive 期间未中断，下载完成状态已实时同步

### 需求:keep-alive 必须使用 display:none 隐藏离屏页面

系统隐藏非激活页面时**必须**使用 `display: none`，**禁止**使用 `visibility: hidden`（仍占布局空间）或 `opacity: 0`（仍参与渲染、浪费合成层内存）。`display: none` 让浏览器完全跳过该子树的 layout 与 paint，是性能最优的隐藏方式。

#### 场景:非激活页面不占布局空间

- **当** 下载页是当前激活页，搜索页处于 keep-alive 隐藏态
- **那么** 搜索页子树应用 `display: none`
- **且** 搜索页不参与页面布局计算、不占用可视空间

#### 场景:非激活页面不参与渲染

- **当** 一个页面处于隐藏态
- **那么** 浏览器跳过该页面子树的 paint
- **且** 该页面不消耗 GPU 合成层资源

### 需求:首次进入页面的 chunk 必须已就绪以保证即时渲染

系统在页面首次挂载渲染真实内容时，**必须**确保高频页面的 lazy chunk 已加载完成（由 `react-code-splitting` 规范的空闲预热机制保证），**禁止**因 chunk 未预热而导致首次进入出现明显的下载等待——否则用户切到高频页面时会看到长时间的空白或 Suspense 加载态。

#### 场景:预热完成后首次进入高频页面即时渲染

- **当** idle prefetch 已完成（高频 chunk 在内存），用户首次点击高频 tab
- **那么** 页面首次挂载时直接渲染真实内容，无 chunk 下载等待
- **且** 真实内容随 tab 切换动画淡入

#### 场景:预热未覆盖的低频页面首次进入由 Suspense 兜底

- **当** 用户首次点击未预热的低频 tab（如 Toolbox），chunk 尚未加载
- **那么** 在 chunk 加载期间显示 `Suspense fallback`（`PageSkeleton`）
- **且** chunk 加载完成后渲染真实页面
