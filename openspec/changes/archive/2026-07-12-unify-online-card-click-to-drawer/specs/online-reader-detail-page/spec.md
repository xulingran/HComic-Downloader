## MODIFIED Requirements

### 需求:详情尾页必须覆盖详情抽屉的完整内容与操作

详情尾页必须与漫画详情抽屉共享同一套详情内容、补全状态和操作语义，包括标题、作者、来源、页数、章节数、更新日期、语言、分类、打开原网页、收藏状态与切换、原著、角色、制作组、标签、详情补全、补全失败重试、标签搜索、推荐与屏蔽、确认操作及用户反馈。详情尾页禁止通过复制一套独立业务逻辑造成与抽屉行为漂移，但不需要显示抽屉遮罩、右滑外壳或抽屉关闭栏。

详情尾页（`surface='reader'`）与详情抽屉（`surface='drawer'`）共享 `ComicDetailSurface` 组件，但两处的内容**必须**在以下方面存在有意的差异：

- **封面图**：详情抽屉**必须**展示漫画封面图（遵循 SFW 门控）；详情尾页**禁止**展示封面图（用户已在阅读器内，封面无意义）。
- **"开始阅读"入口**：详情抽屉**必须**提供"开始阅读"入口以启动预览阅读器；详情尾页**禁止**展示该入口（用户已在阅读器内，启动入口无意义）。
- **历史续读**：详情抽屉在存在 `resumeInfo`（来自历史记录）时**必须**提供"继续阅读"与"从头开始"两个入口；详情尾页不涉及续读入口。

这些差异由 `surface` prop 门控，**禁止**在 `surface='reader'` 分支渲染封面图或"开始阅读"入口。

#### 场景:展示完整基础元数据

- **当** 详情尾页接收到含作者、分类、日期、语言、原著、角色、制作组和标签的漫画
- **那么** 尾页必须展示与详情抽屉相同的全部可用字段和来源/页数/章节信息
- **并且** 必须提供打开原网页操作

#### 场景:详情抽屉展示封面图并遵循 SFW 门控

- **当** 详情抽屉（`surface='drawer'`）打开且 SFW 模式已开启
- **那么** 抽屉封面区**必须**显示 SFW 占位符（`📖`）
- **并且** **禁止**发起封面图片请求或显示真实封面
- **当** 详情抽屉打开且 SFW 模式已关闭
- **那么** 抽屉封面区**必须**显示真实漫画封面图（复用 `CoverImage` / `useCoverImage` 的既有加载与 SFW 门控机制）

#### 场景:详情抽屉提供开始阅读入口

- **当** 详情抽屉（`surface='drawer'`）打开且当前漫画可启动预览阅读器
- **那么** 抽屉**必须**提供"开始阅读"入口
- **并且** 点击该入口**必须**关闭详情抽屉并启动在线预览阅读器（`ComicReaderModal`）
- **并且** 详情尾页（`surface='reader'`）**禁止**渲染该入口

#### 场景:详情抽屉在有历史续读信息时提供继续阅读

- **当** 详情抽屉通过 `useDrawerStore.resumeInfo` 接收到有效的 `lastPage` / `lastChapterId`
- **那么** 抽屉**必须**同时显示"继续阅读"与"从头开始"两个入口
- **并且** "继续阅读"**必须**以传入的 `lastPage` / `lastChapterId` 启动预览阅读器
- **并且** "从头开始"**必须**从第 0 页启动预览阅读器

#### 场景:详情抽屉在无历史续读信息时提供单一开始阅读

- **当** 详情抽屉未接收到 `resumeInfo`（来自搜索页/收藏页）
- **那么** 抽屉**必须**仅显示单一"开始阅读"入口
- **并且** 点击该入口**必须**从第 0 页启动预览阅读器

#### 场景:缺失详情时按需补全

- **当** 当前来源需要详情补全或入口漫画缺少标签，且详情尾页进入或接近可见区域
- **那么** 系统必须通过既有漫画详情 IPC 补全数据并更新尾页
- **并且** 在尾页尚不可见时禁止仅因连续滚动列表已挂载而提前发起补全请求

#### 场景:详情补全失败后重试

- **当** 详情补全抛出异常或返回空漫画
- **那么** 尾页必须显示可见的加载失败状态和重试入口
- **并且** 用户点击重试后必须只重新执行详情补全，不关闭阅读器或重置阅读进度

#### 场景:收藏与标签操作保持一致

- **当** 用户在详情尾页加入或移除收藏，或对标签执行推荐、取消推荐、屏蔽、取消屏蔽
- **那么** 系统必须遵循详情抽屉既有的来源能力门控、服务端结果提交、互斥检查、确认弹窗和成功/失败反馈

#### 场景:详情搜索路由

- **当** 用户点击作者、分类、原著、角色或制作组执行替换式搜索
- **那么** 系统必须写入对应搜索条件并关闭在线阅读器，以显示搜索结果页
- **当** 用户点击普通标签执行追加式搜索
- **那么** 系统必须追加该标签、保留阅读器打开并显示操作反馈
