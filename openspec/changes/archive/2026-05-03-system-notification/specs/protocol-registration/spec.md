## 新增需求

### 需求:Windows URI 协议注册
系统必须支持在 Windows 上注册自定义 URI 协议 `hcomic://`，用于通知点击回调。

#### 场景:注册协议
- **当** 用户在设置面板中点击 "注册通知协议" 按钮
- **那么** 系统在注册表中写入 `hcomic://` 协议配置，指向当前可执行文件

#### 场景:注册成功
- **当** 协议注册成功
- **那么** 系统更新状态显示为 "已注册" 并提示用户

#### 场景:注册失败
- **当** 协议注册失败（如权限不足）
- **那么** 系统提示用户失败原因

### 需求:协议状态检测
系统必须能够检测 URI 协议是否已注册。

#### 场景:检测已注册
- **当** 系统检测到 `hcomic://` 协议已存在于注册表中
- **那么** 系统显示状态为 "已注册"

#### 场景:检测未注册
- **当** 系统检测到 `hcomic://` 协议不存在
- **那么** 系统显示状态为 "未注册"

### 需求:协议注册按钮
系统必须在设置面板中提供 "注册通知协议" 按钮，仅在 Windows 平台显示。

#### 场景:Windows 显示按钮
- **当** 系统运行在 Windows 平台
- **那么** 设置面板显示 "注册通知协议" 按钮和状态信息

#### 场景:非 Windows 隐藏按钮
- **当** 系统运行在 macOS 或 Linux 平台
- **那么** 设置面板不显示 "注册通知协议" 相关 UI

### 需求:单实例检测
系统必须在点击通知时检测是否已有实例运行，避免重复启动。

#### 场景:已有实例运行
- **当** 用户点击通知且已有实例运行
- **那么** 新实例激活已有窗口后退出

#### 场景:无实例运行
- **当** 用户点击通知且无实例运行
- **那么** 系统正常启动并显示窗口
