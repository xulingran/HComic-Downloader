## 新增需求

### 需求:点击通知打开应用窗口
系统必须支持用户点击通知后将应用窗口置于前台。

#### 场景:Windows 点击通知
- **当** 用户在 Windows 上点击通知
- **那么** 系统通过 URI 协议激活已有实例，将窗口置于前台

#### 场景:macOS 点击通知
- **当** 用户在 macOS 上点击通知
- **那么** 系统通过 pyobjc delegate 回调将窗口置于前台

#### 场景:Linux 点击通知
- **当** 用户在 Linux 上点击通知
- **那么** 系统通过 D-Bus ActionInvoked 信号将窗口置于前台

### 需求:窗口提升
系统必须将窗口恢复到正常状态并置于前台。

#### 场景:窗口最小化时提升
- **当** 窗口处于最小化状态且需要提升
- **那么** 系统恢复窗口状态、提升窗口层级、强制获取焦点

#### 场景:窗口隐藏时提升
- **当** 窗口处于隐藏状态且需要提升
- **那么** 系统显示窗口、提升窗口层级、强制获取焦点

### 需求:Linux 工具检测
系统必须检测 wmctrl 或 xdotool 是否存在，不存在则跳过窗口提升并提示用户。

#### 场景:wmctrl 存在
- **当** 系统检测到 wmctrl 可用
- **那么** 系统使用 wmctrl 提升窗口

#### 场景:xdotool 存在
- **当** 系统检测到 xdotool 可用且 wmctrl 不可用
- **那么** 系统使用 xdotool 提升窗口

#### 场景:工具不存在
- **当** wmctrl 和 xdotool 都不存在
- **那么** 系统跳过窗口提升并记录提示日志
