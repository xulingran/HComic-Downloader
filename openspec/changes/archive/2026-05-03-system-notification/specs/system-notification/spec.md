## 新增需求

### 需求:系统通知
系统必须在下载队列完成后发送系统级通知，支持 Windows、macOS、Linux 三平台。

#### 场景:全部成功通知
- **当** 下载队列全部完成且无失败
- **那么** 系统发送通知，标题为 "HComic Downloader"，正文包含成功数量，播放默认声音

#### 场景:有失败通知
- **当** 下载队列完成且存在失败
- **那么** 系统发送通知，标题为 "HComic Downloader"，正文包含成功和失败数量，列出前 3 个失败漫画名和原因，播放警告声音

#### 场景:超长内容截断
- **当** 漫画名超过 20 字符或失败原因超过 40 字符
- **那么** 系统截断超长部分并添加 "..."

### 需求:通知时机配置
系统必须允许用户配置通知时机，支持两种模式：仅窗口非活动时、始终通知。

#### 场景:仅窗口非活动时通知
- **当** 用户设置通知时机为 "inactive" 且窗口处于前台且有焦点
- **那么** 系统不发送通知

#### 场景:窗口最小化时通知
- **当** 用户设置通知时机为 "inactive" 且窗口最小化
- **那么** 系统发送通知

#### 场景:始终通知
- **当** 用户设置通知时机为 "always"
- **那么** 系统无论窗口状态如何都发送通知

### 需求:通知开关
系统必须允许用户开启或关闭系统通知功能。

#### 场景:关闭通知
- **当** 用户关闭 "下载完成时发送系统通知" 开关
- **那么** 系统不发送任何系统通知

#### 场景:开启通知
- **当** 用户开启 "下载完成时发送系统通知" 开关
- **那么** 系统在下载队列完成后发送通知

### 需求:平台降级
系统必须在首选通知库不可用时自动降级到备选方案。

#### 场景:Windows 降级
- **当** winotify 不可用
- **那么** 系统尝试 PowerShell BurntToast，再尝试 PowerShell 原生，最后使用 ctypes MessageBox

#### 场景:macOS 降级
- **当** pyobjc 不可用或权限被拒绝
- **那么** 系统降级到 osascript

#### 场景:Linux 降级
- **当** jeepney 不可用
- **那么** 系统尝试 notify-send，最后使用 tkinter messagebox
