## 新增需求

### 需求:从收藏夹同步检测标签必须显示实时进度

设置页「推荐标签」的「检测标签」候选池必须在用户点击「从收藏夹同步」后显示真实同步进度。进度必须覆盖收藏夹分页扫描阶段与无标签漫画详情补全阶段，禁止仅显示固定的「正在同步...」文案作为唯一反馈。

同步完成后，候选池仍必须展示 `favourite_tag_index` 中统计出的检测标签；这些检测标签仍仅供用户挑选加入 `my_tags`，禁止因同步完成而自动成为搜索结果高亮生效源。

#### 场景:收藏夹页扫描阶段显示页码和已扫描数量

- **当** 用户在支持推荐标签的来源下点击「从收藏夹同步」并且后端正在扫描收藏夹页面
- **那么** UI 必须显示当前页与总页数进度（例如 `3/12`）
- **且** UI 必须展示或可推导已扫描漫画数量
- **且** 按钮必须保持 disabled，防止重复触发同步

#### 场景:无标签漫画补全阶段显示补全进度

- **当** 收藏夹页扫描完成且存在需要详情补全标签的漫画
- **那么** UI 必须进入补全标签阶段文案
- **且** 必须显示当前已补全数量与待补全总数（例如 `17/43`）

#### 场景:同步完成后展示最终统计并刷新候选池

- **当** `sync_favourite_tags` 成功完成
- **那么** UI 必须停止同步中状态
- **且** 必须用返回的 `tags` 刷新「检测标签」候选池
- **且** 必须展示已同步漫画总数

#### 场景:同步失败时清理同步中状态

- **当** `sync_favourite_tags` 抛出错误或后端推送错误进度
- **那么** UI 必须停止同步中状态
- **且** 必须清理或替换旧进度，禁止按钮一直停留在同步中
- **且** 已有检测标签数据不得因一次失败而被前端清空，除非后端返回了新的空结果

#### 场景:来源切换不串显示其他来源进度

- **当** 前端订阅到非当前来源的 favourite tags 同步进度事件
- **那么** 设置页当前来源的进度显示必须忽略该事件
- **且** 当前来源候选池不得被其他来源的进度事件刷新
