## 修改需求

### 需求:加载指示器必须使用 spinner 并尊重 reduced-motion

列表页加载遮罩内的居中加载指示器**必须**是一个旋转的 spinner（不确定性动画，非确定性进度环），并**必须**附带一行辅助文案（语义锚点，避免强遮罩下转动被误判为卡顿）。spinner **必须**尊重用户的 reduced-motion 偏好：当 `prefers-reduced-motion` 启用时，spinner **必须**停止转动（呈现静止环），辅助文案仍**必须**显示。纯静态文字指示器**禁止**作为加载遮罩的唯一指示器。本需求适用于加载遮罩（保留旧结果时）与首次加载（无旧结果时的内联居中加载态）两种场景——后者同样**必须**使用 spinner，而**禁止**仅显示静态文字。

#### 场景:加载遮罩内显示旋转 spinner 与辅助文案

- **当** 任一列表页（搜索/收藏夹/历史）的加载遮罩处于显示状态
- **那么** 遮罩内居中显示一个旋转的 spinner（不确定性动画）
- **且** spinner 下方显示一行辅助文案
- **且** 当 `prefers-reduced-motion` 启用时 spinner 停止转动，文案仍显示

#### 场景:收藏夹页首次加载显示居中 spinner（无旧结果）

- **当** 用户在收藏夹页首次加载（无旧结果列表可见）且处于加载中
- **那么** 显示居中的 spinner 与辅助文案（无网格可遮罩、无旧结果保留）
- **且** 不展示翻页遮罩层（因无旧结果容器）
- **且** **禁止**仅显示静态「加载中...」文字而不带 spinner

#### 场景:历史页首次加载无旧网格时显示居中 spinner

- **当** 用户在历史页首次加载（旧结果列表为空）且处于加载中
- **那么** 显示居中的 spinner 与辅助文案（无网格可遮罩、无旧结果保留）
- **且** 不展示翻页遮罩层（因无旧结果容器）

#### 场景:首次加载的 spinner 与翻页遮罩 spinner 视觉一致

- **当** 任一列表页的首次加载内联加载态处于显示状态
- **那么** 其 spinner 的环样式、尺寸、配色与动画**必须**与翻页遮罩（`LoadingOverlay`）内的 spinner 完全一致
- **且** 两者的辅助文案默认均为「加载中...」
