## 新增需求

### 需求:来源侧边栏顶部间距必须与设置页、工具箱页一致

收藏夹来源侧边栏在未滚动状态下的顶部位置必须与设置页、工具箱页的同类左侧导航保持一致，禁止出现收藏夹侧边栏紧贴内容区顶部而其他页面侧边栏留有间距的不一致。侧边栏容器必须通过显式顶部内边距保证来源导航顶部距内容区顶部至少 24px，该间距不得依赖 flex stretch 与 sticky 的副作用交互——当侧边栏容器因 `self-start` 收缩到内容高度而使 sticky 不再产生初始偏移时，必须由显式内边距补足。

#### 场景:未滚动时收藏夹侧边栏顶部与设置页对齐

- **当** 收藏夹页面已选择来源且未发生滚动
- **那么** 左侧来源导航顶部距内容区顶部必须为 24px
- **且** 必须与设置页、工具箱页侧边栏导航的初始顶部位置一致

#### 场景:来源导航仍低于主内容标题

- **当** 收藏夹页面渲染标题与来源侧边栏
- **那么** 来源导航顶部必须位于主内容标题基线之下
- **且** 与设置页侧边栏导航相对其标题的垂直关系保持一致

#### 场景:滚动吸附行为不受顶部内边距影响

- **当** 用户向下滚动收藏夹内容
- **那么** 来源导航必须仍通过 sticky 布局吸附在滚动容器顶部下方 `top-6` 处
- **且** 顶部内边距不得改变既有的滚动吸附位置与可见性
