## 新增需求

### 需求:收藏夹漫画专辑下载后正确显示已下载状态
当用户从收藏夹批量选择多本漫画作为专辑下载并完成后，收藏夹页面必须正确显示这些漫画的"已下载"状态。

#### 场景:专辑下载完成刷新后显示已下载
- **当** 用户将收藏夹中的多本漫画作为专辑下载完成
- **且** 用户刷新收藏夹页面或切换标签后返回
- **那么** 这些漫画在收藏夹中必须显示"已下载"标记（绿色勾选）

#### 场景:专辑下载后重启应用仍显示已下载
- **当** 用户将收藏夹中的多本漫画作为专辑下载完成
- **且** 用户重启应用后再次打开收藏夹
- **那么** 这些漫画在收藏夹中必须仍然显示"已下载"标记

### 需求:下载状态查询接口支持按主键回退匹配
`check_downloaded_batch` 方法必须支持双路径查询：先按 `(source_site, album_id, comic_source)` 查询，无匹配时按 `(source_site, comic_id, comic_source)` 主键回退查询。

#### 场景:单本下载按 album_id 正常命中
- **当** 收藏夹中的漫画是单独下载的（非专辑方式）
- **那么** `check_downloaded_batch` 通过第一轮 `(source_site, album_id, comic_source)` 查询命中
- **且** 不需要触发回退查询

#### 场景:专辑下载章节按主键回退命中
- **当** 收藏夹中的漫画曾被作为专辑章节下载
- **且** 第一轮 `(source_site, album_id, comic_source)` 查询无结果
- **那么** 触发第二轮 `(source_site, comic_id, comic_source)` 主键回退查询
- **且** 返回正确的已下载状态

### 需求:路径存在性检查在回退路径中准确判定专辑章节
回退查询命中后，必须通过 `CBZBuilder.get_output_path_for_format` 计算预期路径，并通过 `os.path.exists` 验证文件实际存在，确保专辑章节文件的路径判定准确。

#### 场景:专辑章节文件存在于专辑目录结构中
- **当** 漫画是专辑章节（`is_album_chapter=True`）
- **且** 其文件位于 `<album_folder>/<chapter_name>/` 目录下
- **那么** `get_output_path_for_format` 必须正确计算专辑目录路径
- **且** `os.path.exists` 返回 `True`，状态标记为 `"downloaded"`

### 需求:现有功能不受影响
此变更必须保持向后兼容，不能破坏现有的单本下载、多章节专辑下载的已下载状态显示。

#### 场景:单本下载漫画收藏夹状态不变
- **当** 收藏夹中的漫画是单独下载完成的
- **那么** 变更前后该漫画的已下载状态必须一致
- **且** 仍然显示"已下载"标记

#### 场景:章节下载漫画收藏夹状态不变
- **当** 收藏夹中的漫画作为多章节专辑的组成部分下载完成
- **那么** 变更前后该漫画的已下载状态必须一致
- **且** 仍然显示"已下载"标记
