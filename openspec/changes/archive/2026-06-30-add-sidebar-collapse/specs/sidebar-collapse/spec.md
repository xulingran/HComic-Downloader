## 新增需求

### 需求: 侧边栏必须支持收起与展开两态切换

侧边栏必须提供两个互斥的视觉状态：**收起态**（固定窄宽度，仅显示图标）与**展开态**（宽栏，图标右侧显示菜单标题）。两态之间必须能由用户主动切换，状态在本次应用会话内稳定保留。切换入口必须包含侧边栏底部的 toggle 按钮；可选地提供全局快捷键作为补充入口。展开状态必须**禁止**持久化到磁盘——应用重启时必须回到默认的收起态。

#### 场景: 应用首次启动默认收起

- **当** 应用首次启动并渲染侧边栏
- **那么** 侧边栏处于收起态，宽度为窄栏（约 64px），每个菜单项仅渲染图标
- **且** 菜单项的中文标题仅通过原生 `title` 属性作为 hover tooltip 暴露

#### 场景: 用户点击底部 toggle 按钮展开侧边栏

- **当** 用户点击侧边栏底部的 toggle 按钮
- **那么** 侧边栏过渡到展开态，宽度过渡到宽栏（约 208px）
- **且** 每个菜单项在图标右侧渲染其中文标题（搜索 / 下载管理 / 收藏夹 / 历史记录 / 工具箱 / 维护 / 设置 / 关于）
- **且** 菜单项不再设置 `title` 属性（避免原生 tooltip 与可见标题重复）

#### 场景: 展开态下再次点击 toggle 收起

- **当** 侧边栏处于展开态时用户点击 toggle 按钮
- **那么** 侧边栏过渡回收起态，标题从 DOM 移除，宽度回到窄栏
- **且** 菜单项重新设置 `title` 属性以恢复 hover tooltip 提示

#### 场景: 展开状态在会话内跨页面切换保持稳定

- **当** 用户展开侧边栏后切换到其他菜单页面
- **那么** 侧边栏维持展开态（标题持续可见，宽度不变）
- **且** 切换页面不触发收展状态变化

#### 场景: 应用重启回到默认收起态

- **当** 用户展开侧边栏后关闭并重新启动应用
- **那么** 侧边栏回到收起态（展开状态未被持久化）

### 需求: toggle 按钮必须位于侧边栏底部且反映当前状态

切换按钮必须固定在侧边栏的最底部位置（不随菜单项列表滚动或重排），并必须通过其图标/视觉明确反映「即将展开」还是「即将收起」的语义，让用户无需尝试即可预知点击效果。

#### 场景: toggle 按钮位于菜单列表下方

- **当** 侧边栏渲染时（无论收起或展开态）
- **那么** toggle 按钮必须出现在所有菜单项之下，与菜单列表之间有视觉间隔

#### 场景: 收起态 toggle 按钮指示可展开

- **当** 侧边栏处于收起态
- **那么** toggle 按钮显示「展开」语义的图标（如 `»`）
- **且** 该按钮的 `title` 提示为「展开侧边栏」

#### 场景: 展开态 toggle 按钮指示可收起

- **当** 侧边栏处于展开态
- **那么** toggle 按钮显示「收起」语义的图标（如 `«`）
- **且** 该按钮的 `title` 提示为「收起侧边栏」

### 需求: 收展过渡必须顺滑且遵循动画契约

宽度变化必须通过 CSS transition（而非 JS 驱动布局动画）实现，过渡曲线必须使用项目既定的平滑曲线令牌值（`cubic-bezier(0.4, 0, 0.2, 1)`），时长约 300ms。标题的显隐必须通过渐变过渡而非瞬时切换。系统必须在用户启用「减少动画」偏好时，把宽度过渡与标题渐变都降级为近乎瞬时（遵循 `ui-animation` 规范的全局 CSS 兜底与组件级 `useReducedMotionPreference()` 双层策略）。收展动画期间必须**禁止**出现标签文本换行抖动或溢出裁切瞬态。

#### 场景: 展开时宽度平滑过渡

- **当** 用户触发展开
- **那么** 侧边栏宽度通过 CSS `transition-[width]` 从窄栏平滑过渡到宽栏
- **且** 过渡曲线为 `cubic-bezier(0.4, 0, 0.2, 1)`，时长约 300ms
- **且** 过渡期间无主线程长任务导致的肉眼掉帧

#### 场景: 标题渐入渐出

- **当** 展开时（标题出现）或收起时（标题消失）
- **那么** 标题通过 opacity 渐变过渡，而非瞬时显隐

#### 场景: 启用减少动画时收展近乎瞬时

- **当** 用户的系统「减少动画」偏好开启
- **那么** 宽度过渡与标题渐变都被压缩为近乎瞬时完成
- **且** 收展功能本身仍然可用（仅动画效果降级，不丧失切换能力）

#### 场景: 过渡期间标签不换行不溢出

- **当** 宽度过渡进行中（处于窄栏与宽栏之间的中间宽度）
- **那么** 标题文本必须保持单行（`white-space: nowrap`），不发生换行
- **且** 容器必须通过 `overflow: hidden` 防止标题在达到完整宽度前溢出可见区
