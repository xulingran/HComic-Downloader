## 上下文

当前项目是一个多源漫画下载器，支持 hcomic、moeimg、jmcomic 三个来源。收藏夹功能目前仅在 hcomic 中实现，jmcomic 的收藏夹是空实现。

jmcomic 已经集成到项目中，包括：
- 域名解析 (`sources/jmcomic/domain.py`)
- 会话管理 (`sources/jmcomic/session.py`)
- 搜索/详情解析 (`sources/jmcomic/parser.py`)
- 图片反混淆 (`sources/jmcomic/descrambler.py`)

## 目标 / 非目标

**目标：**
- 实现 jmcomic 收藏夹查看功能（分页浏览）
- 实现 jmcomic 收藏状态检查、添加和移除功能
- 更新前端 UI 支持多源收藏夹切换

**非目标：**
- 修改 moeimg 的收藏夹实现（moeimg 可能不支持收藏夹）
- 实现跨源收藏夹同步
- 修改 jmcomic 的搜索或下载功能

## 决策

### 1. 页面解析策略

**决策**：复用 JmParser 现有的 HTML 解析模式，使用 lxml + xpath 解析收藏夹页面。

**理由**：
- jmcomic 搜索页面已经使用此模式，代码风格一致
- 收藏夹页面结构与搜索页面类似（漫画列表 + 分页）
- 不需要引入新的依赖

**替代方案**：
- 使用 API 接口：jmcomic 可能有 API，但需要逆向工程，复杂度高
- 使用 Playwright 渲染：过度工程化，HTML 解析足够

### 2. 分页处理

**决策**：复用 `_parse_pagination()` 方法解析收藏夹分页。

**理由**：
- 收藏夹页面的分页结构与搜索页面相同
- 减少代码重复

### 3. 前端 UI 设计

**决策**：在 FavouritesPage 添加源选择器，复用 SearchPage 的下拉选择模式。

**理由**：
- 用户已经熟悉搜索页面的源选择交互
- 代码复用，减少维护成本

**替代方案**：
- Tab 切换：占用更多空间
- 独立页面：增加导航复杂度

### 4. IPC 接口设计

**决策**：为收藏夹相关 IPC 接口添加可选 `source` 参数，默认值为 "hcomic"。

**理由**：
- 向后兼容，不影响现有调用
- 与搜索接口的 source 参数设计一致

## 风险 / 权衡

| 风险 | 缓解措施 |
|------|----------|
| jmcomic 收藏夹页面结构可能变化 | 使用与搜索页面相同的解析模式，便于维护 |
| 登录状态检测可能不准确 | 复用 `verify_login_status()` 的重定向检测逻辑 |
| 图片 URL 需要反混淆 | 收藏夹只显示封面，封面不需要反混淆 |
| 分页结构可能与搜索不同 | 实现时添加 fallback 逻辑 |
