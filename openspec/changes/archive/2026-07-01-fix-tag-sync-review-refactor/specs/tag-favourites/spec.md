# tag-favourites 规范增量

> 对现有 `tag-favourites` 规范的内部可维护性收紧。需求层面行为不变，仅约束实现可读性与可扩展性。

## 新增需求

### 需求:详情抽屉标签按钮状态必须由单一查询表驱动

ComicInfoDrawer 详情抽屉中 tag chip 小按钮的四态（已屏蔽→取消屏蔽、已推荐→取消推荐、可推荐→加入推荐、不支持推荐→加入屏蔽）的 `action`/`icon`/`color`/`title` 四个渲染字段，必须由单一配置查询表 + 单次状态推导得出，禁止用多个平行三元表达式分别计算每个字段。

#### 场景:标签按钮四字段来自同一查询表

- **当** 详情抽屉渲染某个 tag chip 的小按钮
- **那么** 按钮的 action / icon / color / title 四个字段必须从同一张状态查询表（如 `TAG_BUTTON_STATE`）中按当前 state 一次性取出
- **且** 当前 state 必须由单一推导（基于 blocked / favourited / canRecommend 三输入）得出，禁止每个字段各自重复嵌套三元判定

#### 场景:新增按钮状态只需改查询表与推导

- **当** 维护者未来新增一种按钮状态（如「已锁定」）
- **那么** 修改点必须限于查询表新增一行 + state 推导新增一个分支
- **且** 不得需要同步修改 4 个独立的三元表达式
