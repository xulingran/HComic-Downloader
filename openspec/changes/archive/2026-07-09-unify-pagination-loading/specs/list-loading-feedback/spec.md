## 修改需求

### 需求:翻页时必须保留旧结果并叠加遮罩

当用户在同一结果集内翻页（点击分页器上一页/下一页、页码跳转）时，系统**必须**保留当前页的旧结果继续可见，并在其上叠加半透明遮罩与加载指示器，直到新页结果就绪。遮罩**必须**覆盖整个视口（`fixed inset-0`，而非仅覆盖网格容器），且遮罩内的加载指示器（spinner）**必须**位于视口正中，而**禁止**位于网格容器中心——当内容比视口高或矮时，spinner 仍**必须**在视口正中。遮罩内的居中加载指示器**必须**是一个旋转的 spinner（不确定性动画），并**必须**附带一行辅助文案；静态纯文字指示器**禁止**使用。翻页时清空已浏览内容**禁止**发生。

本需求适用于搜索页、收藏夹页与历史页三个列表页，三者的翻页遮罩与加载指示器**必须**采用一致的视觉模式（保留旧结果 + 全视口遮罩 + spinner + 辅助文案）。历史页翻页时卸载旧网格仅展示文字**禁止**发生。

#### 场景:搜索页翻页显示遮罩与 spinner

- **当** 用户在搜索页已加载某结果集的某一页后，通过分页器或页码跳转请求另一页（同一查询上下文、无缓存）
- **那么** 当前页的旧结果继续显示
- **且** 结果区上方叠加半透明遮罩，遮罩内居中显示旋转的 spinner（不确定性动画）与辅助文案
- **且** 新页结果就绪后替换旧结果、移除遮罩

#### 场景:收藏夹页翻页显示遮罩与 spinner

- **当** 用户在收藏夹页已加载某来源某一页后，通过分页器或页码跳转请求另一页（同一来源、无缓存）
- **那么** 当前页的旧结果继续显示
- **且** 结果区上方叠加半透明遮罩，遮罩内居中显示旋转的 spinner（不确定性动画）与辅助文案
- **且** 新页结果就绪后替换旧结果、移除遮罩

#### 场景:历史页翻页保留旧网格并显示遮罩与 spinner

- **当** 用户在历史页已加载某一页后，通过分页器或页码跳转请求另一页（同一历史上下文、无缓存），且当前已有旧结果列表可见
- **那么** 当前页的旧结果继续显示（不卸载旧网格）
- **且** 结果区上方叠加半透明遮罩，遮罩内居中显示旋转的 spinner（不确定性动画）与辅助文案
- **且** 新页结果就绪后替换旧结果、移除遮罩

#### 场景:历史页首次加载无旧网格时显示居中 spinner

- **当** 用户在历史页首次加载（旧结果列表为空）且处于加载中
- **那么** 显示居中的 spinner 与辅助文案（无网格可遮罩、无旧结果保留）
- **且** 不展示翻页遮罩层（因无旧结果容器）

### 需求:加载遮罩必须按"旧结果是否将被替换"分两档强度

列表页（搜索页、收藏夹页、历史页）在加载期间保留旧结果并叠加遮罩时，遮罩的视觉强度**必须**根据"旧结果是否即将被整页替换"分为两档：

- **轻档**：翻页场景（旧结果仍属同一查询上下文，对用户有参考价值）。
- **重档**：整页替换场景（旧结果即将被新来源/新查询的结果取代），包括切换来源、随机、分类入口、NH 各入口、抽屉/侧栏标签追加搜索等。

两档的视觉差异**必须**足以让用户直观区分。**轻档的模糊强度必须使旧结果基本不可辨认**（旧结果仅剩色块、文字不可读），而**禁止**停留在"旧结果基本可读"的强度；重档强度**必须**明显高于轻档以保持两档视觉差异。轻档与重档**禁止**使用相同的模糊强度或背景不透明度。具体而言，轻档使用约 8px 的 backdrop-blur 与约 80% 的背景不透明度，重档使用约 16px 的 backdrop-blur 与约 92% 的背景不透明度。

#### 场景:翻页时显示轻档遮罩

- **当** 用户在列表页已加载某结果集的某一页后，通过分页器或页码跳转请求同一查询上下文的另一页（无缓存），触发 `keepExisting: true` 的加载路径
- **那么** 叠加的遮罩使用轻档视觉（约 8px backdrop-blur + 约 80% 背景不透明度），旧结果基本不可辨认
- **且** 遮罩内居中显示 spinner 与辅助文案

#### 场景:切换来源认证校验期间显示重档遮罩

- **当** 用户在搜索页切换到一个需要认证的来源（jm/bika/copymanga），当前已有旧结果，系统进入 `verifySourceAuth` 校验窗口（旧结果尚未清空、`isLoading` 为 true）
- **那么** 叠加的遮罩使用重档视觉（约 16px backdrop-blur + 约 92% 背景不透明度），旧结果几乎完全不可辨认
- **且** 遮罩内居中显示 spinner 与辅助文案

#### 场景:同来源新查询显示重档遮罩

- **当** 用户在搜索页同一来源下触发整页替换（随机、分类入口、NH 入口、抽屉/侧栏标签追加搜索），当前已有旧结果且未走骨架路径
- **那么** 叠加的遮罩使用重档视觉
- **且** 遮罩内居中显示 spinner 与辅助文案

### 需求:加载指示器必须使用 spinner 并尊重 reduced-motion

列表页加载遮罩内的居中加载指示器**必须**是一个旋转的 spinner（不确定性动画，非确定性进度环），并**必须**附带一行辅助文案（语义锚点，避免强遮罩下转动被误判为卡顿）。spinner **必须**尊重用户的 reduced-motion 偏好：当 `prefers-reduced-motion` 启用时，spinner **必须**停止转动（呈现静止环），辅助文案仍**必须**显示。纯静态文字指示器**禁止**作为加载遮罩的唯一指示器。

#### 场景:加载遮罩内显示旋转 spinner 与辅助文案

- **当** 任一列表页（搜索/收藏夹/历史）的加载遮罩处于显示状态
- **那么** 遮罩内居中显示一个旋转的 spinner（不确定性动画）
- **且** spinner 下方或旁边显示一行辅助文案（如「加载中...」）
- **且** spinner 不可为显示百分比的确定性进度环

#### 场景:reduced-motion 启用时 spinner 停转

- **当** 用户系统启用了 `prefers-reduced-motion`，且列表页加载遮罩处于显示状态
- **那么** spinner 停止转动，呈现静止环
- **且** 辅助文案照常显示
