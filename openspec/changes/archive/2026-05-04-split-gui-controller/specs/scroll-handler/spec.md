## ADDED Requirements

### 需求:跨平台滚动事件处理
系统必须将 MouseWheel、TouchpadScroll、Button-4/5 三类滚动事件统一处理，仅响应发生在搜索结果区域（Canvas 及其内部 scrollable_frame）内的事件。

#### 场景:Windows 鼠标滚轮滚动
- **当** 用户在搜索结果区域内滚动鼠标滚轮
- **那么** Canvas 按整数单位（120 为一档）垂直滚动，余量累积到下一次

#### 场景:macOS 触控板滚动
- **当** 用户在搜索结果区域内使用触控板滑动
- **那么** Canvas 按像素增量平滑滚动，比例换算基于可滚动区间

#### 场景:Linux Button-4/5 滚动
- **当** 用户在搜索结果区域内触发 Button-4 或 Button-5 事件
- **那么** Canvas 分别向上或向下滚动一个单位

#### 场景:滚动区域外的事件忽略
- **当** 滚动事件发生在输入框、文本选择区域等非结果区域内
- **那么** 事件不被 ScrollHandler 拦截，交给原生控件处理

### 需求:滚动状态通知
系统必须在滚动激活和空闲时通知订阅者，空闲延迟为 120ms。

#### 场景:滚动期间标记激活
- **当** ScrollHandler 接收到滚动事件
- **那么** 标记滚动激活状态为 True，重置空闲计时器

#### 场景:滚动空闲后通知
- **当** 滚动事件停止超过 120ms
- **那么** 标记滚动激活状态为 False，通知所有空闲回调
