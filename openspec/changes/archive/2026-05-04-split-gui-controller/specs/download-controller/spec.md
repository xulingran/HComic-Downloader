## ADDED Requirements

### 需求:单个漫画下载
系统必须支持下载单个漫画，流程为：详情预取 → 文件冲突检测 → 用户确认 → 后台下载 → 打包。

#### 场景:下载流程
- **当** 用户点击漫画卡片上的下载按钮且无其他下载进行中
- **那么** 在后台线程预取详情，检测文件冲突，弹出确认对话框，确认后开始下载

#### 场景:下载进行中禁止新下载
- **当** 单个下载或批量下载正在进行中
- **那么** 禁止启动新的下载任务并提示用户

#### 场景:下载完成
- **当** 下载和打包成功完成
- **那么** 显示成功消息，包含保存路径

#### 场景:下载失败
- **当** 下载或打包过程中发生异常
- **那么** 显示错误消息，保留临时文件路径供用户手动恢复

### 需求:批量下载
系统必须支持批量选择多个漫画后一次性下载，使用下载管理器队列。

#### 场景:批量下载流程
- **当** 用户在批量选择模式下选中漫画并点击批量下载
- **那么** 在后台线程预取所有选中漫画的详情，弹出确认对话框，确认后添加到下载管理器队列

#### 场景:批量下载完成
- **当** 下载管理器队列中所有任务完成
- **那么** 显示汇总消息（成功/失败数量和失败详情），清空选择状态

### 需求:文件冲突检测与处理
系统必须在下载前检测输出路径是否已存在同名文件，提供覆盖、跳过、全部覆盖、全部跳过的选项。

#### 场景:检测到冲突
- **当** 输出路径已存在文件
- **那么** 弹出冲突对话框，列出冲突文件名，让用户选择处理方式

#### 场景:用户取消冲突处理
- **当** 用户关闭冲突对话框
- **那么** 所有冲突文件视为跳过

### 需求:下载队列 UI 集成
系统必须在下载开始时自动展开下载管理器面板，实时更新任务状态和进度。

#### 场景:队列任务更新
- **当** 下载管理器报告任务状态变化
- **那么** 更新下载管理器 UI 中对应任务的状态图标和进度

#### 场景:底部进度条同步
- **当** 当前正在处理的任务进度更新
- **那么** 同步更新底部状态栏的进度条和状态文本
