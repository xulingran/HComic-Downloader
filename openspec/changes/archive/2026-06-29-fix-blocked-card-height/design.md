## 上下文

搜索结果中命中 tag 黑名单的漫画在 `SearchPage.tsx` 的 `filteredComics` 里被标记为 `isBlocked`，渲染时用 `BlockedPlaceholder` 替换 `ComicCard`。该占位符有封面（cover）与详细（detailed）两个变体。

封面变体（`SearchPage.tsx` 原 line 944-962）的内容区结构与正常 `CoverCard`（`ComicCard.tsx` line 191-240）不一致：

| 维度 | 正常 `CoverCard` | 屏蔽占位符（封面） |
|------|------------------|---------------------|
| 内容区 padding | `p-2` | `p-3`（更大） |
| 标题最小高度 | `min-h-[2.5rem]` | 无 |
| 作者栏 | `h-4 mt-0.5`（含 `&nbsp;` 占位） | 无 |

在 CSS Grid 网格布局中，同一行所有卡片由最高的卡片撑出行高。屏蔽卡片少一行作者栏高度，导致它在该行的底部留空、与同行正常卡片底部不对齐；当某行混有屏蔽与正常卡片时，整行视觉错乱。

约束：屏蔽卡片是"简化占位符"——设计意图是 line-through 标题、无作者文字、封面区用 🚫 + 「已屏蔽」标签传达屏蔽语义。修复**不能**让它恢复成正常卡片的信息密度（不展示作者/页数等），只需补齐高度占位。

## 目标 / 非目标

**目标：**
- 封面模式下的屏蔽占位符与正常 `CoverCard` 在网格行内的总高度对齐（padding、标题最小高度、作者占位行三者都对齐）。
- 保留屏蔽占位符的简化视觉契约（line-through 标题、屏蔽图标、不展示作者等信息）。
- 通过渲染测试断言高度结构对齐，防止回归。

**非目标：**
- 不修改详细列表（detailed）模式下的屏蔽占位符——它本身高度一致。
- 不改变屏蔽判定逻辑（`filteredComics` / `isBlocked`）。
- 不调整正常卡片 `CoverCard` 的结构（它是高度基准，不能动）。
- 不调整网格容器、`AnimatedCardWrapper`、过滤逻辑。

## 决策

### 决策 1：用结构对齐而非 CSS 锚定行高

**选择：** 在 `BlockedPlaceholder` 封面变体内补齐 padding / min-height / 空作者占位行，使其 DOM 高度结构与 `CoverCard` 完全对齐。

**替代方案 A：给 grid 子项设统一 min-height。** 否决——会牵连 `AnimatedCardWrapper` 和 grid 容器，影响面大；且 Tailwind 的 `aspect-[6/7]` 封面区高度随列宽变化，硬编码 min-height 会与响应式列宽冲突。

**替代方案 B：屏蔽卡片也展示作者文字。** 否决——违反"屏蔽=简化占位符"的设计意图，恢复了不应展示的信息密度。

**理由：** 高度差异的根因是 DOM 结构差异。让占位符的结构（padding + 标题 min-height + 一行占位）与基准卡片一致，是改动最小、最稳定的对齐方式。封面区 `aspect-[6/7]` 两者本来就一致，无需处理。

### 决策 2：作者占位行用空 div 而非文字

**选择：** 标题下方补一个 `h-4 mt-0.5` 的空占位（不渲染文本，或仅渲染不可见的占位符）。

**替代方案：把封面区已有的「已屏蔽」小标签移到这里。** 否决——封面区是更合适的位置（视觉中心、与 🚫 图标同区），移动会分散屏蔽语义。

**理由：** 作者占位行的唯一作用是占高度。空 div 最简洁，不引入额外视觉噪声。

## 风险 / 权衡

- **[风险] Tailwind 类名拼写错误导致占位行实际不渲染** → 缓解：渲染测试断言 `BlockedPlaceholder` 渲染出的元素结构（padding 类、min-h 类、占位 div 数量）与正常 `CoverCard` 对应位置一致。
- **[风险] 正常卡片结构未来变更，屏蔽占位符未同步** → 缓解：本变更同步新增的规范 `blocked-card-placeholder` 把"高度对齐"作为需求固化，后续若改 `CoverCard` 结构需对照同步占位符。
- **[权衡] 仅对齐 DOM 结构高度，未用 CSS 显式保证像素级相等** → 可接受。两套结构在相同 padding/min-height/aspect-ratio 下计算出的高度必然相等，无需引入脆弱的像素常量。
