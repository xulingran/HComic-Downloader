## 修改需求

### 需求:核心信息和主要操作必须优先保持在窗口高度内

抽屉必须将标题、作者、来源、页数或章节等核心信息以及收藏和阅读操作置于高优先级区域。在默认 800px 高窗口中，常规长度内容必须无需滚动即可看到上述信息、关联信息摘要、完整标签列表和主要操作；当标签数量导致内容超出可用高度时，标签必须保持完整并由独立内容区提供纵向滚动。在最小 600px 高窗口中，主要操作必须始终可见，其余内容只能在独立内容区内产生最小必要滚动。

#### 场景:默认高度展示常规详情

- **当** 应用窗口高度为 800px 且漫画标题、关联信息和标签数量处于常规范围
- **那么** 用户必须无需滚动即可看到封面摘要、核心元数据、完整标签列表以及全部主要操作

#### 场景:大量标签使用内容区滚动

- **当** 完整标签列表使详情内容超过可用高度
- **那么** drawer 必须在独立内容区内允许纵向滚动以访问全部标签
- **且** 禁止截断或折叠标签列表
- **且** 底部主要操作必须保持在视口内

#### 场景:最小高度保持操作可达

- **当** 应用窗口高度为 600px
- **那么** 抽屉必须启用更紧凑的封面、间距或排布
- **且** 收藏、开始阅读、继续阅读和从头开始等当前适用的主要操作必须保持在视口内
- **且** 只有中间详情内容区允许纵向滚动，顶部关闭栏和底部操作区禁止随内容滚出视口

### 需求:标签列表必须完整展示

drawer 和 reader 表面的标签列表必须始终展示当前漫画的全部标签，禁止按数量截断、折叠或渲染标签展开/收起控件。标签原有的搜索、推荐、屏蔽、状态高亮和来源能力门控必须保持可用。

#### 场景:drawer 展示全部标签

- **当** 用户打开包含任意数量标签的漫画详情抽屉
- **那么** drawer 必须展示当前漫画的全部标签
- **且** 禁止渲染“+N 展开”“展开”或“收起”控件

#### 场景:切换漫画后展示新漫画的全部标签

- **当** drawer 从一部漫画切换到另一部漫画
- **那么** 标签区域必须展示新漫画的全部标签
- **且** 禁止保留与上一部漫画相关的标签展示状态

#### 场景:阅读器尾页保持完整标签

- **当** 同一详情内容以 `surface="reader"` 显示在在线阅读器尾页
- **那么** 必须继续展示完整标签列表且不得出现 drawer 专属的展开或收起控件

### 需求:详情补全状态必须在紧凑布局中保持可见

标签详情补全的加载、失败、重试和成功状态必须继续在标签区域显示，禁止因完整标签列表或固定操作区隐藏错误反馈。

#### 场景:标签补全失败

- **当** 详情补全请求抛出异常或返回空漫画
- **那么** 抽屉必须显示标签加载失败反馈和可操作的重试入口

#### 场景:标签补全成功

- **当** 重试或首次详情补全成功返回标签
- **那么** 标签区域必须显示返回的全部标签，并移除失败反馈
