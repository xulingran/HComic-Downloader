## 修改需求

### 需求:NH 搜索栏标签面板

系统必须将 NH 接入搜索栏旁的标签按钮和标签选择弹窗，在弹窗中提供热门与 A-Z 排序切换，并保证最终展示顺序与用户当前选择一致。异步读取标签时，系统必须仅允许当前来源与排序对应的最新请求更新标签列表和加载状态。

#### 场景:NH 来源显示标签按钮
- **当** 用户在搜索页选择 NH 来源
- **那么** 搜索栏旁必须显示标签按钮
- **且** 点击标签按钮必须打开标签选择弹窗

#### 场景:标签弹窗切换热门排序
- **当** 用户在 NH 标签选择弹窗中切换为「热门」排序
- **那么** 前端必须使用 `popular` 重新读取标签列表
- **且** 合并收藏标签后的最终列表必须按 count 降序、标签名升序展示
- **且** 已选择的标签必须保留

#### 场景:标签弹窗切换 A-Z 排序
- **当** 用户在 NH 标签选择弹窗中切换为「A-Z」排序
- **那么** 前端必须使用 `name` 重新读取标签列表
- **且** 合并收藏标签后的最终列表必须按标签名升序展示，禁止再次按 count 重排
- **且** 已选择的标签必须保留

#### 场景:快速切换排序只提交最新结果
- **当** 用户在前一个标签读取完成前再次切换排序
- **那么** 只有最后一次排序对应的响应可以更新标签列表和加载状态
- **且** 较早请求即使较晚返回也禁止覆盖最新结果

#### 场景:切换来源只提交当前来源结果
- **当** 标签读取期间用户切换漫画来源
- **那么** 旧来源响应禁止更新新来源的标签列表或加载状态
- **且** 新来源必须使用默认 `popular` 排序读取标签

#### 场景:刷新 NH 标签列表
- **当** 用户在 NH 标签弹窗中点击刷新
- **那么** 系统必须触发 NH 原始标签目录同步
- **且** 刷新完成后必须按当前排序方式重新加载标签列表

#### 场景:全量标签同步进度展示
- **当** 用户在 NH 标签弹窗中触发全量刷新
- **那么** 系统必须在同步期间展示当前页、总页数与已收集标签数
- **且** 若同步过程中出现页面失败，必须展示错误状态或失败提示
- **且** 同步完成后必须恢复为正常标签列表展示

### 需求:NH 精确标签搜索

系统必须将 NH 标签面板和入口页触发的标签搜索转换为 NH 支持的精确标签查询语法。无论用户选择标签前处于最近更新、热门排行、普通关键词还是已有标签结果上下文，标签面板选择都必须切换为明确的标签搜索语义，禁止继续按原 mode 忽略所选标签。

#### 场景:单标签搜索
- **当** 用户选择一个 NH 标签并触发搜索
- **那么** 前端必须以 tag 搜索语义提交该标签
- **且** 后端必须将该标签转换为 `tag:"<标签名>"` 查询
- **且** 必须调用 NH 搜索接口获取结果

#### 场景:从最近更新选择标签
- **当** 用户正在查看 NH 最近更新并从标签弹窗选择标签
- **那么** 系统必须切换到 tag 搜索语义并清除最近更新的空关键词上下文
- **且** 搜索结果必须由所选标签过滤，禁止回退为最近更新列表

#### 场景:从热门排行选择标签
- **当** 用户正在查看 NH 热门排行并从标签弹窗选择标签
- **那么** 系统必须切换到 tag 搜索语义并停止使用 popular 排序请求语义
- **且** 搜索结果必须由所选标签过滤，禁止忽略标签继续返回热门排行

#### 场景:多标签搜索
- **当** 用户选择多个 NH 标签并触发搜索
- **那么** 后端必须将每个标签转换为独立的 `tag:"<标签名>"` 查询片段
- **且** 必须用空格拼接这些查询片段

#### 场景:标签名包含引号
- **当** NH 标签名包含双引号
- **那么** 后端构造查询时必须转义双引号
- **且** 禁止生成破坏查询结构的未转义引号
