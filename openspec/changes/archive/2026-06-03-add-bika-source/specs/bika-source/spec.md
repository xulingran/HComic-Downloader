## 新增需求

### 需求:Bika 漫画搜索
系统必须支持从 Bika 平台搜索漫画。用户输入关键词后，系统调用 Bika API 的 `comics/advanced-search` 端点获取结果。

#### 场景:关键词搜索
- **当** 用户选择 Bika 来源并输入搜索关键词
- **那么** 系统返回包含标题、作者、封面、页数、章节数的漫画列表

#### 场景:分页搜索
- **当** 用户在 Bika 搜索结果中翻页
- **那么** 系统请求对应页码的结果并更新分页信息

### 需求:Bika 漫画详情
系统必须支持获取 Bika 漫画的详细信息，包括章节列表。

#### 场景:获取详情
- **当** 用户打开 Bika 漫画的详情抽屉
- **那么** 系统调用 `comics/{id}` 获取详情，显示标题、作者、标签、页数、章节数

#### 场景:获取章节列表
- **当** 漫画有多个章节（epsCount > 1）
- **那么** 系统调用 `comics/{id}/eps` 获取章节列表，每个章节包含 id、名称、序号

### 需求:Bika 收藏功能
系统必须支持 Bika 平台的收藏夹功能。

#### 场景:查看收藏
- **当** 用户切换到 Bika 来源并查看收藏夹
- **那么** 系统调用 `users/favourite` 获取收藏列表

#### 场景:添加收藏
- **当** 用户在详情抽屉点击收藏按钮
- **那么** 系统调用 `comics/{id}/favourite` 添加收藏

#### 场景:取消收藏
- **当** 用户在详情抽屉取消收藏
- **那么** 系统再次调用 `comics/{id}/favourite` 切换状态

### 需求:Bika 下载功能
系统必须支持下载 Bika 漫画。

#### 场景:单章节下载
- **当** 用户选择下载单章节 Bika 漫画
- **那么** 系统调用 `comics/{id}/order/{order}/pages` 获取图片列表并下载

#### 场景:多章节下载
- **当** 用户选择下载多章节 Bika 漫画
- **那么** 系统逐章获取图片列表并下载，合并为一个 CBZ 文件

### 需求:章节数量显示
系统必须在漫画详情抽屉中显示章节数量（适用于所有有多章节的来源）。

#### 场景:显示章节数
- **当** 漫画的 albumTotalChapters > 1
- **那么** 详情抽屉显示 "{n} 章" 信息
