## 上下文

现有中文筛选贯穿 `SearchPage`、搜索缓存/预加载、Electron IPC、Python `SearchMixin`、`MultiSourceParser` 和 NH parser，但各层来源守卫及命名均固定为 NH。moeimg 的 `/spa/latest-manga`、`/spa/search`、作者与标签端点返回列表项时带有 `language` 字段，当前 `_parse_search_manga_list` 却未写入 `ComicInfo.language`；站点另提供 `/spa/language/chinese?page=N`，返回服务端过滤后的 40 条分页结果。

实测 `/spa/language/chinese` 会忽略 `query`，而 `/spa/search` 会忽略额外 `language=chinese`，因此 moeimg 只能在默认浏览模式实现准确的服务端中文分页。关键词、作者和标签模式只能基于当前返回页做前端占位式过滤。所有请求必须继续复用 `MoeImgParser.session`，确保系统代理和认证状态不被绕过。

## 目标 / 非目标

**目标：**

- 在 moeimg 来源显示并支持现有“仅显示中文”二态开关。
- 默认浏览优先使用 moeimg 服务端中文分页，并在专用接口不可用时保持浏览可用。
- 通过列表语言元数据把已知非中文结果渲染成现有屏蔽占位符。
- 保证筛选状态贯穿翻页、缓存、预加载和 IPC，且与 NH 筛选上下文隔离。
- 保持固定枚举、来源白名单和系统代理约束。

**非目标：**

- 不增加日文、英文或任意语言选择器。
- 不为关键词、作者或标签搜索伪造服务端中文过滤及重算分页。
- 不通过逐项详情请求补全缺失语言。
- 不把语言筛选持久化到配置。
- 不改变其他四个来源的搜索语义。

## 决策

### 1. 默认浏览采用服务端优先、最近更新回退

当 moeimg 收到空关键词且 `language_filter='chinese'` 时，先使用同一 parser Session 请求 `/spa/language/chinese`。只有该请求发生网络/HTTP/JSON/响应结构错误时，才重试 `/spa/latest-manga`；不得把合法空结果视为故障，否则会错误展示未过滤内容。

回退结果仍携带筛选上下文，由前端对已知非中文项做占位式过滤。相比失败后直接报错，此设计保留最近更新的可用性；相比始终用最近更新再过滤，正常路径能获得准确的中文分页。

### 2. parser 权威透传列表语言，前端统一执行防御性过滤

`_parse_search_manga_list` 从每个列表项提取、去空白并透传 `language`。前端在来源为 moeimg 且筛选开启时，把规范化后存在且不等于 `chinese` 的条目标记为 blocked；`language` 缺失时保持可见，与标签缺失时不命中黑名单的现有语义一致。

即使专用中文接口成功，前端也执行同一判断，作为响应漂移的防御层。语言筛选不依赖全局 `filterEnabled`，但语言屏蔽与标签屏蔽合并为一个 `isBlocked`，继续复用 `BlockedPlaceholder`，避免引入第二套卡片体系。

### 3. 关键词、作者和标签模式只做当前页占位过滤

moeimg 没有可组合语言谓词的搜索 API。开启筛选后这些模式继续走原端点和原分页；已知非中文项显示为占位符，分页总数不修改。这样明确保留服务端分页真实性，也避免抓取大量页面或逐项详情请求。

### 4. 将 NH 专属状态泛化为受支持来源状态

前端把 `nhLanguageFilter`、对应 ref、handler 与 props 泛化为 `languageFilter`，有效来源集合为 `nh` 和 `moeimg`。来源切换时只向受支持来源传递筛选；每个搜索上下文仍由 source 与 languageFilter 共同区分。是否在两个来源间保留同一开关值属于运行期 UI 状态，不落盘。

### 5. IPC 与 Python 双层执行来源白名单

共享枚举继续只有 `chinese`。preload 校验枚举，Electron 主进程只允许 `nh` 或 `moeimg` 携带筛选，Python `SearchMixin` 再做相同来源和值校验。`MultiSourceParser.search` 对这两个来源显式传递 `language_filter`，其他 parser 签名不变。

### 6. 全量屏蔽提示泛化

`blockedCount === comics.length` 时的文案改为“所有结果均已被筛选”，因为屏蔽原因可能是标签、语言或二者共同作用。卡片本身继续显示统一“已屏蔽”，不暴露筛选原因。

## 风险 / 权衡

- **[专用中文端点返回 200 但结构悄然漂移]** → 在确认 `manga_list` 与分页结构可解析后才采用，否则回退最近更新并由前端过滤。
- **[回退页或搜索页存在缺失 language 的条目]** → 按既有缺失标签语义保持可见，避免把未知误判为非中文；详情抽屉仍可用于人工确认。
- **[当前页多数漫画被占位后有效内容很少]** → 保留服务端分页与占位位置，明确这是 moeimg API 无法组合筛选的权衡，不抓取额外页面。
- **[NH 专属变量泛化造成入口动作遗漏]** → 覆盖默认浏览、搜索、翻页、标签切换、缓存恢复和相邻页预加载测试。
- **[回退掩盖长期接口故障]** → 记录中文端点失败日志并保留原异常上下文；最近更新回退也失败时正常向上抛错。

## 迁移计划

无数据迁移。先扩展后端/IPC 契约与 parser，再泛化前端状态和呈现，最后运行定向测试及完整验证。回滚时可移除 moeimg 来源白名单和开关展示，NH 原有参数与缓存键保持兼容。

## 待解决问题

无。缺失语言保持可见、其他搜索模式采用占位式当前页过滤均已确定。
