## 修改需求

### 需求:屏蔽占位符必须保留简化视觉语义

屏蔽占位符作为“被过滤内容”的视觉占位，必须统一承载标签黑名单和 moeimg 语言筛选产生的屏蔽结果：标题使用删除线（`line-through`）、封面区显示屏蔽图标与“已屏蔽”提示、整体降低不透明度。屏蔽占位符禁止展示作者、页数、章节数等正常卡片才有的元信息，禁止渲染可下载按钮。

#### 场景:屏蔽占位符标题显示删除线
- **当** 因标签或语言筛选渲染屏蔽占位符（封面或详细任一模式）
- **那么** 标题必须应用 `line-through` 删除线样式
- **且** 禁止展示作者、页数、章节等元信息文本

#### 场景:屏蔽占位符点击标题打开详情抽屉
- **当** 用户点击屏蔽占位符的标题
- **那么** 必须阻止事件冒泡并打开该漫画的详情抽屉（`openDrawer(comic)`），让用户仍可查看被屏蔽项的完整信息

## 新增需求

### 需求:全量屏蔽提示必须涵盖所有筛选原因

当当前结果中的全部漫画均被标记为 blocked 时，系统必须显示不限定于标签原因的通用筛选提示，禁止声称结果一定由标签过滤。

#### 场景:语言筛选屏蔽全部结果
- **当** moeimg 中文筛选使当前页全部漫画呈现为屏蔽占位符
- **那么** 页面必须显示“所有结果均已被筛选”或等价通用文案
- **且** 禁止显示“所有结果均已被标签过滤”
