## ADDED Requirements

### 需求:在线阅读器进度导航必须识别详情尾页语义

当在线阅读器为当前阅读序列提供详情尾页时，进度导航必须把详情作为最后一个可达导航项，但必须以“详情”而不是不存在的图片页表达当前状态。滑块、连续滚动定位和显示模式切换必须能到达并保持详情；本地漫画库阅读器必须维持既有纯图片页范围与数字页码显示。

#### 场景:滑块最右端进入详情

- **当** 在线阅读器当前序列包含详情尾页，用户把底部进度滑块拖到最右端
- **那么** 阅读器必须定位到详情尾页
- **并且** 禁止尝试渲染或请求一张 `N+1` 图片

#### 场景:详情页显示语义标签

- **当** 当前导航项为详情尾页
- **那么** 阅读器页码区域必须显示“详情”，滑块必须显示 100% 终点状态
- **并且** 滑块可访问文本必须表达“漫画详情”，禁止仅把详情显示为普通数字图片页

#### 场景:连续滚动滑块定位详情

- **当** 在线阅读器处于连续滚动模式且用户通过进度滑块选择详情终点
- **那么** 详情容器必须即时进入阅读视口
- **并且** 页面可见性追踪稳定后必须保持详情为当前导航项

#### 场景:从详情向左拖回图片

- **当** 用户停留在详情尾页并把进度滑块向左拖到一个真实图片页位
- **那么** 阅读器必须定位到对应图片或包含该图片的合法双页 spread
- **并且** 后续图片预加载必须以该真实图片页作为目标

#### 场景:本地阅读器进度范围不变

- **当** 用户打开本地漫画库阅读器并操作进度滑块
- **那么** 有效范围必须仍只覆盖本地图片页与既有双页补白语义
- **并且** 页码区域禁止出现在线漫画详情终点
