## 修改需求

### 需求:本地封面按需提取并缓存

系统必须使用资产的第一张有效图片作为默认封面，并按资产 ID、修改时间和图片条目生成缓存键。对于多章节文件夹合集，系统必须按已索引的自然章节顺序查找封面，使用最早可读取章节中的第一张有效图片；当某个靠前章节缺失、为空或读取失败时，必须继续尝试后续章节，禁止仅因合集根目录没有直属图片而判定封面提取失败。封面提取必须按需或限流执行，禁止首次扫描时无上限并发解压所有资产；SFW 模式必须继续隐藏真实封面。

#### 场景:首次展示无缓存的 CBZ
- **当** 漫画卡进入可视区域且对应封面缓存不存在
- **那么** 后端只读取首张有效图片生成缩略图，前端在完成前显示骨架占位

#### 场景:多章节文件夹合集生成默认封面
- **当** 多章节文件夹合集的根目录没有直属图片，但自然顺序第一章包含有效图片
- **那么** 后端使用第一章的第一张有效图片生成封面并持久化该资产的封面缓存键

#### 场景:合集首章无法提供封面
- **当** 多章节文件夹合集的自然顺序第一章缺失、为空或读取失败，而后续章节包含有效图片
- **那么** 后端跳过不可用章节并使用最早可读取后续章节的第一张有效图片生成封面

#### 场景:合集没有任何可读取图片
- **当** 多章节文件夹合集的所有已索引章节均无法提供有效图片
- **那么** 后端报告封面提取失败且不写入错误的封面缓存键

#### 场景:漫画文件内容发生变化
- **当** 已缓存封面的资产修改时间发生变化
- **那么** 原封面缓存键失效，系统重新提取封面
