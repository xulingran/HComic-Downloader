## 新增需求

### 需求:阅读器显示模式切换必须使用符合语义的过渡

阅读器在连续滚动、单页和双页模式之间切换时必须播放显示模式过渡。连续滚动与任一分页模式之间必须使用先退出、后进入的 fade-through，任意时刻禁止同时显示两套完整阅读内容；单页与双页之间必须使用页面布局重排，使切换前的当前实际页保持为视觉锚点。模式切换禁止复用表示内容前进或后退的整屏横向翻页动画。

#### 场景:连续滚动切换到单页

- **当** 用户在连续滚动模式中选择单页模式
- **那么** 滚动视图必须先淡出并隐藏，单页视图再以同一实际页为目标淡入，禁止两套页面内容交叉重影

#### 场景:双页切换到连续滚动

- **当** 用户在双页模式中选择连续滚动模式
- **那么** 双页视图必须先淡出，滚动视图必须定位到切换前双页中作为锚点的实际页后再淡入

#### 场景:单页切换到双页

- **当** 用户在单页模式中选择双页模式
- **那么** 当前实际页必须平滑移动并缩放到合法双页布局中的对应位置，伴随页必须淡入，禁止把该模式变化表现为向前或向后翻页

#### 场景:双页切换到单页

- **当** 用户在双页模式中选择单页模式
- **那么** 双页中作为锚点的实际页必须平滑展开到居中的单页布局，另一页必须淡出，禁止先显示错误页再二次跳页

### 需求:模式切换必须在目标视图就绪后才显示

阅读器必须在目标视图隐藏期间完成目标页计算和位置准备。进入连续滚动模式时必须在淡入前将锚点页即时定位到阅读视口；进入单页或双页模式时必须在首次可见帧直接渲染合法目标页或页组。禁止先显示默认第一页、错误双页组合或未定位内容，再通过 effect 进行可见修正。

#### 场景:从单页返回长章节的连续滚动模式

- **当** 用户正在第 80 页并从单页切换到连续滚动模式
- **那么** 第 80 页容器必须在滚动视图淡入前完成即时定位，用户禁止看到第一页后再跳到第 80 页

#### 场景:偶数单页进入双页模式

- **当** 用户在无补白的单页模式第 6 页选择双页模式
- **那么** 首个可见双页帧必须直接显示包含第 6 页的合法页组，禁止先显示第 6、7 页再修正为第 5、6 页

#### 场景:补白双页退出到单页

- **当** 用户从包含前补白或后补白的双页布局切换到单页模式
- **那么** 阅读器必须把合法实际图片页而非补白页或虚拟页位作为单页锚点，禁止请求不存在的图片索引

### 需求:模式切换必须原子保持阅读位置

阅读器必须在提交目标显示模式前，根据当前模式、目标模式、当前页、总页数和补白位置计算合法实际页锚点及双页起始页位，并在同一模式切换事务中更新页码与模式。在线预览和本地漫画库阅读器必须复用同一套计算规则，禁止通过模式提交后的补偿 effect 修正可见页码。

#### 场景:当前单页成为双页右页

- **当** 当前实际页按既有双页配对规则应位于右侧
- **那么** 阅读器必须选择包含该实际页的合法双页起始页位，并在首次双页渲染中保留该页可见

#### 场景:章节首尾切换模式

- **当** 用户在章节第一页或最后一页切换任一显示模式
- **那么** 目标锚点和双页起始页位必须限制在合法范围内，并继续遵守既有补白页规则

#### 场景:在线与本地阅读器切换同一页

- **当** 在线阅读器和本地阅读器分别在相同页码、总页数和补白配置下执行相同模式切换
- **那么** 两者必须得到相同的实际页锚点和双页页组

### 需求:模式过渡期间必须协调输入和页追踪

显示模式过渡期间，阅读器必须冻结可能与过渡状态竞争的滚轮翻页、图片拖拽平移、进度条拖拽和可见页追踪写入。过渡成功、取消、被新目标取代或组件卸载时必须可靠解除冻结；禁止让阅读器永久停留在不可交互状态。

#### 场景:过渡期间滚动滚轮

- **当** 用户在旧模式退出或新模式进入期间滚动滚轮
- **那么** 阅读器必须忽略本次翻页或滚动输入，禁止改变目标页或叠加翻页动画

#### 场景:滚动视图定位触发观察器通知

- **当** 进入连续滚动模式的程序化定位产生旧页或中间页的可见性观察通知
- **那么** 页追踪必须保持冻结，直到目标页定位稳定后才恢复，禁止覆盖模式切换锚点

#### 场景:过渡中关闭阅读器

- **当** 用户在显示模式过渡期间关闭阅读器
- **那么** 所有过渡回调、定时器和输入冻结状态必须被清理，禁止在卸载后更新状态

### 需求:快速连续模式选择必须以最新意图为准

模式过渡协调器必须采用 latest-intent-wins。任一时刻最多只能有一个有效目标模式和一组可见阅读内容；所有异步动画完成回调必须校验过渡身份，过期回调禁止提交旧目标、改变页码、改变可见性或解除新过渡的输入冻结。

#### 场景:退出阶段连续选择单页和双页

- **当** 用户在连续滚动视图淡出期间先选择单页、随后选择双页
- **那么** 阅读器必须跳过过时的单页目标并最终进入双页模式，禁止依次播放两次完整过渡

#### 场景:过期完成回调晚到

- **当** 旧模式切换的动画完成回调在更新的目标模式已经开始处理后返回
- **那么** 该回调禁止覆盖当前目标模式、页码、过渡阶段或交互冻结状态

#### 场景:选择当前模式

- **当** 用户点击当前已经稳定显示的模式
- **那么** 阅读器不得播放动画、改变页码或重新挂载阅读内容

### 需求:模式选择控件必须同步显示过渡目标

阅读设置中的模式选择控件必须使用单一活动指示背景平滑移动到最新目标模式，并通过 `aria-pressed` 或等效可访问状态暴露当前选择。活动指示必须立即响应最新用户意图，禁止等待内容动画完成后才更新，也禁止为三个按钮分别播放互不协调的背景淡入淡出。

#### 场景:选择另一个显示模式

- **当** 用户点击不同的显示模式按钮
- **那么** 活动指示必须立即移动到目标按钮，目标按钮必须暴露选中状态，内容区随后完成对应模式过渡

#### 场景:快速改变目标模式

- **当** 用户在活动指示尚未结束移动时选择第三个模式
- **那么** 指示器必须从当前位置连续移动到最新目标，禁止先完成过时目标的动画

### 需求:模式切换必须尊重 reduced-motion 并限制动画成本

当用户启用 `prefers-reduced-motion` 时，模式切换必须移除位移与缩放并退化为不超过 `DURATION.fast` 的纯透明度过渡或由全局策略压缩为瞬时切换。普通模式过渡必须复用 `src/lib/anim.ts` 的集中令牌和无 overshoot 平滑曲线；连续滚动长列表禁止整体执行 transform、layout 或常驻 `will-change` 动画。

#### 场景:reduced-motion 下切换单页和双页

- **当** 用户启用 reduced-motion 并在单页与双页之间切换
- **那么** 页面禁止移动或缩放，只允许短时纯透明度变化或瞬时显示目标布局

#### 场景:长章节从连续滚动切换到单页

- **当** 连续滚动视图包含数百个已挂载页面并开始模式切换
- **那么** 系统只允许对裁剪后的阅读内容视口执行透明度过渡，禁止对整个长列表执行 transform 或 layout 动画

#### 场景:模式过渡完成

- **当** 任一普通模式过渡结束
- **那么** 动画容器禁止保留常驻 `will-change`，且阅读内容必须恢复正常输入处理

### 需求:普通翻页动画必须与显示模式重排动画互斥

当 `currentPage` 在稳定的 single 或 double 模式中变化时，系统必须只运行外层方向感知横向翻页动画。页槽的 layout、presence、opacity variants 和模式 transition 必须保持禁用；仅当显示模式协调器处于过渡阶段时才允许页槽参与模式重排。禁止普通翻页同时执行两套动画导致轨迹漂移、缩放或不自然的复合淡入淡出。

#### 场景:双页模式正常翻到下一组

- **当** 阅读器稳定处于 double 模式且当前页组向前变化
- **那么** 两页作为静态整体执行一次向左退出/从右进入的横向动画，内部页槽不得额外执行 layout 或 presence 动画

#### 场景:单页与双页进行模式重排

- **当** 用户从 single 切换到 double 或从 double 切换到 single
- **那么** 页槽允许执行模式 layout/opacity 重排，且本次变化不得启动普通横向翻页动画

### 需求:双页模式必须按高度最大化并保持无缝中缝

double 模式必须在保持图片原始宽高比的前提下尽量使用阅读内容区的完整高度，禁止复用单页 `imageWidth` 百分比把两页整体缩小并产生上下留白。左右漫画页之间的布局 gap 必须为 0，禁止在书脊位置插入人为间距。

#### 场景:宽窗口显示两张竖版漫画页

- **当** 两张竖版页面并排后的自然宽度不超过阅读内容区宽度
- **那么** 两页必须顶满可用高度并在中缝处无间距贴合，图片不得横向拉伸

### 需求:阅读器显示模式必须记忆并跨阅读器实例同步

系统必须把最后选择的 `scroll`、`single` 或 `double` 持久化到 localStorage，并让在线预览与本地漫画阅读器的所有已挂载设置实例共享同一快照。重新打开阅读器或重启应用时必须恢复最后选择，禁止固定回退到 double 或由陈旧实例覆盖最新选择。

#### 场景:在线阅读器选择单页后打开本地阅读器

- **当** 用户在在线阅读器选择 single 并关闭，随后打开本地漫画阅读器
- **那么** 本地阅读器必须以 single 打开，模式控件与内容视图保持一致

#### 场景:应用重启后恢复显示模式

- **当** 用户最后选择 double 后关闭并重新启动应用
- **那么** 新打开的任一阅读器必须恢复 double；若存储值无效则回退到 scroll

### 需求:模式协调器必须兼容 React StrictMode 生命周期探测

模式协调器必须在每次 effect setup 时恢复挂载状态，并在 cleanup 时取消定时器和失效旧 token。React StrictMode 执行 setup-cleanup-setup 后，滚动与分页模式切换仍必须完成并解除输入冻结。

#### 场景:StrictMode 下切换到连续滚动

- **当** 开发环境 StrictMode 完成 effect 生命周期探测后用户切换到 scroll
- **那么** 目标滚动层必须完成准备并恢复可见，禁止永久停留在隐藏的 exiting/preparing 阶段

#### 场景:StrictMode 下单页切换双页

- **当** 用户从 single 切换到 double
- **那么** entering 阶段必须按时结束并解除滚轮、键盘、点击和拖拽输入冻结
