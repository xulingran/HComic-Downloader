## 修改需求

### 需求:本地阅读复用现有阅读体验

本地阅读必须复用现有滚动、单页/双页、缩放、页间距、预加载、失败重试、显示模式过渡和 reduced-motion 行为；本地与远程阅读器必须共享显示模式切换时的实际页锚点、双页配对、补白页映射、快速切换、输入门控和最后选择的持久化显示模式。显示模式切换必须继续复用已物化的本地图片 URL 和共享页面缓存，禁止因切换模式重复读取或物化已缓存的当前页。双页必须按阅读区高度最大化显示且两页中缝为零。新增本地数据源禁止改变远程预览阅读路径的既有行为。

#### 场景:在本地漫画中使用双页模式

- **当** 用户将本地阅读模式从单页切换为双页并继续翻页
- **那么** 阅读器必须沿用现有双页布局、空白页规则和模式重排动画，图片数据来自本地资产页接口

#### 场景:本地滚动模式切换到分页模式

- **当** 用户在本地连续滚动模式的某一实际页切换到单页或双页模式
- **那么** 目标模式必须保留该实际页可见并复用已物化的图片 URL，禁止重新调用本地页面物化接口

#### 场景:本地分页模式切换到滚动模式

- **当** 用户从本地单页或双页模式切换到连续滚动模式
- **那么** 滚动视图必须在淡入前定位到切换锚点，并在定位稳定后恢复可见页追踪

#### 场景:本地与在线模式切换一致

- **当** 本地阅读器和在线预览阅读器以相同页码、页数和补白配置执行相同模式切换
- **那么** 两者必须采用相同过渡语义并显示相同实际页或双页组合

#### 场景:本地与在线阅读器共享最后显示模式

- **当** 用户在任一阅读器选择 single、double 或 scroll 后关闭，并打开另一种阅读器
- **那么** 新阅读器必须恢复最后选择的模式，禁止使用自身挂载时的陈旧模式

#### 场景:本地双页使用完整高度且无中缝

- **当** 本地漫画以 double 模式显示且两页自然宽度允许占满阅读区高度
- **那么** 两页必须保持比例、尽量占满上下空间并以 0 间距贴合

#### 场景:打开远程预览

- **当** 用户从搜索页打开远程漫画预览
- **那么** 阅读器必须继续通过既有预览 IPC 获取图片并使用共享模式过渡，禁止依赖漫画库索引或本地页面接口

## 移除需求

无。
