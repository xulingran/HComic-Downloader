## 修改需求

### 需求:未设置默认来源时启动后首次进入收藏夹必须引导选择来源

当 `defaultFavouriteSource` 配置为空且本次应用启动内尚未处理过收藏夹来源选择时，收藏夹 tab 在用户第一次进入时必须弹出来源选择器，禁止自动从任何来源加载收藏夹。来源侧边栏可以在弹窗背后渲染，但在用户真正选择前禁止显示选中来源。

#### 场景:未设置默认来源且本会话首次进入

- **当** 用户未在设置中配置 `defaultFavouriteSource`（值为空字符串）且本次应用启动后首次点击收藏夹 tab
- **那么** 系统必须弹出来源选择器（仅包含 `SOURCES_WITH_FAVOURITES` 中的来源），并且禁止在弹出期间从任何来源发起 `getFavourites` 请求
- **且** 来源侧边栏禁止高亮任何来源

#### 场景:用户在弹窗中选择来源

- **当** 用户在来源选择器中点击某个来源（如 jm）
- **那么** 系统必须关闭弹窗、将该来源设为当前来源、标记本会话已处理过选择，并以该来源加载收藏夹第一页
- **且** 来源侧边栏必须高亮该来源

#### 场景:用户关闭弹窗但未选择

- **当** 用户通过 ESC、遮罩或“稍后再说”关闭来源选择器而未选择任何来源
- **那么** 系统必须标记本会话已展示过选择器、禁止再次自动弹出，并显示无收藏列表、无加载中的未选择状态
- **且** 系统必须保留常驻来源侧边栏作为直接选择入口，侧边栏中禁止高亮任何来源

#### 场景:关闭弹窗后通过侧边栏选择来源

- **当** 用户关闭来源选择器后点击侧边栏中的某个来源
- **那么** 系统必须将该来源设为当前来源并加载其第一页
- **且** 本会话后续进入收藏夹 tab 禁止再次自动弹出来源选择器

#### 场景:关闭弹窗后手动重新打开选择器

- **当** 用户关闭来源选择器后点击未选择空状态中的“选择来源”按钮
- **那么** 系统必须允许重新打开来源选择器
- **且** 用户随后选择来源时必须执行与首次选择相同的加载和高亮行为

### 需求:后续操作必须延续现有逻辑

来源选择器处理完成后，收藏夹 tab 内的刷新、分页、批量下载和预加载必须延续既有行为；后续来源切换必须改由常驻来源侧边栏触发，并复用统一的来源加载路径。

#### 场景:选择来源后切换其他来源

- **当** 用户通过来源选择器选择 JM 并加载完成后，再通过来源侧边栏切换到哔咔
- **那么** 系统必须把哔咔设为当前来源并加载哔咔第一页，命中缓存时立即展示缓存并后台刷新
- **且** 禁止再次自动弹出来源选择器

#### 场景:选择来源后的分页与刷新

- **当** 用户在来源选择器选择来源后执行翻页或点击刷新按钮
- **那么** 系统必须按现有分页与刷新逻辑处理，行为与引入来源侧边栏前保持一致
