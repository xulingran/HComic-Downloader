## 为什么

收藏夹目前依赖标题旁的原生下拉框切换来源，入口不够醒目，也无法直观表达“不同来源拥有彼此独立的收藏夹”。同时该下拉框使用全量漫画来源，可能展示并不支持收藏夹的拷贝漫画与 NH，给用户造成可以切换但实际不可用的错觉。

## 变更内容

- 在收藏夹页面左侧新增类似设置页的来源侧边栏，持续展示可用收藏来源并高亮当前来源。
- 侧边栏严格使用 `SOURCES_WITH_FAVOURITES`，仅展示 HComic、MoeImg、JM、哔咔。
- 移除收藏夹标题旁的来源下拉框；用户点击侧边栏来源时复用现有缓存、加载、预加载中断和错误处理路径。
- 来源切换统一从目标来源第一页开始，若第一页已有缓存则立即显示并在后台刷新。
- 保留默认收藏夹来源和首次进入来源选择弹窗；用户跳过选择时侧边栏保持无选中状态，并可作为后续选择入口。
- 调整收藏夹内容区的网格断点与顶部工具栏换行规则，避免全局侧边栏和来源侧边栏同时占宽后卡片或操作区被过度挤压。

## 功能 (Capabilities)

### 新增功能

- `favourite-source-sidebar`: 定义收藏夹来源侧边栏的来源范围、选中状态、切换行为、响应式布局与可访问性要求。

### 修改功能

- `favourite-source-picker`: 将首次选择弹窗之后的手动来源切换入口从顶部下拉框改为常驻来源侧边栏，并定义跳过选择时的侧边栏状态。
