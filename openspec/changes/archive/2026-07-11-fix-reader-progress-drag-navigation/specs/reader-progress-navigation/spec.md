## 新增需求

### 需求:阅读器进度条必须支持连续拖拽定位

在线预览阅读器和本地漫画库阅读器的底部页面进度条必须在用户按下时根据指针横向位置计算当前章节内的目标页，并在指针保持按下且移动时持续更新目标页。目标页必须限制在 `1..effectiveTotal` 范围内；拖动到轨道外侧时禁止产生越界页码。

#### 场景:按下轨道立即定位

- **当** 当前章节共有 20 个有效页位，用户在进度轨道横向 50% 处按下指针
- **那么** 阅读器必须将目标页更新为第 10 页，并立即呈现第 10 页对应的阅读位置

#### 场景:保持按下并持续拖动

- **当** 用户按住进度条并从第 3 页对应位置拖动到第 15 页对应位置
- **那么** 阅读器必须随离散目标页变化持续更新阅读位置，禁止只在释放指针后才跳转

#### 场景:拖动超出轨道边界

- **当** 用户捕获指针后将其拖到进度轨道左侧或右侧之外
- **那么** 阅读器必须分别将目标限制为第一页或最后一个有效页位，禁止写入 0 或超出总页数的页码

### 需求:连续滚动模式必须将目标页同步为正文位置

当显示模式为连续滚动且页面内容已加载时，进度条产生的新目标页必须使对应页面容器即时进入阅读视口。页码、进度条视觉位置和正文位置必须指向同一目标页；仅更新页码状态而不移动正文禁止发生。

#### 场景:本地阅读器拖动后滚到目标页

- **当** 用户在本地漫画库阅读器的连续滚动模式中把进度条从第 1 页拖到第 8 页
- **那么** 第 8 页的页面容器必须即时滚动到阅读视口起始位置，页码指示必须显示第 8 页

#### 场景:在线阅读器拖动后滚到目标页

- **当** 用户在在线预览阅读器的连续滚动模式中把进度条拖到一个已渲染的目标页
- **那么** 该目标页容器必须即时进入阅读视口，且在线预加载机制必须继续以最终目标页作为跳转目标

#### 场景:目标页容器暂时不可用

- **当** 拖动目标页对应的页面容器尚未挂载或已经卸载
- **那么** 阅读器必须安全保留受限后的目标页状态并跳过本次滚动，禁止抛出异常或关闭阅读器

### 需求:拖动期间页面可见性追踪不得覆盖用户目标

连续滚动模式开始进度条拖动后，阅读器必须暂停由页面可见性观察器发起的当前页写入。拖动完成或取消后，阅读器必须在程序化滚动产生的观察通知稳定后恢复正常可见页追踪。

#### 场景:观察器在拖动期间报告旧页面

- **当** 用户正在把进度条拖向第 12 页，页面可见性观察器同时报告原先可见的第 2 页
- **那么** 阅读器必须保留第 12 页作为拖动目标，禁止被观察器改回第 2 页

#### 场景:拖动结束后恢复手动滚动追踪

- **当** 用户释放进度条且程序化滚动产生的观察通知已经稳定，随后手动滚动到另一页
- **那么** 阅读器必须恢复可见页追踪，并将当前页更新为手动滚动后实际可见的页面

### 需求:单页与双页模式必须直接切换到目标页位

当显示模式为单页或双页时，进度条拖动必须直接更新 `PageFlipView` 使用的当前页位。双页模式的目标范围必须使用包含前补白页的有效总页数，并继续遵守现有双页布局与补白页解释，禁止把补白页错误映射为不存在的图片索引。

#### 场景:单页模式拖动跳转

- **当** 用户在单页模式中将进度条拖到第 7 页
- **那么** `PageFlipView` 必须立即显示以第 7 页为当前页的内容

#### 场景:双页前补白模式拖到末端

- **当** 双页模式启用前补白且有效总页数比图片总数多 1，用户将进度条拖到最右端
- **那么** 阅读器必须使用最后一个有效页位渲染既有双页布局，禁止产生越界图片请求

### 需求:拖动生命周期必须可靠清理

阅读器必须通过 pointer capture 在指针离开轨道后继续接收拖动事件。正常释放时必须结束拖动态并对有效最终页执行一次拖动完成回调；`pointercancel` 或丢失 pointer capture 时必须幂等清除拖动态与页追踪冻结，禁止让阅读器永久停留在拖动状态。

#### 场景:在轨道外释放指针

- **当** 用户从进度条开始拖动、把指针移出轨道后释放
- **那么** 阅读器必须完成到受限最终页的跳转、结束拖动视觉状态，并允许后续页面追踪恢复

#### 场景:系统取消指针序列

- **当** 拖动过程中发生 `pointercancel` 或组件丢失 pointer capture
- **那么** 阅读器必须清除拖动态和冻结状态，后续再次拖动必须能够正常开始
