## 新增需求

### 需求:NH 搜索页必须提供仅中文筛选

系统必须仅在当前来源为 NH 时展示“仅显示中文”筛选控件。筛选必须默认关闭，且不得写入持久化配置；其他来源禁止展示或应用该筛选。

#### 场景:进入 NH 来源时展示默认状态
- **当** 用户冷启动进入 NH 来源且本次运行尚未修改筛选
- **那么** 搜索页必须展示“仅显示中文”控件
- **且** 控件必须处于关闭状态

#### 场景:非 NH 来源不展示筛选
- **当** 当前搜索来源不是 NH
- **那么** 搜索页禁止展示“仅显示中文”控件
- **且** 搜索请求禁止携带 NH 语言筛选参数

#### 场景:在 NH 入口页切换筛选
- **当** 用户位于没有可见结果的 NH 入口页并开启或关闭“仅显示中文”
- **那么** 系统必须记录本次运行中的筛选状态
- **且** 禁止仅因切换筛选立即发起内容请求
- **且** 用户随后进入最近更新、热门排行或热门标签时必须应用该状态

#### 场景:在结果页切换筛选
- **当** 用户正在查看 NH 搜索结果并改变“仅显示中文”状态
- **那么** 系统必须清除当前批量选择并从第 1 页重新搜索
- **且** 禁止继续展示与新筛选状态不一致的旧页结果

### 需求:仅中文筛选必须由 NH 服务端查询执行

筛选开启时，系统必须使用 NH 搜索 API 的 `language:"chinese"` 限定组合请求，禁止把 `tag:"chinese"` 当作语言限定，也禁止依赖搜索列表项中的 `ComicInfo.language` 在前端逐项剔除。筛选关闭时必须保持现有 NH 搜索语义。

#### 场景:筛选最近更新
- **当** 用户开启仅中文筛选并进入 NH 最近更新
- **那么** 系统必须通过搜索 API 请求 `language:"chinese"` 的第 1 页
- **且** 必须使用默认日期顺序而非热门排序

#### 场景:筛选关键词结果
- **当** 用户开启仅中文筛选并以关键词 `sample` 搜索 NH
- **那么** 服务端查询必须同时包含 `sample` 与 `language:"chinese"`
- **且** 搜索框和搜索历史必须只保留用户输入的 `sample`，禁止暴露内部语言限定

#### 场景:筛选精确标签结果
- **当** 用户开启仅中文筛选并选择 NH 标签 `full color`
- **那么** 服务端查询必须同时包含 `tag:"full color"` 与 `language:"chinese"`
- **且** 禁止改用普通关键词或 `tag:"chinese"` 查询

#### 场景:筛选热门排行
- **当** 用户开启仅中文筛选并查看任一 NH 热门排行
- **那么** 服务端查询必须包含 `language:"chinese"`
- **且** 必须保留所选 `popular-today`、`popular-week`、`popular-month` 或 `popular` 排序

#### 场景:关闭筛选保持原行为
- **当** 用户关闭仅中文筛选后执行 NH 搜索
- **那么** 系统禁止附加 `language:"chinese"`
- **且** 最近更新、关键词、标签和排行必须保持现有查询行为

### 需求:筛选状态必须参与分页缓存与预加载上下文

系统必须把 NH 语言筛选作为搜索上下文的独立维度。筛选结果的分页信息必须直接采用 NH 服务端过滤后的 `total` 与 `num_pages`；翻页、缓存命中和相邻页预加载必须继续携带同一筛选状态。

#### 场景:过滤后的分页准确
- **当** NH API 返回仅中文查询的分页总数和结果总数
- **那么** 搜索页必须展示这些服务端分页值
- **且** 禁止用未过滤查询的总数或当前页前端过滤后的数量替代

#### 场景:筛选结果翻页
- **当** 用户在仅中文结果中从第 1 页导航到第 2 页
- **那么** 第 2 页请求必须继续携带中文语言筛选
- **且** 返回结果必须写入仅中文搜索上下文的第 2 页缓存

#### 场景:筛选缓存相互隔离
- **当** 同一 NH 查询分别存在筛选开启和关闭的缓存页
- **那么** 两种状态必须生成不同的搜索上下文键
- **且** 任一状态禁止读取或覆盖另一状态的缓存

#### 场景:切换筛选时中断旧预加载上下文
- **当** 旧筛选状态的相邻页预加载尚未完成且用户切换筛选
- **那么** 旧结果禁止提交到新筛选状态的缓存或可见列表
- **且** 新预加载请求必须携带新筛选状态

### 需求:中文筛选结果必须携带可信语言元数据

当服务端查询已由 `language:"chinese"` 限定时，系统必须为列表返回项补充 `language="chinese"`，即使 NH 搜索响应只提供 `tag_ids`。未启用筛选时，系统禁止仅凭缺失的列表标签推断语言。

#### 场景:筛选响应缺少完整标签
- **当** 仅中文搜索响应的列表项包含 `tag_ids` 但不包含 `tags`
- **那么** 返回前端的 `ComicInfo.language` 必须为 `chinese`

#### 场景:未筛选响应缺少完整标签
- **当** 未筛选搜索响应的列表项不包含可解析的语言标签
- **那么** 系统禁止将其语言默认标记为 `chinese`
