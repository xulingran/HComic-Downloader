## 新增需求

### 需求:下载到打包必须验证真实文件系统行为

系统必须提供集成测试，用真实文件系统（而非 mock 的文件操作）验证从下载任务创建到 CBZ 文件生成的完整流转。

#### 场景:完整下载流程生成有效 CBZ

- **当** 创建一个下载任务，其图片数据由 fixture bytes 提供（而非真实 HTTP）
- **那么** 任务状态必须流转为 `completed`，且目标路径必须生成一个有效的 `.cbz` 文件，文件内必须包含正确命名的图片和 `ComicInfo.xml`

#### 场景:下载中断后恢复不损坏已有数据

- **当** 下载进行中被取消或中断，随后重试
- **那么** 已下载的分片不得损坏，最终 CBZ 必须完整且可正常解压

### 需求:IPC 契约必须验证运行时返回结构

系统必须提供运行时契约测试，验证 Python 后端关键方法的返回结构符合前端 TypeScript 类型定义，防止前后端契约漂移。

#### 场景:get_config 返回结构匹配前端类型

- **当** 进程内实例化 `IPCServer` 并调用 `get_config`
- **那么** 返回的字典必须包含前端 `Config` 类型定义的全部必需字段，且字段类型匹配

#### 场景:search 返回结构匹配前端类型

- **当** 调用 `search` 方法（解析器以 fixture 数据注入）
- **那么** 返回结构必须匹配前端搜索响应类型，包括分页元信息和漫画条目字段

### 需求:端到端冒烟必须守护进程间通信边界

系统必须提供至少一条真实 spawn Python 子进程的冒烟测试，验证 JSON-RPC over stdin/stdout 通信链路的完整性。

#### 场景:真实子进程响应 JSON-RPC 请求

- **当** 真实 spawn `ipc_server.py` 子进程，发送合法的 `get_config` JSON-RPC 请求
- **那么** 必须收到合法的 JSON 响应，响应必须包含预期的配置字段

#### 场景:子进程优雅响应 shutdown

- **当** 向运行中的子进程发送 `shutdown` 请求
- **那么** 子进程必须在合理时间内优雅退出，退出码必须为 0

> 注：本需求的端到端冒烟标记为可选（Phase 4），实施时可根据复杂度决定是否启用。启用时必须用 pytest marker（如 `@pytest.mark.smoke`）标记，支持通过 `pytest -m "not smoke"` 在快速反馈场景跳过。
