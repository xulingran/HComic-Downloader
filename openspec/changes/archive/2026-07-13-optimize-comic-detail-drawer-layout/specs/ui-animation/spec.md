## 修改需求

### 需求: ComicInfoDrawer 的 tag 列表必须错峰出现

ComicInfoDrawer 的 drawer 表面在首次打开时，当前折叠态实际可见的标签**必须**复用 `tagListVariants` 与 `tagItemVariants` 错峰出现，间隔必须由集中动画令牌统一控制（当前为 20ms，起始延迟 100ms），参与错峰的可见标签最多 20 个。因用户点击“+N 展开”而新增的标签必须立即出现或以一次短时整体淡入出现，**禁止**对完整长标签列表再次逐项级联导致明显等待。reader 表面继续展示完整标签，并必须遵循同一最多 20 项的错峰上限。用户启用 reduced-motion 时，所有标签必须直接出现且禁止位移动画。

#### 场景:抽屉打开时仅对可见标签错峰

- **当** ComicInfoDrawer 的 drawer 表面打开且折叠态显示 N 个标签（N ≤ 20）
- **那么** 这 N 个可见标签必须按集中令牌的 20ms 间隔依次淡入上移，总时长由令牌统一控制
- **且** 尚未展开的标签禁止参与动画或占用动画时长

#### 场景:展开标签不触发长级联

- **当** 用户点击“+N 展开”显示折叠上限之外的标签
- **那么** 新增标签必须立即出现或作为一个整体短时淡入
- **且** 禁止对新增标签逐项执行无上限的 `staggerChildren`

#### 场景:reader 完整标签动画封顶

- **当** reader 表面包含超过 20 个标签
- **那么** 仅前 20 个标签允许参与错峰，第 21 个及之后必须立即出现，避免动画总时长随标签总量持续增长

#### 场景:减少动画时标签直接出现

- **当** 用户启用「减少动画」并打开 drawer 或 reader 详情表面
- **那么** 所有当前渲染的标签必须直接出现，禁止淡入上移或错峰延迟
