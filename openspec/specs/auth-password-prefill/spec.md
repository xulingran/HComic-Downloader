# auth-password-prefill 规范

## 目的

定义设置页对已保存账号密码凭据的回填行为，确保支持账号密码登录的来源在重新进入设置页时可继续显示已保存凭据，同时保持密码输入框默认隐藏等既有安全交互。

## 需求

### 需求:设置页必须回填已保存的账号密码

对于支持账号密码登录的来源（hcomic、moeimg、bika），设置页中的用户名和密码输入框必须显示当前配置中已保存的对应值，使用户无需重新输入即可再次点击登录。

#### 场景:配置异步加载后用户名正确回填

- **当** 用户打开设置页且后端配置中包含 `hcomicUsername`/`moeimgUsername`/`bikaUsername`
- **那么** 对应来源的用户名输入框必须在配置加载完成后显示保存的值

#### 场景:配置异步加载后密码正确回填

- **当** 用户打开设置页且后端配置中包含 `hcomicPassword`/`moeimgPassword`/`bikaPassword`
- **那么** 对应来源的密码输入框必须在配置加载完成后显示保存的值

#### 场景:登录失败后重新进入设置页凭据仍可见

- **当** 用户此前因网络异常或密码错误导致账号密码登录失败，且凭据已被持久化
- **那么** 再次打开设置页时，账号和密码输入框必须显示上次提交的值

### 需求:密码回填不得改变既有安全行为

密码在前端必须以受控输入框形式呈现，默认隐藏字符，用户可通过显式操作临时查看明文；除此之外不得额外暴露或打印密码。

#### 场景:密码输入框默认隐藏字符

- **当** 密码输入框被回填后
- **那么** 输入框类型必须为 `password`，字符默认不可见

#### 场景:用户可临时查看明文密码

- **当** 用户按下/点击密码输入框旁的眼睛图标
- **那么** 输入框类型临时切换为 `text`，松开/离开后恢复为 `password`
