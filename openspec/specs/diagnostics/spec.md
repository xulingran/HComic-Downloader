# diagnostics 规范

## 目的

定义应用诊断信息的收集、敏感内容提示与剪贴板导出行为，使用户能够安全地提供环境和日志线索，同时避免未经提示泄露潜在隐私数据。

## 需求

### 需求: 系统必须支持一键复制诊断信息到剪贴板

系统必须提供一个 IPC 通道，由 Electron 主进程收集环境信息与日志文件尾部，组装成结构化诊断报告，供渲染进程一键复制到剪贴板。

#### 场景: 用户复制诊断日志

- **当** 用户在设置页或致命横幅点击"复制诊断日志"
- **那么** 渲染进程调用 `window.hcomic.getDiagnostics()`，主进程读取 `main.log` 和 `python.log` 尾部 + 拼装环境信息，返回结构化字符串，渲染进程写入剪贴板

#### 场景: 诊断报告包含环境元信息

- **当** 生成诊断报告
- **那么** 报告包含：应用版本、平台、Electron 版本、生成时间

#### 场景: 诊断报告包含两端日志尾部

- **当** 生成诊断报告
- **那么** 报告包含 `main.log` 最近若干行（默认 200 行）和 `python.log` 最近若干行（默认 200 行）

#### 场景: 日志文件不存在时降级处理

- **当** 日志文件尚未生成或读取失败
- **那么** 报告中对应部分显示"(日志文件不存在)"或"(读取失败)"，不阻断整体报告生成

### 需求: 复制诊断信息前必须提示潜在敏感内容

系统必须在用户触发复制诊断信息前，提示日志可能包含 cookie、搜索词等敏感信息，由用户确认后再复制。

#### 场景: 提示敏感信息

- **当** 用户点击"复制诊断日志"
- **那么** 系统弹出确认提示，说明日志可能包含敏感信息，用户确认后才执行复制
