# nh-entry-page 规范

## 目的

定义 NH 来源搜索入口、最近更新、热门排行、热门标签以及返回入口时的路由与交互契约。确保匿名用户进入该来源时不会误触随机请求、认证阻断或跨来源内容降级。

## 需求

### 需求: 不支持随机的来源禁止跨来源降级

随机请求必须仅接受明确支持随机能力的来源。对于 NH 等不支持随机的来源，后端必须返回明确校验错误，禁止静默改用 HComic 或返回任何其他来源的漫画。

#### 场景:NH 随机请求被拒绝

- **当** 后端收到 `source="nh"` 的随机请求
- **那么** 请求必须以“不支持随机”错误失败
- **且** 禁止调用 HComic parser 或返回 HComic 漫画

#### 场景:支持随机的来源保持原行为

- **当** 后端收到 HComic、JM 或哔咔等已声明支持随机的来源请求
- **那么** 系统必须调用对应来源的随机实现
- **且** 禁止因本变更改变其结果来源

### 需求: NH 来源入口页

系统必须将 NH 入口页作为该来源的显式初始路由。用户切换到 NH，或应用冷启动且默认来源为 NH、同时没有可恢复的 NH 搜索缓存时，系统必须清空旧来源结果并展示入口页；禁止自动执行搜索、认证预校验或随机请求。入口页必须提供最近更新、热门排行和热门标签快捷入口。存在 keep-alive 搜索缓存时可以恢复用户之前的 NH 结果，但必须保留返回入口页的操作。NH 认证状态仅由 API Key 决定。

#### 场景:从其他来源切换到 NH

- **当** 用户在搜索页从任意其他来源选择 NH
- **那么** 系统必须清空旧来源漫画、分页、错误、查询词和已选标签
- **且** 必须展示含「最近更新」「热门排行」和热门标签区域的 NH 入口页
- **且** 禁止调用随机接口或自动加载最近更新

#### 场景:已配置 API Key 时切换到 NH

- **当** 用户已配置有效 NH API Key，并从其他来源切换到 NH
- **那么** 系统仍必须先展示 NH 入口页
- **且** 禁止因为认证有效而自动调用随机、搜索或收藏接口

#### 场景:未配置 API Key 时切换到 NH

- **当** 用户未配置 NH API Key 并切换到 NH
- **那么** 系统必须允许进入 NH 入口页并使用匿名浏览能力
- **且** 禁止显示全页登录阻断

#### 场景:冷启动默认来源为 NH

- **当** 应用冷启动读取到 `defaultSource="nh"` 且没有可恢复的搜索缓存
- **那么** 系统必须直接展示 NH 入口页
- **且** 禁止自动执行空关键词搜索或随机请求

#### 场景:恢复 NH 搜索缓存

- **当** SearchPage 通过 keep-alive 恢复到先前的 NH 最近更新、热门排行或标签结果
- **那么** 系统可以恢复缓存结果及其查询上下文
- **且** 必须提供“返回 NH 入口”操作

#### 场景:进入最近更新
- **当** 用户点击 NH 入口页的「最近更新」入口
- **那么** 系统必须以 NH 来源请求第一页最近更新结果
- **且** 请求必须使用空关键词的普通搜索语义
- **且** 结果加载后入口页必须隐藏并展示漫画列表

#### 场景:进入热门排行
- **当** 用户点击 NH 入口页的「热门排行」入口
- **那么** 系统必须以 NH 来源请求第一页 popular 排序结果
- **且** 前端搜索上下文必须能区分热门排行与最近更新，避免分页或缓存混用
- **且** 结果加载后入口页必须隐藏并展示漫画列表

#### 场景:返回入口页

- **当** 用户正在查看 NH 最近更新、热门排行或标签快捷入口结果并点击返回入口
- **那么** 系统必须清空当前漫画结果、分页和错误状态
- **且** 必须重新展示 NH 入口页而不发起网络内容请求

### 需求: NH 入口页热门标签快捷入口

系统必须在 NH 入口页使用 NH 标签目录数据展示热门标签快捷入口，禁止使用仅写死在前端的静态热门标签列表作为主要数据源。

#### 场景:展示已同步的热门标签
- **当** 本地标签目录中存在 NH 标签数据
- **那么** NH 入口页必须按热门排序展示前若干个标签
- **且** 每个标签必须显示标签名
- **且** 当 count 可用时必须显示数量

#### 场景:标签未同步时提示刷新
- **当** 本地标签目录中没有 NH 标签数据
- **那么** NH 入口页必须显示标签同步提示
- **且** 用户必须可以从该区域触发标签目录刷新
- **且** 标签目录刷新失败时不得影响最近更新和热门排行入口的可用性

#### 场景:点击热门标签执行精确标签搜索
- **当** 用户点击 NH 入口页中的热门标签
- **那么** 系统必须以 NH 来源执行标签搜索
- **且** 搜索语义必须限定到该标签，而不是普通关键词搜索
- **且** 结果加载后必须展示漫画列表和分页控件

### 需求: NH 入口子功能返回按钮持续可见

用户从 NH 入口页进入任一子功能（最近更新、热门排行、热门标签）后，「返回 NH 入口」按钮必须持续可见，直到用户通过显式退出动作（点击「返回 NH 入口」、切换到其他来源、点击随机）离开入口体系。在入口子功能内执行的后续操作——包括关键词搜索、翻页、排行下拉切换、标签增删——禁止隐藏该按钮。该按钮的显隐控制必须与 NH 入口页网格（`NhEntryGrid`）的渲染控制相互独立，禁止由单个状态同时承担两个职责，以避免事件处理器写入语义不一致导致的按钮误隐藏与入口页网格误重渲染。

#### 场景:进入热门排行后执行关键词搜索时按钮仍可见
- **当** 用户从 NH 入口页点击「热门排行」进入排行结果，且结果加载完成显示「返回 NH 入口」按钮
- **且** 用户在搜索框输入关键词并点击「搜索」触发关键词搜索
- **那么** 关键词搜索加载完成后，「返回 NH 入口」按钮必须仍然可见
- **且** 禁止因搜索模式为 keyword 而隐藏该按钮

#### 场景:进入最近更新后翻页时按钮仍可见
- **当** 用户从 NH 入口页点击「最近更新」进入结果列表，且结果加载完成显示「返回 NH 入口」按钮
- **且** 用户点击分页控件翻到下一页
- **那么** 翻页加载完成后，「返回 NH 入口」按钮必须仍然可见

#### 场景:在入口子功能内切换排行下拉时按钮仍可见
- **当** 用户已在 NH 热门排行结果中，「返回 NH 入口」按钮可见
- **且** 用户通过排行下拉切换到另一个排行选项
- **那么** 新排行结果加载完成后，「返回 NH 入口」按钮必须仍然可见

#### 场景:在入口子功能内增删标签时按钮仍可见
- **当** 用户已通过 NH 入口页热门标签进入标签搜索结果，「返回 NH 入口」按钮可见
- **且** 用户通过标签面板新增或移除标签触发新的标签搜索
- **那么** 新的标签搜索结果加载完成后，「返回 NH 入口」按钮必须仍然可见

#### 场景:点击返回入口页时按钮隐藏且入口页网格重现
- **当** 用户正在查看 NH 入口子功能结果，「返回 NH 入口」按钮可见
- **且** 用户点击「返回 NH 入口」按钮
- **那么** 系统必须清空当前漫画结果、分页和错误状态
- **且** 「返回 NH 入口」按钮必须隐藏
- **且** NH 入口页网格（最近更新 / 热门排行 / 热门标签）必须重新展示而不发起网络内容请求

#### 场景:切换到其他来源时按钮隐藏
- **当** 用户正在查看 NH 入口子功能结果，「返回 NH 入口」按钮可见
- **且** 用户通过来源下拉切换到非 NH 来源
- **那么** 「返回 NH 入口」按钮必须隐藏

#### 场景:点击随机时按钮隐藏
- **当** 用户正在查看 NH 入口子功能结果，「返回 NH 入口」按钮可见
- **且** 用户点击「随机」按钮（仅适用于支持随机的来源；NH 本身不支持随机，故此场景描述用户先切到支持随机的来源或本场景作为按钮隐藏语义的对称约束）
- **那么** 「返回 NH 入口」按钮必须隐藏

#### 场景:入口子功能内关键词搜索无果时不重渲染入口页网格
- **当** 用户从 NH 入口页进入热门排行或热门标签子功能，「返回 NH 入口」按钮可见
- **且** 用户进一步执行关键词搜索且无结果
- **那么** 系统必须显示「暂无搜索结果」空状态而非 NH 入口页网格
- **且** 「返回 NH 入口」按钮必须仍然可见，允许用户一键回到入口页
