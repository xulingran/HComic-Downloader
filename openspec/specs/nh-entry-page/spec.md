# nh-entry-page 规范

## 目的

定义 NH 来源搜索入口、最近更新、热门排行、热门标签以及返回入口时的路由与交互契约。确保匿名用户进入该来源时不会误触随机请求、认证阻断或跨来源内容降级。

## 需求

### 需求: 不支持随机的来源禁止跨来源降级

随机请求必须仅接受明确支持随机能力的来源。对于 NH 等不支持随机的来源，后端必须返回明确校验错误，禁止静默改用 HComic 或返回任何其他来源的漫画。

#### 场景:NH 随机请求被拒绝

- **当** 后端收到 `source="nh"` 的随机请求
- **那么** 请求必须以“不支持随机”错误失败
- **且** 禁止调用 HComic parser 或返回 HComic 漫画

#### 场景:支持随机的来源保持原行为

- **当** 后端收到 HComic、JM 或哔咔等已声明支持随机的来源请求
- **那么** 系统必须调用对应来源的随机实现
- **且** 禁止因本变更改变其结果来源

### 需求: NH 来源入口页

系统必须将 NH 入口页作为该来源的显式初始路由。用户切换到 NH，或应用冷启动且默认来源为 NH、同时没有可恢复的 NH 搜索缓存时，系统必须清空旧来源结果并展示入口页；禁止自动执行搜索、认证预校验或随机请求。入口页必须提供最近更新、热门排行和热门标签快捷入口。存在 keep-alive 搜索缓存时可以恢复用户之前的 NH 结果，但必须保留返回入口页的操作。NH 认证状态仅由 API Key 决定。

#### 场景:从其他来源切换到 NH

- **当** 用户在搜索页从任意其他来源选择 NH
- **那么** 系统必须清空旧来源漫画、分页、错误、查询词和已选标签
- **且** 必须展示含「最近更新」「热门排行」和热门标签区域的 NH 入口页
- **且** 禁止调用随机接口或自动加载最近更新

#### 场景:已配置 API Key 时切换到 NH

- **当** 用户已配置有效 NH API Key，并从其他来源切换到 NH
- **那么** 系统仍必须先展示 NH 入口页
- **且** 禁止因为认证有效而自动调用随机、搜索或收藏接口

#### 场景:未配置 API Key 时切换到 NH

- **当** 用户未配置 NH API Key 并切换到 NH
- **那么** 系统必须允许进入 NH 入口页并使用匿名浏览能力
- **且** 禁止显示全页登录阻断

#### 场景:冷启动默认来源为 NH

- **当** 应用冷启动读取到 `defaultSource="nh"` 且没有可恢复的搜索缓存
- **那么** 系统必须直接展示 NH 入口页
- **且** 禁止自动执行空关键词搜索或随机请求

#### 场景:恢复 NH 搜索缓存

- **当** SearchPage 通过 keep-alive 恢复到先前的 NH 最近更新、热门排行或标签结果
- **那么** 系统可以恢复缓存结果及其查询上下文
- **且** 必须提供“返回 NH 入口”操作

#### 场景:进入最近更新
- **当** 用户点击 NH 入口页的「最近更新」入口
- **那么** 系统必须以 NH 来源请求第一页最近更新结果
- **且** 请求必须使用空关键词的普通搜索语义
- **且** 结果加载后入口页必须隐藏并展示漫画列表

#### 场景:进入热门排行
- **当** 用户点击 NH 入口页的「热门排行」入口
- **那么** 系统必须以 NH 来源请求第一页 popular 排序结果
- **且** 前端搜索上下文必须能区分热门排行与最近更新，避免分页或缓存混用
- **且** 结果加载后入口页必须隐藏并展示漫画列表

#### 场景:返回入口页

- **当** 用户正在查看 NH 最近更新、热门排行或标签快捷入口结果并点击返回入口
- **那么** 系统必须清空当前漫画结果、分页和错误状态
- **且** 必须重新展示 NH 入口页而不发起网络内容请求

### 需求: NH 入口页热门标签快捷入口

系统必须在 NH 入口页使用 NH 标签目录数据展示热门标签快捷入口，禁止使用仅写死在前端的静态热门标签列表作为主要数据源。标签必须按 `count` 降序分为热门头部、中段、长尾三档，分别采用递减的视觉权重，并通过共享 stagger 动画错峰进场，reduced-motion 下退化为纯淡入。

#### 场景:展示已同步的热门标签
- **当** 本地标签目录中存在 NH 标签数据
- **那么** NH 入口页必须按热门排序展示前若干个标签
- **且** 每个标签必须显示标签名
- **且** 当 count 可用时必须显示数量
- **且** 标签必须按 `count` 降序分档展示，头部档位标签视觉权重最高，长尾档位视觉权重最低
- **且** 标签进场必须复用项目共享的 stagger 动画系统

#### 场景:标签未同步时提示刷新
- **当** 本地标签目录中没有 NH 标签数据
- **那么** NH 入口页必须显示标签同步提示
- **且** 用户必须可以从该区域触发标签目录刷新
- **且** 标签目录刷新失败时不得影响最近更新和热门排行入口的可用性

#### 场景:点击热门标签执行精确标签搜索
- **当** 用户点击 NH 入口页中的热门标签
- **那么** 系统必须以 NH 来源执行标签搜索
- **且** 搜索语义必须限定到该标签，而不是普通关键词搜索
- **且** 结果加载后必须展示漫画列表和分页控件

### 需求: NH 入口页热门标签按热度分档呈现

NH 入口页的热门标签区必须按标签 `count` 将标签分为「热门头部」「中段」「长尾」三档，并赋予递减的视觉权重，使热度差异可一眼辨识。分档必须基于已加载标签数据的 `count` 字段在前端推导，禁止依赖额外网络请求或后端新增字段。当标签总数不足以填满任一档位时，允许该档位为空，且剩余档位仍必须按规则展示。

#### 场景:标签数量充足时三档均展示

- **当** NH 入口页加载到至少 11 个带 `count` 的热门标签
- **那么** 系统必须按 `count` 降序排列标签
- **且** 前 N₁ 个标签必须以「热门头部」样式展示（强调实心底 + 加大字号 + 计数醒目）
- **且** 其后 N₂ 个标签必须以「中段」样式展示（淡强调底 + 中字号 + 计数可见）
- **且** 其余标签必须以「长尾」样式展示（中性底 + 小字号 + 计数弱化）

#### 场景:标签数量不足以填满头部档位

- **当** 加载到的热门标签总数少于头部档位阈值 N₁
- **那么** 系统必须把全部标签归入「热门头部」样式
- **且** 中段与长尾档位必须为空且不渲染占位

#### 场景:标签数量仅够头部档位

- **当** 加载到的热门标签总数恰好等于头部档位阈值 N₁
- **那么** 头部档位必须填满，中段与长尾档位必须为空

#### 场景:标签 count 全部为零时退化为统一档位

- **当** 加载到的标签 `count` 全部为 0 或不可用
- **那么** 系统必须把所有标签按单一中性样式渲染
- **且** 禁止按 `count` 分档，以避免无意义的档位区分

#### 场景:计数展示可读

- **当** 任一档位标签的 `count > 0`
- **那么** 该标签必须显示计数
- **且** 计数必须以独立、可读的徽章或加粗内联形式呈现
- **且** 禁止使用难以辨认的 10px 以下无对比字号紧贴标签名

#### 场景:同档位内保持热度顺序

- **当** 多个标签落入同一档位
- **那么** 同档位标签之间必须保持按 `count` 降序、标签名升序的排列
- **且** 禁止在档位内重新打乱顺序

### 需求: NH 入口页热门标签分档档位阈值

档位阈值必须为固定常量，禁止随标签总数动态变化，以保证视觉层级稳定。头部档位阈值必须为 5，中段档位阈值必须为 5，剩余全部归入长尾档位。当头部 + 中段标签数超过可用标签时，长尾档位可为空。

#### 场景:头部档位固定为前 5 个

- **当** 标签按 `count` 降序排列后
- **那么** 前 5 个标签必须归入「热门头部」档位
- **且** 该数量不得随标签总数变化而调整

#### 场景:中段档位固定为第 6 至第 10 个

- **当** 标签按 `count` 降序排列且总数超过 5
- **那么** 第 6 至第 10 个标签必须归入「中段」档位
- **且** 中段档位最多容纳 5 个标签

#### 场景:长尾档位为第 11 个及之后

- **当** 标签按 `count` 降序排列且总数超过 10
- **那么** 第 11 个及之后的全部标签必须归入「长尾」档位
- **且** 长尾档位无数量上限

### 需求: NH 入口页热门标签进场动画与 reduced-motion 退化

热门标签进场时必须复用项目共享的 `tagListVariants` / `tagItemVariants` stagger 动画，实现错峰淡入。当用户启用 `prefers-reduced-motion` 时，系统必须退化为无位移的纯淡入，并禁用 stagger 错峰。

#### 场景:正常偏好下标签错峰淡入

- **当** 用户未启用 `prefers-reduced-motion` 且热门标签数据加载完成
- **那么** 标签必须以 stagger 顺序错峰淡入
- **且** 每个标签必须带有轻微上移位移
- **且** stagger 间隔必须复用 `tagItemVariants` 定义（DURATION.fast，约 20ms 间隔）

#### 场景:reduced-motion 下退化为纯淡入

- **当** 用户启用 `prefers-reduced-motion` 且热门标签数据加载完成
- **那么** 标签必须以纯 opacity 淡入，无位移、无缩放
- **且** 禁止触发 stagger 错峰，所有标签同时淡入

#### 场景:标签刷新后重新触发动画

- **当** 用户点击「刷新热门标签」并成功获取新数据
- **那么** 新标签必须重新触发进场动画（错峰或退化形式取决于当前偏好）
- **且** 禁止因 AnimatePresence key 复用导致新标签无动画直接显示

### 需求: NH 入口子功能返回按钮持续可见

用户从 NH 入口页进入任一子功能（最近更新、热门排行、热门标签）后，「返回 NH 入口」按钮必须持续可见，直到用户通过显式退出动作（点击「返回 NH 入口」、切换到其他来源、点击随机）离开入口体系。在入口子功能内执行的后续操作——包括关键词搜索、翻页、排行下拉切换、标签增删——禁止隐藏该按钮。该按钮的显隐控制必须与 NH 入口页网格（`NhEntryGrid`）的渲染控制相互独立，禁止由单个状态同时承担两个职责，以避免事件处理器写入语义不一致导致的按钮误隐藏与入口页网格误重渲染。

#### 场景:进入热门排行后执行关键词搜索时按钮仍可见
- **当** 用户从 NH 入口页点击「热门排行」进入排行结果，且结果加载完成显示「返回 NH 入口」按钮
- **且** 用户在搜索框输入关键词并点击「搜索」触发关键词搜索
- **那么** 关键词搜索加载完成后，「返回 NH 入口」按钮必须仍然可见
- **且** 禁止因搜索模式为 keyword 而隐藏该按钮

#### 场景:进入最近更新后翻页时按钮仍可见
- **当** 用户从 NH 入口页点击「最近更新」进入结果列表，且结果加载完成显示「返回 NH 入口」按钮
- **且** 用户点击分页控件翻到下一页
- **那么** 翻页加载完成后，「返回 NH 入口」按钮必须仍然可见

#### 场景:在入口子功能内切换排行下拉时按钮仍可见
- **当** 用户已在 NH 热门排行结果中，「返回 NH 入口」按钮可见
- **且** 用户通过排行下拉切换到另一个排行选项
- **那么** 新排行结果加载完成后，「返回 NH 入口」按钮必须仍然可见

#### 场景:在入口子功能内增删标签时按钮仍可见
- **当** 用户已通过 NH 入口页热门标签进入标签搜索结果，「返回 NH 入口」按钮可见
- **且** 用户通过标签面板新增或移除标签触发新的标签搜索
- **那么** 新的标签搜索结果加载完成后，「返回 NH 入口」按钮必须仍然可见

#### 场景:点击返回入口页时按钮隐藏且入口页网格重现
- **当** 用户正在查看 NH 入口子功能结果，「返回 NH 入口」按钮可见
- **且** 用户点击「返回 NH 入口」按钮
- **那么** 系统必须清空当前漫画结果、分页和错误状态
- **且** 「返回 NH 入口」按钮必须隐藏
- **且** NH 入口页网格（最近更新 / 热门排行 / 热门标签）必须重新展示而不发起网络内容请求

#### 场景:切换到其他来源时按钮隐藏
- **当** 用户正在查看 NH 入口子功能结果，「返回 NH 入口」按钮可见
- **且** 用户通过来源下拉切换到非 NH 来源
- **那么** 「返回 NH 入口」按钮必须隐藏

#### 场景:点击随机时按钮隐藏
- **当** 用户正在查看 NH 入口子功能结果，「返回 NH 入口」按钮可见
- **且** 用户点击「随机」按钮（仅适用于支持随机的来源；NH 本身不支持随机，故此场景描述用户先切到支持随机的来源或本场景作为按钮隐藏语义的对称约束）
- **那么** 「返回 NH 入口」按钮必须隐藏

#### 场景:入口子功能内关键词搜索无果时不重渲染入口页网格
- **当** 用户从 NH 入口页进入热门排行或热门标签子功能，「返回 NH 入口」按钮可见
- **且** 用户进一步执行关键词搜索且无结果
- **那么** 系统必须显示「暂无搜索结果」空状态而非 NH 入口页网格
- **且** 「返回 NH 入口」按钮必须仍然可见，允许用户一键回到入口页
