## 需求

### 需求:NH 来源入口页

系统必须在 NH 搜索来源处于初始空状态时展示来源入口页，入口页必须提供最近更新、热门排行和热门标签快捷入口。

#### 场景:切换到 NH 显示入口页
- **当** 用户在搜索页选择 NH 来源且当前没有搜索结果、没有加载中状态、没有错误
- **那么** 系统必须展示 NH 入口页
- **且** 入口页必须显示「最近更新」「热门排行」两个主入口
- **且** 入口页必须显示热门标签区域或标签同步提示

#### 场景:进入最近更新
- **当** 用户点击 NH 入口页的「最近更新」入口
- **那么** 系统必须以 NH 来源请求第一页最近更新结果
- **且** 请求必须使用空关键词的普通搜索语义
- **且** 结果加载后入口页必须隐藏并展示漫画列表

#### 场景:进入热门排行
- **当** 用户点击 NH 入口页的「热门排行」入口
- **那么** 系统必须以 NH 来源请求第一页 popular 排序结果
- **且** 前端搜索上下文必须能区分热门排行与最近更新，避免分页或缓存混用
- **且** 结果加载后入口页必须隐藏并展示漫画列表

#### 场景:返回入口页
- **当** 用户正在查看 NH 最近更新、热门排行或标签快捷入口结果
- **那么** 系统必须提供返回 NH 入口页的操作
- **且** 返回后必须清空当前漫画结果、分页和错误状态

### 需求:NH 入口页热门标签快捷入口

系统必须在 NH 入口页使用 NH 标签目录数据展示热门标签快捷入口，禁止使用仅写死在前端的静态热门标签列表作为主要数据源。

#### 场景:展示已同步的热门标签
- **当** 本地标签目录中存在 NH 标签数据
- **那么** NH 入口页必须按热门排序展示前若干个标签
- **且** 每个标签必须显示标签名
- **且** 当 count 可用时必须显示数量

#### 场景:标签未同步时提示刷新
- **当** 本地标签目录中没有 NH 标签数据
- **那么** NH 入口页必须显示标签同步提示
- **且** 用户必须可以从该区域触发标签目录刷新
- **且** 标签目录刷新失败时不得影响最近更新和热门排行入口的可用性

#### 场景:点击热门标签执行精确标签搜索
- **当** 用户点击 NH 入口页中的热门标签
- **那么** 系统必须以 NH 来源执行标签搜索
- **且** 搜索语义必须限定到该标签，而不是普通关键词搜索
- **且** 结果加载后必须展示漫画列表和分页控件
