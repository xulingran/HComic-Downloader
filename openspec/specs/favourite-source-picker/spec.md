### 需求:未设置默认来源时启动后首次进入收藏夹必须引导选择来源

当 `defaultFavouriteSource` 配置为空且本次应用启动内尚未处理过收藏夹来源选择时，收藏夹 tab 在用户第一次进入时必须弹出来源选择器，禁止自动从任何来源加载收藏夹。

#### 场景:未设置默认来源且本会话首次进入

- **当** 用户未在设置中配置 `defaultFavouriteSource`（值为空字符串）且本次应用启动后首次点击收藏夹 tab
- **那么** 系统必须弹出来源选择器（仅包含 `SOURCES_WITH_FAVOURITES` 中的来源），并且禁止在弹出期间从任何来源发起 `getFavourites` 请求

#### 场景:用户在弹窗中选择来源

- **当** 用户在来源选择器中点击某个来源（如 jm）
- **那么** 系统必须关闭弹窗、将该来源设为当前来源、标记本会话已处理过选择，并以该来源加载收藏夹第一页

#### 场景:用户关闭弹窗但未选择

- **当** 用户通过 ESC 或遮罩关闭来源选择器而未选择任何来源
- **那么** 系统必须标记本会话已展示过选择器、禁止再次自动弹出，并显示空状态（无收藏列表、无加载中），但必须保留一个可重新打开选择器的入口（如「选择来源」按钮或顶部下拉框）

#### 场景:关闭弹窗后重新打开选择器

- **当** 用户在关闭来源选择器后，点击空状态中的「选择来源」按钮或在顶部下拉框中选择某个来源
- **那么** 系统必须以新选择的来源加载收藏夹，且本会话后续进入收藏夹 tab 不再自动弹出选择器

### 需求:已设置默认来源时必须直接加载且禁止弹窗

当用户在设置中配置了 `defaultFavouriteSource` 时，进入收藏夹 tab 必须直接以该来源加载收藏夹，禁止弹出来源选择器，无论本次启动是否首次进入。

#### 场景:已设置默认来源时进入收藏夹

- **当** 用户已配置 `defaultFavouriteSource`（如 bika）且点击收藏夹 tab
- **那么** 系统必须以 bika 为当前来源加载收藏夹，且禁止弹出来源选择器

#### 场景:已设置默认来源时再次进入

- **当** 用户在同一会话内多次切换进出收藏夹 tab 且已配置默认来源
- **那么** 系统每次都必须复用现有缓存优先逻辑加载默认来源，禁止任何来源选择器出现

### 需求:会话标志必须仅随应用进程生命周期存在

来源选择器的「本会话是否已处理过选择」标志必须仅存在于内存中，禁止持久化到磁盘；应用重启后该标志必须自动重置为未处理状态。

#### 场景:同一应用会话内多次进入

- **当** 用户在本次启动内已通过弹窗选择或关闭来源选择器，随后再次切换进出收藏夹 tab
- **那么** 系统必须跳过自动弹窗（本会话标志为已处理），按当前来源复用现有缓存逻辑

#### 场景:应用重启后会话标志归零

- **当** 用户关闭并重新启动应用后首次进入收藏夹 tab，且 `defaultFavouriteSource` 仍为空
- **那么** 系统必须再次弹出来源选择器（标志已随旧进程销毁而归零）

### 需求:来源选择器必须仅展示支持收藏夹的来源

来源选择器中列出的来源必须严格等于 `SOURCES_WITH_FAVOURITES`，禁止展示 `supportsFavourites` 为 false 的来源（如 copymanga）。

#### 场景:选择器来源列表

- **当** 来源选择器展示来源选项
- **那么** 选项必须恰好为 hcomic、moeimg、jm、bika 四项（即 `SOURCES_WITH_FAVOURITES`），禁止包含 copymanga

### 需求:后续操作必须延续现有逻辑

来源选择器处理完成后（无论用户选择或跳过），收藏夹 tab 内的所有后续交互（来源切换、刷新、分页、批量下载、预加载）必须完全复用现有代码路径，禁止因新增选择器而改变既有行为。

#### 场景:选择来源后切换其他来源

- **当** 用户通过来源选择器选择 jm 并加载完成后，再通过顶部下拉框切换到 bika
- **那么** 系统必须按现有下拉框切换逻辑处理（命中缓存则显示缓存，否则重新加载），且禁止再次弹出选择器

#### 场景:选择来源后的分页与刷新

- **当** 用户在来源选择器选择来源后执行翻页或点击刷新按钮
- **那么** 系统必须按现有分页与刷新逻辑处理，行为与未引入选择器时完全一致
