# list-loading-feedback 规范

## 目的
定义搜索结果列表在各类加载场景下的展示策略：何时清空旧结果展示骨架/加载态、何时保留旧结果叠加遮罩、遮罩强度如何分档，以及未提交输入时如何保持渲染稳定。覆盖新查询、翻页、缓存命中、认证校验、输入未提交等全部数据流转与中间态。
## 需求
### 需求:新查询发起时必须清空旧结果并展示加载态

当用户触发一个会产生全新结果集的查询时（切换来源、切换筛选标签、提交新搜索、从抽屉/侧栏追加标签搜索、点击随机、点击 NH/Bika 各类入口），系统**必须**在发起网络请求的同时立即清空当前列表，使旧结果不再可见，并展示明确的加载态。旧结果在请求往返期间持续显示**禁止**发生。

#### 场景:切换来源时旧结果立即消失

- **当** 用户在搜索页或收藏夹页切换到一个新来源（且新来源当前上下文无缓存）
- **那么** 当前显示的旧结果列表立即被清空
- **且** 在网络返回前展示加载态（搜索页展示骨架网格，收藏夹页展示「加载中…」文字）

#### 场景:切换筛选标签时旧结果立即消失

- **当** 用户在搜索页切换/追加/清除筛选标签触发新搜索
- **那么** 当前显示的旧结果列表立即被清空
- **且** 在网络返回前展示骨架网格

#### 场景:提交新搜索时旧结果立即消失

- **当** 用户在搜索页输入新关键词提交搜索（非翻页），或从抽屉/侧栏点击标签触发追加式搜索
- **那么** 当前显示的旧结果列表立即被清空
- **且** 在网络返回前展示骨架网格

### 需求:翻页时必须保留旧结果并叠加遮罩

当用户在同一结果集内翻页（点击分页器上一页/下一页、页码跳转）时，系统**必须**保留当前页的旧结果继续可见，并在其上叠加半透明遮罩与加载指示器，直到新页结果就绪。遮罩**必须**覆盖整个视口（`fixed inset-0`，而非仅覆盖网格容器），且遮罩内的加载指示器（spinner）**必须**位于视口正中，而**禁止**位于网格容器中心——当内容比视口高或矮时，spinner 仍**必须**在视口正中。遮罩内的居中加载指示器**必须**是一个旋转的 spinner（不确定性动画），并**必须**附带一行辅助文案；静态纯文字指示器**禁止**使用。翻页时清空已浏览内容**禁止**发生。

本需求适用于搜索页、收藏夹页与历史页三个列表页，三者的翻页遮罩与加载指示器**必须**采用一致的视觉模式（保留旧结果 + 全视口遮罩 + spinner + 辅助文案）。历史页翻页时卸载旧网格仅展示文字**禁止**发生。

#### 场景:搜索页翻页显示遮罩与 spinner

- **当** 用户在搜索页已加载某结果集的某一页后，通过分页器或页码跳转请求另一页（同一查询上下文、无缓存）
- **那么** 当前页的旧结果继续显示
- **且** 结果区上方叠加半透明遮罩，遮罩内居中显示旋转的 spinner（不确定性动画）与辅助文案
- **且** 新页结果就绪后替换旧结果、移除遮罩

#### 场景:收藏夹页翻页显示遮罩与 spinner

- **当** 用户在收藏夹页已加载某来源某一页后，通过分页器或页码跳转请求另一页（同一来源、无缓存）
- **那么** 当前页的旧结果继续显示
- **且** 结果区上方叠加半透明遮罩，遮罩内居中显示旋转的 spinner（不确定性动画）与辅助文案
- **且** 新页结果就绪后替换旧结果、移除遮罩

#### 场景:历史页翻页保留旧网格并显示遮罩与 spinner

- **当** 用户在历史页已加载某一页后，通过分页器或页码跳转请求另一页（同一历史上下文、无缓存），且当前已有旧结果列表可见
- **那么** 当前页的旧结果继续显示（不卸载旧网格）
- **且** 结果区上方叠加半透明遮罩，遮罩内居中显示旋转的 spinner（不确定性动画）与辅助文案
- **且** 新页结果就绪后替换旧结果、移除遮罩

#### 场景:历史页首次加载无旧网格时显示居中 spinner

- **当** 用户在历史页首次加载（旧结果列表为空）且处于加载中
- **那么** 显示居中的 spinner 与辅助文案（无网格可遮罩、无旧结果保留）
- **且** 不展示翻页遮罩层（因无旧结果容器）

### 需求:新查询与翻页必须按查询上下文精确区分

系统区分「新查询」与「翻页」的依据**必须**是「当前列表所属的查询上下文」与「新请求的查询上下文」是否一致，而非请求的页码参数。上下文一致即为翻页，否则即为新查询。

#### 场景:同一结果集换页判定为翻页

- **当** 当前列表属于上下文 A，用户请求上下文 A 的另一页
- **那么** 系统将其判定为翻页，保留旧结果并叠加遮罩

#### 场景:换结果集换回第 1 页判定为新查询

- **当** 当前列表属于上下文 A，用户请求上下文 B 的第 1 页（B ≠ A）
- **那么** 系统将其判定为新查询，清空旧结果并展示骨架/加载态

#### 场景:同一结果集从高页重按搜索回到第 1 页判定为新查询

- **当** 当前列表属于上下文 A 的第 3 页，用户重按搜索按钮请求上下文 C 的第 1 页（C ≠ A）
- **那么** 系统将其判定为新查询（尽管目标页为第 1 页），清空旧结果并展示骨架/加载态

### 需求:缓存命中路径不受加载态变更影响

当请求的目标页已有缓存时，系统**必须**即时显示缓存内容（不进入骨架/遮罩加载态），并照常发起后台静默刷新。本能力的加载态变更**禁止**影响缓存优先路径。

#### 场景:搜索页缓存命中即时显示

- **当** 用户在搜索页请求某上下文某页，且该页已有缓存
- **那么** 缓存内容即时显示（无骨架、无遮罩）
- **且** 系统在后台静默刷新该页，刷新结果就绪后替换

#### 场景:收藏夹页缓存命中即时显示

- **当** 用户在收藏夹页请求某来源某页，且该页已有缓存
- **那么** 缓存内容即时显示（无加载文字、无遮罩）
- **且** 系统在后台静默刷新该页，刷新结果就绪后替换

### 需求:加载遮罩必须按"旧结果是否将被替换"分两档强度

列表页（搜索页、收藏夹页、历史页）在加载期间保留旧结果并叠加遮罩时，遮罩的视觉强度**必须**根据"旧结果是否即将被整页替换"分为两档：

- **轻档**：翻页场景（旧结果仍属同一查询上下文，对用户有参考价值）。
- **重档**：整页替换场景（旧结果即将被新来源/新查询的结果取代），包括切换来源、随机、分类入口、NH 各入口、抽屉/侧栏标签追加搜索等。

两档的视觉差异**必须**足以让用户直观区分。**轻档的模糊强度必须使旧结果基本不可辨认**（旧结果仅剩色块、文字不可读），而**禁止**停留在"旧结果基本可读"的强度；重档强度**必须**明显高于轻档以保持两档视觉差异。轻档与重档**禁止**使用相同的模糊强度或背景不透明度。具体而言，轻档使用约 8px 的 backdrop-blur 与约 80% 的背景不透明度，重档使用约 16px 的 backdrop-blur 与约 92% 的背景不透明度。

#### 场景:翻页时显示轻档遮罩

- **当** 用户在列表页已加载某结果集的某一页后，通过分页器或页码跳转请求同一查询上下文的另一页（无缓存），触发 `keepExisting: true` 的加载路径
- **那么** 叠加的遮罩使用轻档视觉（约 8px backdrop-blur + 约 80% 背景不透明度），旧结果基本不可辨认
- **且** 遮罩内居中显示 spinner 与辅助文案

#### 场景:切换来源认证校验期间显示重档遮罩

- **当** 用户在搜索页切换到一个需要认证的来源（jm/bika/copymanga），当前已有旧结果，系统进入 `verifySourceAuth` 校验窗口（旧结果尚未清空、`isLoading` 为 true）
- **那么** 叠加的遮罩使用重档视觉（约 16px backdrop-blur + 约 92% 背景不透明度），旧结果几乎完全不可辨认
- **且** 遮罩内居中显示 spinner 与辅助文案

#### 场景:同来源新查询显示重档遮罩

- **当** 用户在搜索页同一来源下触发整页替换（随机、分类入口、NH 入口、抽屉/侧栏标签追加搜索），当前已有旧结果且未走骨架路径
- **那么** 叠加的遮罩使用重档视觉
- **且** 遮罩内居中显示 spinner 与辅助文案

### 需求:加载指示器必须使用 spinner 并尊重 reduced-motion

列表页加载遮罩内的居中加载指示器**必须**是一个旋转的 spinner（不确定性动画，非确定性进度环），并**必须**附带一行辅助文案（语义锚点，避免强遮罩下转动被误判为卡顿）。spinner **必须**尊重用户的 reduced-motion 偏好：当 `prefers-reduced-motion` 启用时，spinner **必须**停止转动（呈现静止环），辅助文案仍**必须**显示。纯静态文字指示器**禁止**作为加载遮罩的唯一指示器。

#### 场景:加载遮罩内显示旋转 spinner 与辅助文案

- **当** 任一列表页（搜索/收藏夹/历史）的加载遮罩处于显示状态
- **那么** 遮罩内居中显示一个旋转的 spinner（不确定性动画）
- **且** spinner 下方或旁边显示一行辅助文案（如「加载中...」）
- **且** spinner 不可为显示百分比的确定性进度环

#### 场景:reduced-motion 启用时 spinner 停转

- **当** 用户系统启用了 `prefers-reduced-motion`，且列表页加载遮罩处于显示状态
- **那么** spinner 停止转动，呈现静止环
- **且** 辅助文案照常显示

### 需求:认证来源切换的校验窗口必须视为"整页替换前奏"

当用户切换到需要认证的来源（jm/bika/copymanga），系统在发起 `verifySourceAuth` 校验时，若当前仍有旧结果显示，**必须**按整页替换场景对待遮罩强度（重档），而**禁止**按翻页（轻档）处理。校验通过后转入实际搜索/随机的加载态时，遮罩强度**必须**保持重档直至新结果就绪或旧结果被清空。

#### 场景:认证校验期间遮罩强度不降级

- **当** 用户从来源 A 切换到需认证的来源 B，A 的旧结果仍在，系统开始 `verifySourceAuth(B)`
- **那么** 校验窗口期间遮罩为重档
- **且** 校验通过后进入 `withLoading` 触发的搜索/随机加载期间，遮罩仍为重档
- **且** 新结果就绪后遮罩移除

#### 场景:认证失败转登录态时不残留重档遮罩

- **当** 用户切换到需认证来源 B 的校验窗口期间（重档遮罩显示），`verifySourceAuth` 判定无效
- **那么** 系统转入登录态（`needsLogin=true`），遮罩移除
- **且** 加载强度状态被清空，不残留为下次加载的默认值

### 需求:未提交的输入变化必须保持结果列表渲染稳定

当用户在搜索栏输入或删除文字但尚未提交搜索（未按 Enter、未点击搜索按钮、未触发任何会产生新结果集的动作）时，已加载的结果列表**必须**保持渲染稳定：结果容器的 React `key` **禁止**因实时输入值的变化而改变，已加载的卡片**禁止**发生整体卸载/重挂载，卡片进出场动画（enter/exit）**禁止**被重放。结果容器的 `key` **必须**派生自「当前已加载结果所属的查询上下文」（即最近一次成功完成搜索时记录的 query/mode/source/searchTags 组合），而**禁止**派生自实时输入值。

#### 场景:打字输入新字符时结果列表不闪烁

- **当** 用户在搜索栏已加载某查询上下文的结果列表后，连续输入若干字符修改 `query` 但未按 Enter、未点击搜索按钮
- **那么** 结果容器的 `key` 保持不变（与已加载上下文一致）
- **且** 已显示的卡片不发生卸载/重挂载
- **且** 卡片进出场动画不被重放（无闪烁）

#### 场景:删除搜索栏文字时结果列表不闪烁

- **当** 用户在搜索栏已加载某查询上下文的结果列表后，通过退格键删除若干字符修改 `query` 但未提交搜索
- **那么** 结果容器的 `key` 保持不变
- **且** 已显示的卡片不发生卸载/重挂载
- **且** 卡片进出场动画不被重放

#### 场景:提交搜索后容器 key 按预期切换

- **当** 用户在搜索栏修改 `query` 后按 Enter 或点击搜索按钮，新搜索成功完成并加载了新结果
- **那么** 结果容器的 `key` 切换为新的已加载查询上下文
- **且** 这一切换是预期的整页替换行为（与切换来源、新查询的现有契约一致）

#### 场景:首次进入页面无搜索时容器拥有稳定占位 key

- **当** 用户首次进入搜索页且尚未发起任何搜索（`loadedContextKey` 为空）
- **那么** 结果容器使用一个稳定的占位 `key`（不派生自实时输入）
- **且** 该占位 key 在用户输入未提交的 query 期间保持不变

#### 场景:翻页时容器 key 仍按页码切换

- **当** 用户在已加载某查询上下文的结果列表中翻页（同一上下文、不同页码）
- **那么** 结果容器的 `key` 因页码部分变化而切换（保留现有「翻页 remount 以规避 framer-motion 测量竞态」的设计）
- **且** 这一行为与「未提交输入不变 key」的契约不冲突（翻页是显式数据变更动作，不是实时输入）
