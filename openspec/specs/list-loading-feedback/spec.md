# list-loading-feedback 规范

### 需求:新查询发起时必须清空旧结果并展示加载态

当用户触发一个会产生全新结果集的查询时（切换来源、切换筛选标签、提交新搜索、从抽屉/侧栏追加标签搜索、点击随机、点击 NH/Bika 各类入口），系统**必须**在发起网络请求的同时立即清空当前列表，使旧结果不再可见，并展示明确的加载态。旧结果在请求往返期间持续显示**禁止**发生。

#### 场景:切换来源时旧结果立即消失

- **当** 用户在搜索页或收藏夹页切换到一个新来源（且新来源当前上下文无缓存）
- **那么** 当前显示的旧结果列表立即被清空
- **且** 在网络返回前展示加载态（搜索页展示骨架网格，收藏夹页展示「加载中…」文字）

#### 场景:切换筛选标签时旧结果立即消失

- **当** 用户在搜索页切换/追加/清除筛选标签触发新搜索
- **那么** 当前显示的旧结果列表立即被清空
- **且** 在网络返回前展示骨架网格

#### 场景:提交新搜索时旧结果立即消失

- **当** 用户在搜索页输入新关键词提交搜索（非翻页），或从抽屉/侧栏点击标签触发追加式搜索
- **那么** 当前显示的旧结果列表立即被清空
- **且** 在网络返回前展示骨架网格

### 需求:翻页时必须保留旧结果并叠加遮罩

当用户在同一结果集内翻页（点击分页器上一页/下一页、页码跳转）时，系统**必须**保留当前页的旧结果继续可见，并在结果区上方叠加半透明遮罩与加载提示，直到新页结果就绪。翻页时清空已浏览内容**禁止**发生。

#### 场景:搜索页翻页显示遮罩

- **当** 用户在搜索页已加载某结果集的某一页后，通过分页器或页码跳转请求另一页（同一查询上下文、无缓存）
- **那么** 当前页的旧结果继续显示
- **且** 结果区上方叠加半透明遮罩与「加载中…」提示
- **且** 新页结果就绪后替换旧结果、移除遮罩

#### 场景:收藏夹页翻页显示遮罩

- **当** 用户在收藏夹页已加载某来源某一页后，通过分页器或页码跳转请求另一页（同一来源、无缓存）
- **那么** 当前页的旧结果继续显示
- **且** 结果区上方叠加半透明遮罩与「加载中…」提示
- **且** 新页结果就绪后替换旧结果、移除遮罩

### 需求:新查询与翻页必须按查询上下文精确区分

系统区分「新查询」与「翻页」的依据**必须**是「当前列表所属的查询上下文」与「新请求的查询上下文」是否一致，而非请求的页码参数。上下文一致即为翻页，否则即为新查询。

#### 场景:同一结果集换页判定为翻页

- **当** 当前列表属于上下文 A，用户请求上下文 A 的另一页
- **那么** 系统将其判定为翻页，保留旧结果并叠加遮罩

#### 场景:换结果集换回第 1 页判定为新查询

- **当** 当前列表属于上下文 A，用户请求上下文 B 的第 1 页（B ≠ A）
- **那么** 系统将其判定为新查询，清空旧结果并展示骨架/加载态

#### 场景:同一结果集从高页重按搜索回到第 1 页判定为新查询

- **当** 当前列表属于上下文 A 的第 3 页，用户重按搜索按钮请求上下文 C 的第 1 页（C ≠ A）
- **那么** 系统将其判定为新查询（尽管目标页为第 1 页），清空旧结果并展示骨架/加载态

### 需求:缓存命中路径不受加载态变更影响

当请求的目标页已有缓存时，系统**必须**即时显示缓存内容（不进入骨架/遮罩加载态），并照常发起后台静默刷新。本能力的加载态变更**禁止**影响缓存优先路径。

#### 场景:搜索页缓存命中即时显示

- **当** 用户在搜索页请求某上下文某页，且该页已有缓存
- **那么** 缓存内容即时显示（无骨架、无遮罩）
- **且** 系统在后台静默刷新该页，刷新结果就绪后替换

#### 场景:收藏夹页缓存命中即时显示

- **当** 用户在收藏夹页请求某来源某页，且该页已有缓存
- **那么** 缓存内容即时显示（无加载文字、无遮罩）
- **且** 系统在后台静默刷新该页，刷新结果就绪后替换

### 需求:加载遮罩必须按"旧结果是否将被替换"分两档强度

搜索页在加载期间保留旧结果并叠加遮罩时，遮罩的视觉强度**必须**根据"旧结果是否即将被整页替换"分为两档：

- **轻档**：翻页场景（旧结果仍属同一查询上下文，对用户有参考价值）。
- **重档**：整页替换场景（旧结果即将被新来源/新查询的结果取代），包括切换来源、随机、分类入口、NH 各入口、抽屉/侧栏标签追加搜索等。

两档的视觉差异**必须**足以让用户直观区分（轻档旧结果基本可读，重档旧结果几乎不可辨认）。轻档与重档**禁止**使用相同的模糊强度或背景不透明度。

#### 场景:翻页时显示轻档遮罩

- **当** 用户在搜索页已加载某结果集的某一页后，通过分页器或页码跳转请求同一查询上下文的另一页（无缓存），触发 `withLoading(..., { keepExisting: true })`
- **那么** 叠加的遮罩使用轻档视觉（较低背景不透明度 + 较轻模糊），旧结果仍基本可读
- **且** 居中文字提示加载进行中

#### 场景:切换来源认证校验期间显示重档遮罩

- **当** 用户在搜索页切换到一个需要认证的来源（jm/bika/copymanga），当前已有旧结果，系统进入 `verifySourceAuth` 校验窗口（旧结果尚未清空、`isLoading` 为 true）
- **那么** 叠加的遮罩使用重档视觉（高背景不透明度 + 重模糊），旧结果几乎不可辨认
- **且** 居中文字保持加载语义（统一「加载中...」），强度差异由模糊与背景不透明度承担

#### 场景:同来源新查询显示重档遮罩

- **当** 用户在搜索页同一来源下触发整页替换（随机、分类入口、NH 入口、抽屉/侧栏标签追加搜索），当前已有旧结果且未走骨架路径
- **那么** 叠加的遮罩使用重档视觉
- **且** 居中文字保持加载语义（统一「加载中...」）

### 需求:认证来源切换的校验窗口必须视为"整页替换前奏"

当用户切换到需要认证的来源（jm/bika/copymanga），系统在发起 `verifySourceAuth` 校验时，若当前仍有旧结果显示，**必须**按整页替换场景对待遮罩强度（重档），而**禁止**按翻页（轻档）处理。校验通过后转入实际搜索/随机的加载态时，遮罩强度**必须**保持重档直至新结果就绪或旧结果被清空。

#### 场景:认证校验期间遮罩强度不降级

- **当** 用户从来源 A 切换到需认证的来源 B，A 的旧结果仍在，系统开始 `verifySourceAuth(B)`
- **那么** 校验窗口期间遮罩为重档
- **且** 校验通过后进入 `withLoading` 触发的搜索/随机加载期间，遮罩仍为重档
- **且** 新结果就绪后遮罩移除

#### 场景:认证失败转登录态时不残留重档遮罩

- **当** 用户切换到需认证来源 B 的校验窗口期间（重档遮罩显示），`verifySourceAuth` 判定无效
- **那么** 系统转入登录态（`needsLogin=true`），遮罩移除
- **且** 加载强度状态被清空，不残留为下次加载的默认值
