## 需求

### 需求:NH 原始标签目录同步

系统必须支持从 NH 原始标签目录页面同步标签名和标签数量，并将同步结果写入本地标签目录缓存。

#### 场景:从 popular 标签页同步标签
- **当** 用户对 NH 来源触发标签列表刷新
- **那么** 后端必须请求 `https://nhentai.net/api/v2/tags/tag` 的 popular 排序接口及后续分页
- **且** 请求必须通过 NH 解析器持有的系统代理 Session 发出
- **且** 后端必须解析官方接口返回的标签名和标签数量
- **且** 同步完成后必须将标签保存到本地标签目录缓存

#### 场景:解析精确标签数量
- **当** 官方接口返回结构化标签数据与 count 字段
- **那么** 系统必须直接使用官方 count 作为精确数量
- **且** 不得优先依赖页面简写文本或手工估算

#### 场景:同步失败不清空旧数据
- **当** NH 标签目录刷新过程中请求或解析失败
- **那么** 系统禁止在失败前清空已有 NH 标签缓存
- **且** 必须向调用方返回错误或失败状态

### 需求:NH 标签列表排序和过滤

系统必须允许 NH 标签列表按热门和 A-Z 两种方式展示，并支持现有标签面板的关键词过滤能力。

#### 场景:按热门排序读取标签
- **当** 前端请求 NH 标签列表且排序方式为 popular 或未指定排序方式
- **那么** 后端必须按标签数量降序、标签名升序返回标签
- **且** 返回结果必须包含标签名和 count

#### 场景:按 A-Z 排序读取标签
- **当** 前端请求 NH 标签列表且排序方式为 name
- **那么** 后端必须按标签名升序返回标签
- **且** 返回结果必须包含标签名和 count

#### 场景:按关键词过滤标签
- **当** 前端请求 NH 标签列表并提供关键词
- **那么** 后端必须只返回标签名包含该关键词的标签
- **且** 过滤结果仍必须遵守请求的排序方式

### 需求:NH 搜索栏标签面板

系统必须将 NH 接入搜索栏旁的标签按钮和标签选择弹窗，并在弹窗中提供热门与 A-Z 排序切换。

#### 场景:NH 来源显示标签按钮
- **当** 用户在搜索页选择 NH 来源
- **那么** 搜索栏旁必须显示标签按钮
- **且** 点击标签按钮必须打开标签选择弹窗

#### 场景:标签弹窗切换排序
- **当** 用户在标签选择弹窗中切换「热门」或「A-Z」排序
- **那么** 前端必须使用对应排序方式重新读取标签列表
- **且** 已选择的标签必须保留

#### 场景:刷新 NH 标签列表
- **当** 用户在 NH 标签弹窗中点击刷新
- **那么** 系统必须触发 NH 原始标签目录同步
- **且** 刷新完成后必须按当前排序方式重新加载标签列表

#### 场景:全量标签同步进度展示
- **当** 用户在 NH 标签弹窗中触发全量刷新
- **那么** 系统必须在同步期间展示当前页、总页数与已收集标签数
- **且** 若同步过程中出现页面失败，必须展示错误状态或失败提示
- **且** 同步完成后必须恢复为正常标签列表展示

### 需求:NH 入口页热门标签刷新语义

NH 入口页的热门标签刷新必须与搜索栏标签弹窗的全量同步逻辑分离，避免每次刷新都遍历全部标签页。

#### 场景:入口页仅刷新热门头部数据
- **当** 用户在 NH 入口页点击热门标签区域的刷新
- **那么** 系统必须仅重新获取热门靠前部分的标签数据
- **且** 不得触发完整分页的标签目录同步
- **且** 刷新完成后必须更新入口页的热门标签展示

#### 场景:标签弹窗仍保留全量同步能力
- **当** 用户在搜索栏标签弹窗中执行刷新
- **那么** 系统仍必须保留全量标签同步能力
- **且** 入口页的轻量刷新不得取代全量同步

### 需求:NH 精确标签搜索

系统必须将 NH 标签面板和入口页触发的标签搜索转换为 NH 支持的精确标签查询语法。

#### 场景:单标签搜索
- **当** 用户选择一个 NH 标签并触发搜索
- **那么** 后端必须将该标签转换为 `tag:"<标签名>"` 查询
- **且** 必须调用 NH 搜索接口获取结果

#### 场景:多标签搜索
- **当** 用户选择多个 NH 标签并触发搜索
- **那么** 后端必须将每个标签转换为独立的 `tag:"<标签名>"` 查询片段
- **且** 必须用空格拼接这些查询片段

#### 场景:标签名包含引号
- **当** NH 标签名包含双引号
- **那么** 后端构造查询时必须转义双引号
- **且** 禁止生成破坏查询结构的未转义引号

### 需求:NH 热门排行搜索语义

系统必须支持通过搜索模式表达 NH popular 排序，避免与最近更新空关键词搜索混淆。

#### 场景:ranking 模式转换为 popular 请求
- **当** 后端收到 NH 来源、ranking 模式、query 为 popular 的搜索请求
- **那么** 后端必须请求 NH popular 排序结果
- **且** 不得将 popular 当作普通关键词搜索

#### 场景:ranking 模式分页
- **当** 用户在 NH 热门排行结果中翻页
- **那么** 后续分页请求必须继续使用 popular 排序语义
- **且** 分页不得回退为最近更新结果
