# download-tab-visual-spacing 规范

## 目的
为下载管理页建立独立、响应式的页面留白与清晰的内容层级，统一页头、目录信息、任务卡和空状态的视觉节奏，同时保持既有下载操作、状态语义与动画行为不变。
## 需求
### 需求:下载管理页必须具有独立且响应式的页面留白
下载管理页必须在自身根布局中提供页面级内边距，禁止依赖外层 keep-alive 容器的 padding 副作用。常规桌面宽度下页面四周必须保留约 24px 的视觉间隔；可用内容宽度较小时，间隔可以收敛但不得小于 16px。内容区必须水平居中，并限制在约 1152px 的最大宽度内，避免超宽窗口下任务信息横跨整个页面。

#### 场景:常规桌面窗口显示下载列表
- **当** 用户在常规桌面窗口打开下载管理页
- **那么** 标题、工具栏和任务卡与可滚动区域边缘之间必须存在约 24px 的留白
- **并且** 内容区必须居中且宽度不得超过约 1152px

#### 场景:最小支持窗口显示下载列表
- **当** 用户将主窗口缩小到应用允许的最小宽度
- **那么** 下载页内容与可滚动区域边缘之间必须仍保留至少 16px 的留白
- **并且** 页面不得因新增间距产生水平滚动条

### 需求:页头必须区分主要操作与下载目录信息
下载管理页页头必须将页面标题和全局操作组织为主要工具栏，并将下载目录组织为独立的次级信息行。主要操作组必须允许换行，下载目录文本必须可截断且不得挤压“打开”按钮；禁止通过缩小到难以阅读的字号来维持单行布局。

#### 场景:下载目录路径较长
- **当** 已配置的下载目录路径超过当前信息行的可用宽度
- **那么** 路径文本必须以截断形式显示并保留完整路径提示
- **并且** “打开”按钮必须保持完整可见和可点击

#### 场景:工具栏可用宽度不足
- **当** 标题、状态筛选、暂停或恢复按钮以及刷新按钮无法在同一行舒适显示
- **那么** 操作组必须自然换行且保持至少约 8px 的控件间距
- **并且** 控件不得重叠、被裁切或溢出页面内容区

#### 场景:未配置下载目录
- **当** 配置中没有可显示的下载目录
- **那么** 页面必须省略次级目录信息行
- **并且** 标题和全局操作的布局不得留下无意义的空白占位

### 需求:下载任务内容必须使用一致的视觉间距层级
顶层任务列表中的相邻任务卡必须保留约 16px 的垂直间隔。普通任务卡与专辑卡必须使用一致的圆角、背景和基础边界表达，并提供约 20px 的内部留白；专辑卡必须继续保留 accent 左侧强调边。展开的章节子行必须使用次级背景与正向内边距，禁止标题、按钮或进度条紧贴子行边缘。

#### 场景:同时显示专辑和独立任务
- **当** 下载列表同时包含专辑卡与独立任务卡
- **那么** 两类卡片必须共享一致的基础轮廓和内部留白
- **并且** 专辑卡必须仍能通过左侧 accent 强调边与普通任务卡区分

#### 场景:展开多章节专辑
- **当** 用户展开一个包含多个章节的专辑卡
- **那么** 相邻章节子行之间必须具有明确的正向间距
- **并且** 每个章节的标题、操作按钮和进度条必须与子行边缘保持可辨识的内部留白

### 需求:空任务状态必须占据稳定的内容面板
当下载列表为空时，“暂无下载任务”必须显示在具有与任务卡一致背景、圆角和边界语义的内容面板中，并提供充分的垂直留白；禁止将空状态文案直接悬浮在页面背景上。

#### 场景:下载队列为空
- **当** 下载管理页加载完成且不存在下载任务
- **那么** 页面必须在任务内容区域显示包含“暂无下载任务”的空状态面板
- **并且** 空状态面板必须与页头保持约 24px 的垂直节奏并提供充分的内部留白

### 需求:视觉优化不得改变下载管理行为
间距和结构调整必须保持现有状态筛选、全局暂停或恢复、刷新、打开目录、单任务操作、专辑级操作、专辑折叠展开、进度展示以及任务进出场动画行为。不得修改下载状态集合、IPC 调用参数或 reduced-motion 语义。

#### 场景:用户执行既有下载操作
- **当** 用户在优化后的页面点击筛选、暂停、恢复、刷新、打开目录、重试、取消或专辑展开折叠控件
- **那么** 对应操作必须继续调用原有处理逻辑并产生与变更前一致的行为结果

#### 场景:用户启用减少动态效果
- **当** 系统的 `prefers-reduced-motion` 设置为 `reduce`
- **那么** 任务列表必须继续遵守现有 reduced-motion 规则
- **并且** 本变更不得重新引入位移或缩放动画
