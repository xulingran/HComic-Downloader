# regression-guards 规范

## 目的

定义迁移冲突、阅读器预加载、SSRF 防护、断点续传与下载状态机等高风险路径的回归守护，确保已修复缺陷不会因后续重构或平台差异再次出现。

## 需求

### 需求:migration 目标冲突必须在所有平台一致报错

针对已发现的跨平台文件覆盖 bug，系统必须建立回归测试，锁定"目标已存在时主动报错且不破坏源文件"的行为在所有平台一致。

#### 场景:同盘迁移目标已存在时报告错误

- **当** 同盘迁移，且目标位置已存在同名文件
- **那么** 迁移必须将该条目标记为 `failed`，错误信息必须包含"目标文件已存在"，且源文件内容必须保持不变

#### 场景:跨盘迁移目标已存在时报告错误

- **当** 跨盘迁移，且目标位置已存在同名文件
- **那么** 迁移必须将该条目标记为 `failed`，错误信息必须包含"目标文件已存在"，且源文件必须不被复制覆盖

#### 场景:迁移完成后整体状态正确

- **当** 一条迁移计划中有成功和失败的混合条目
- **那么** 迁移整体状态必须为 `completed`（非中止），失败条目必须出现在 `failed_items` 中，且 `on_error` 回调必须对每个失败条目触发

### 需求:阅读器自适应预加载必须保持已加载内容不丢失

针对已发现的 params 抖动导致预加载重启 bug，系统必须建立回归测试，验证"快速参数变化时已预加载的页面不会从可用集合中消失"。

#### 场景:快速连续参数变化不丢失已加载页面

- **当** 阅读器参数（如页码、章节）在短时间内快速连续变化（抖动）
- **那么** 已经成功预加载的页面必须保持可用，不得因抖动被清空

#### 场景:滚动联动不重置有效预加载状态

- **当** 用户滚动触发预加载状态派生
- **那么** 派生必须稳定，不得在 params 未实质变化时重启预加载循环

### 需求:SSRF 防护必须建立回归守护

系统必须对 `UrlValidator` 的关键安全行为建立回归测试，锁定 SSRF 拦截、DNS 校验、可信白名单绕过等行为不因重构而退化。

#### 场景:内网 IP 拦截不被重构破坏

- **当** 对 `UrlValidator` 进行任何重构或修改
- **那么** 回归测试必须确保所有内网与保留 IP（IPv4/IPv6）仍被拦截，可信 CDN 白名单仍跳过 DNS 解析

### 需求:断点续传数据完整性必须建立回归守护

系统必须对下载中断恢复的数据完整性建立回归测试，锁定断点分片与最终拼装不损坏已有数据的正确行为。

#### 场景:中断恢复后文件完整可校验

- **当** 回归测试覆盖下载中断恢复场景
- **那么** 恢复后生成的最终文件必须内容完整、可正常校验，且与一次性下载结果一致

### 需求:下载状态机并发不变量必须建立回归守护

系统必须对下载状态机的并发安全建立回归测试，锁定竞态条件下任务不丢失、状态不回滚的不变量。

#### 场景:并发操作后任务集合守恒

- **当** 多个线程并发执行 add/cancel/pause 操作
- **那么** 回归测试必须确保任务集合守恒（无丢失、无重复 ID），已完成任务状态不被回滚
