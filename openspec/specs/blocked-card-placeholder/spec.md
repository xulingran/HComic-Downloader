# blocked-card-placeholder 规范

## 目的
定义搜索结果中被 tag 黑名单命中的漫画卡片（`BlockedPlaceholder`）的占位呈现规范。覆盖封面（cover/网格）模式下屏蔽占位符必须与正常 `CoverCard` 保持网格行高度对齐的约束，以及屏蔽态的简化视觉契约（line-through 标题、屏蔽图标、不展示作者/页数等元信息）。

## 需求

### 需求:封面模式屏蔽占位符必须与正常卡片保持网格行高度对齐

在搜索结果封面（cover/网格）视图下，被 tag 黑名单命中而渲染为屏蔽占位符（`BlockedPlaceholder`）的卡片，其内容区的垂直高度结构**必须**与正常 `CoverCard` 对齐，确保同一网格行内屏蔽卡片与正常卡片底部对齐、行高一致。占位符**禁止**因缺少作者栏、缺少标题最小高度、或更大的内容区 padding 而矮于正常卡片。

#### 场景:屏蔽占位符内容区 padding 必须与正常卡片一致

- **当** 封面模式下渲染一张屏蔽占位符卡片
- **那么** 其标题所在内容区容器的 padding **必须**为 `p-2`，与正常 `CoverCard` 的内容区 padding 一致
- **并且** **禁止**使用 `p-3` 或其他更大的 padding

#### 场景:屏蔽占位符标题必须保留与正常卡片一致的最小高度

- **当** 封面模式下渲染一张屏蔽占位符卡片的标题
- **那么** 标题容器**必须**应用 `min-h-[2.5rem]` 最小高度，与正常 `CoverCard` 的标题最小高度一致
- **并且** **禁止**省略该最小高度导致单行标题卡片比双行标题卡片更矮

#### 场景:屏蔽占位符必须保留作者占位行高度

- **当** 封面模式下渲染一张屏蔽占位符卡片
- **那么** 其标题下方**必须**存在一个与正常 `CoverCard` 作者栏等高的占位（高度 `h-4`、上边距 `mt-0.5`）
- **并且** 该占位行**禁止**渲染任何作者文字（屏蔽卡片是简化占位符，不展示作者信息）

### 需求:屏蔽占位符必须保留简化视觉语义

屏蔽占位符作为"被过滤内容"的视觉占位，**必须**用统一的简化样式传达屏蔽语义：标题使用删除线（`line-through`）、封面区显示屏蔽图标与「已屏蔽」提示、整体降低不透明度。屏蔽占位符**禁止**展示作者、页数、章节数等正常卡片才有的元信息，**禁止**渲染可下载按钮。

#### 场景:屏蔽占位符标题显示删除线

- **当** 渲染屏蔽占位符（封面或详细任一模式）
- **那么** 标题**必须**应用 `line-through` 删除线样式
- **并且** **禁止**展示作者、页数、章节等元信息文本

#### 场景:屏蔽占位符点击标题打开详情抽屉

- **当** 用户点击屏蔽占位符的标题
- **那么** **必须**阻止事件冒泡并打开该漫画的详情抽屉（`openDrawer(comic)`），让用户仍可查看被屏蔽项的完整信息
