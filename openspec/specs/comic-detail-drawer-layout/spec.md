## 需求
### 需求:漫画详情抽屉必须使用紧凑摘要布局

搜索页、收藏夹页和历史页共用的漫画详情抽屉必须以紧凑摘要形式展示封面、标题、作者和核心元数据，禁止继续使用最高占据 40% 视口高度的全宽封面把其余信息推到首屏之外。抽屉可以适度增加宽度，但在窄窗口下必须保持不超出视口。

#### 场景:有封面的漫画使用双列摘要

- **当** 用户打开一部包含封面的漫画详情抽屉
- **那么** 封面必须以紧凑缩略图显示在摘要区一侧，标题、作者、来源、页数、章节数、更新时间、语言和分类等已有字段必须在另一侧或其紧邻区域呈现
- **且** 标题过长时必须限制摘要区高度，同时保留访问完整标题的方式

#### 场景:无封面的漫画仍完整利用摘要宽度

- **当** 用户打开一部没有封面的漫画详情抽屉
- **那么** 标题和元数据必须占用完整摘要宽度，禁止为缺失封面保留空白列

### 需求:核心信息和主要操作必须优先保持在窗口高度内

抽屉必须将标题、作者、来源、页数或章节等核心信息以及收藏和阅读操作置于高优先级区域。在默认 800px 高窗口中，常规长度内容必须无需滚动即可看到上述信息、关联信息摘要、折叠标签摘要和主要操作；在最小 600px 高窗口中，主要操作必须始终可见，其余内容只能在独立内容区内产生最小必要滚动。

#### 场景:默认高度展示常规详情

- **当** 应用窗口高度为 800px 且漫画标题、关联信息和标签数量处于常规范围
- **那么** 用户必须无需滚动即可看到封面摘要、核心元数据、折叠标签摘要以及全部主要操作

#### 场景:最小高度保持操作可达

- **当** 应用窗口高度为 600px
- **那么** 抽屉必须启用更紧凑的封面、间距或排布
- **且** 收藏、开始阅读、继续阅读和从头开始等当前适用的主要操作必须保持在视口内
- **且** 只有中间详情内容区允许纵向滚动，顶部关闭栏和底部操作区禁止随内容滚出视口

### 需求:关联元数据必须紧凑展示并保留交互

作者、分类、原著、角色、制作组和标签等信息必须减少重复标题与纵向留白，但禁止移除已有搜索语义、来源能力门控、推荐/屏蔽状态或原网页入口。

#### 场景:点击紧凑关联信息继续搜索

- **当** 用户点击作者、分类、原著、角色、制作组或标签中的可搜索项
- **那么** 系统必须继续使用该字段原有的搜索模式和替换/追加语义发起搜索

#### 场景:来源不支持某项能力

- **当** 当前漫画来源不支持收藏或标签推荐
- **那么** 抽屉必须继续隐藏或降级对应操作，禁止因新布局暴露无法生效的入口

### 需求:大量标签必须渐进展开

drawer 表面的标签列表必须默认仅渲染一个固定上限内的标签，并对剩余数量提供明确的展开入口。展开和收起必须可由键盘操作并具备可识别的按钮名称；reader 表面禁止套用 drawer 的标签折叠限制。

#### 场景:标签数量未超过折叠上限

- **当** 漫画标签数量不超过折叠上限
- **那么** drawer 必须显示全部标签且禁止渲染无意义的展开按钮

#### 场景:标签数量超过折叠上限

- **当** 漫画标签数量超过折叠上限
- **那么** drawer 初始必须只显示上限内的标签，并显示包含剩余数量的“+N 展开”按钮
- **当** 用户激活该按钮
- **那么** drawer 必须显示完整标签列表并提供“收起”按钮

#### 场景:切换漫画后恢复折叠态

- **当** 用户在展开标签后关闭抽屉并打开另一部漫画，或抽屉切换到另一漫画
- **那么** 新漫画的标签列表必须恢复默认折叠态，禁止继承上一漫画的展开状态

#### 场景:阅读器尾页保持完整标签

- **当** 同一详情内容以 `surface="reader"` 显示在在线阅读器尾页
- **那么** 必须继续展示完整标签列表且不得出现 drawer 专属的“+N 展开”或“收起”控件

### 需求:详情补全状态必须在紧凑布局中保持可见

标签详情补全的加载、失败、重试和成功状态必须继续在标签区域显示，禁止因折叠或固定操作区隐藏错误反馈。

#### 场景:标签补全失败

- **当** 详情补全请求抛出异常或返回空漫画
- **那么** 抽屉必须显示标签加载失败反馈和可操作的重试入口

#### 场景:标签补全成功

- **当** 重试或首次详情补全成功返回标签
- **那么** 标签区域必须按当前 drawer 折叠规则显示返回的标签，并移除失败反馈

### 需求:抽屉遮罩必须阻止点击穿透

漫画详情抽屉打开时，覆盖窗口的根层和遮罩层必须接管指针事件。用户点击抽屉面板之外的遮罩空白区域时必须关闭抽屉，禁止把同一次点击穿透或转发给背景页面控件；抽屉关闭后，已失活的根层必须停止拦截页面交互。

#### 场景:点击遮罩关闭且不触发背景控件

- **当** 漫画详情抽屉处于打开状态，且遮罩下方存在可点击的页面控件
- **当** 用户点击抽屉面板之外的遮罩空白区域
- **那么** 系统必须调用一次抽屉关闭操作
- **且** 背景页面控件禁止收到该次点击

#### 场景:关闭后页面恢复交互

- **当** 漫画详情抽屉已关闭并结束活动状态
- **那么** 抽屉根层必须停止接管指针事件，背景页面控件必须可以正常交互
