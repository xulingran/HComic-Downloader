# jmcomic-id-search 规范

## 目的

定义 JM 关键词模式对纯数字专辑 ID 的优先解析和普通搜索回退行为，并规范详情页兜底识别，确保直接编号查询既高效又不会破坏既有搜索能力。

## 需求

### 需求:jm keyword 模式必须支持纯数字专辑 ID 搜索
当用户在 keyword 模式下输入纯数字字符串时，系统必须按 jm 专辑 ID 处理，优先请求详情页并返回单条结果；若失败则必须 fallback 到普通关键词搜索。

#### 场景:ID 存在并返回单条结果
- **当** 用户输入 keyword 为 `430371`
- **那么** 系统请求 `https://{domain}/album/430371`
- **并且** 解析成功后返回仅包含该漫画的列表
- **并且** 分页信息的 `currentPage` 为 1、`totalPages` 为 1、`totalItems` 为 1

#### 场景:ID 不存在时 fallback 到普通搜索
- **当** 用户输入 keyword 为纯数字但详情页请求失败或返回 None
- **那么** 系统必须继续请求 `/search/photos?search_query={keyword}`
- **并且** 按普通关键词搜索返回结果

### 需求:jm 搜索结果解析必须支持详情页兜底识别
当搜索响应实际上是专辑详情页时，系统必须识别并解析为单条结果，而不是返回空列表。

#### 场景:搜索响应为详情页
- **当** 搜索请求返回的 HTML 同时包含 `album_photo_cover` 与 `var aid = {数字};`
- **那么** 系统提取 `aid` 作为 comic_id
- **并且** 调用详情页解析逻辑返回单条漫画

#### 场景:搜索响应为普通列表页
- **当** 搜索请求返回的 HTML 不包含详情页特征
- **那么** 系统仍按原有 `thumb-overlay` 卡片列表解析

无。
