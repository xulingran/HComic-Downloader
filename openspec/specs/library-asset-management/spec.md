# library-asset-management 规范

## 目的

定义漫画库资产的安全管理操作：在系统文件管理器中定位、单项健康检查、移入系统回收站、安全重命名、原子编辑受支持元数据，以及索引与下载历史关联同步。所有操作必须限定在当前漫画库根目录内，受活动占用保护，并以资产 ID 为唯一入口，禁止渲染进程直接操作任意绝对路径。

## 需求

### 需求:在系统文件管理器中定位漫画资产

系统必须允许用户在系统文件管理器中定位选定资产。定位前必须通过资产 ID 解析当前路径并验证其仍位于漫画库根目录内；文件资产应选中文件，目录资产应打开或选中该目录。

#### 场景:定位 CBZ 文件
- **当** 用户对仍存在的 CBZ 点击“在文件管理器中显示”
- **那么** 系统打开原生文件管理器并定位该 CBZ，渲染进程不直接接收可执行文件操作权限

### 需求:对单个漫画资产运行健康检查

系统必须允许从漫画详情或卡片对单个资产运行健康检查，并复用现有健康检查的压缩包完整性、图片可读性和页数对账逻辑，禁止复制一套语义不同的检查器。

#### 场景:检查未关联下载历史的 CBZ
- **当** 用户检查一个没有 `download_history` 记录的 CBZ
- **那么** 系统仍检查压缩包和图片可读性，并在缺少可靠期望页数时跳过期望页数对账而不误报

### 需求:删除操作必须移入系统回收站

用户删除漫画资产时，系统必须显示包含标题、路径类型和大小的确认信息，并通过系统回收站 API 移走资产；禁止默认永久删除。只有系统确认回收站操作成功后，系统才可移除漫画库索引并清理精确匹配该资产的下载历史关联。

#### 场景:成功删除漫画
- **当** 用户确认删除且系统回收站操作成功
- **那么** 资产从漫画库索引和列表移除，对应精确路径的下载历史关联被清理或标记失效，并返回释放空间大小

#### 场景:回收站操作失败
- **当** 系统因权限、文件占用或平台错误无法移入回收站
- **那么** 系统保留文件、索引和下载历史不变，并向用户显示失败原因

### 需求:破坏性操作必须受活动占用保护

正在下载、打包、迁移、扫描写入、健康检查或本地阅读中持有写冲突的资产，必须禁止删除、重命名和元数据编辑。执行操作前必须重新检查占用状态，禁止只依赖打开确认框时的旧快照。

#### 场景:确认期间资产开始被占用
- **当** 用户打开删除确认框后对应资产开始被打包，并随后确认删除
- **那么** 后端在执行前重新检查并拒绝删除，禁止移动正在写入的文件

### 需求:安全重命名漫画资产

系统必须允许用户修改资产显示名称并选择同步重命名磁盘文件。新名称必须经过既有文件名清理和保留名校验；目标路径已存在时必须拒绝覆盖。成功后必须原子更新索引、下载历史输出路径和阅读进度关联，资产 ID 保持不变。

#### 场景:重命名 CBZ
- **当** 用户将某 CBZ 重命名为一个合法且无冲突的新名称
- **那么** 文件在同一漫画库根目录内完成重命名，索引和下载历史指向新路径，阅读进度继续关联原资产 ID

#### 场景:目标名称已存在
- **当** 新名称解析到的目标路径已存在
- **那么** 系统拒绝重命名，源文件及所有数据库记录保持不变

### 需求:以原子方式编辑受支持元数据

系统必须允许编辑标题、作者、标签和其他已支持的 `ComicInfo.xml` 字段。CBZ 编辑必须写入同目录临时压缩包、保留全部原条目并通过完整性校验后原子替换；ZIP 和文件夹无法安全持久化的字段必须仅作为索引覆盖值保存并明确标识，禁止伪称已写入文件。

#### 场景:成功编辑 CBZ 元数据
- **当** 用户修改 CBZ 标题和标签并确认
- **那么** 系统生成包含新 `ComicInfo.xml` 和全部原图片的有效临时 CBZ，校验成功后原子替换原文件并刷新索引版本

#### 场景:重写 CBZ 中途失败
- **当** 写临时文件、复制条目或完整性校验失败
- **那么** 原 CBZ 字节保持不变，临时文件被清理，索引继续显示原元数据

### 需求:所有管理操作必须限制在当前漫画库根目录

定位、健康检查、删除、重命名和元数据编辑必须只接受资产 ID，并在操作时解析真实路径、验证根目录包含关系和拒绝符号链接逃逸。切换下载目录后，旧根目录资产 ID 禁止继续执行文件操作。

#### 场景:使用旧漫画库资产 ID
- **当** 下载目录已迁移或切换后，渲染进程提交旧根目录中的资产 ID 请求删除
- **那么** 后端拒绝操作并要求刷新漫画库，旧目录文件不被修改
