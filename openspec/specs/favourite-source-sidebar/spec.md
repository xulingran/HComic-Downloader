# favourite-source-sidebar 规范

## 目的

定义收藏夹页面常驻来源侧边栏的来源范围、选中语义、切换行为、异步结果隔离、响应式布局与键盘可访问性。
## 需求
### 需求:收藏夹页面必须提供常驻来源侧边栏

收藏夹页面必须在内容区左侧显示常驻来源侧边栏，并禁止继续渲染标题旁的来源下拉框。侧边栏必须在页面滚动时保持可见，以便用户无需回到页面顶部即可切换来源。

#### 场景:进入已选择来源的收藏夹页面

- **当** 收藏夹页面已有当前来源
- **那么** 页面左侧必须显示来源侧边栏，右侧显示该来源的标题、操作区与收藏内容
- **且** 标题旁禁止显示来源 `<select>` 下拉框

#### 场景:滚动收藏夹内容

- **当** 用户向下滚动多页收藏内容
- **那么** 来源导航必须通过 sticky 布局保持在可操作位置

### 需求:侧边栏必须只展示支持收藏夹的来源

来源侧边栏中的按钮必须严格由 `SOURCES_WITH_FAVOURITES` 派生，禁止展示任何 `supportsFavourites=false` 的来源。

#### 场景:渲染来源列表

- **当** 收藏夹来源侧边栏渲染
- **那么** 必须恰好显示 HComic、MoeImg、JM、哔咔四个来源
- **且** 禁止显示拷贝漫画或 NH

### 需求:侧边栏必须明确反映当前选择状态

当前来源按钮必须使用 accent 选中样式并通过 `aria-current="page"` 暴露语义；未选择任何来源时必须禁止高亮由内部默认值产生的伪选中项。

#### 场景:当前来源已确定

- **当** 当前收藏来源为 JM
- **那么** JM 按钮必须显示选中样式并具有 `aria-current="page"`
- **且** 其他来源按钮必须保持未选中状态

#### 场景:首次来源弹窗被跳过

- **当** 用户关闭首次来源选择弹窗且未选择来源
- **那么** 侧边栏四个来源按钮必须全部保持未选中状态
- **且** 禁止因为 store 的初始来源值而高亮 HComic

### 需求:点击来源必须通过统一路径切换到目标来源第一页

用户点击不同来源时，系统必须把该来源设为当前查看来源并加载其第一页。若目标来源第一页已有缓存，必须立即显示缓存并按既有缓存刷新路径后台更新；若无缓存，必须显示既有加载反馈并发起第一页请求。侧栏切换必须禁止修改持久化的 `defaultFavouriteSource`。

#### 场景:切换到无缓存来源

- **当** 用户正在查看 HComic 并点击尚无缓存的 JM
- **那么** 系统必须将 JM 标记为当前来源并请求 JM 第 1 页
- **且** 必须按既有逻辑展示加载反馈

#### 场景:切换到已有第一页缓存的来源

- **当** 用户点击一个已有第 1 页缓存的来源
- **那么** 系统必须立即显示该来源第 1 页缓存
- **且** 必须复用既有后台刷新逻辑更新缓存内容

#### 场景:重复点击当前来源

- **当** 用户点击已经处于选中状态的来源按钮
- **那么** 系统必须保持当前内容
- **且** 禁止为该点击重复发起收藏夹请求

#### 场景:侧栏切换不改变默认来源

- **当** 用户通过侧栏从默认来源 HComic 切换到 JM
- **那么** 当前查看来源必须变为 JM
- **且** 持久化配置 `defaultFavouriteSource` 必须仍为 HComic

### 需求:快速连续切换必须隔离迟到请求

系统必须区分不同来源的用户加载上下文。旧来源请求晚于新来源返回时，可以写入旧来源自己的缓存，但禁止覆盖当前来源的可见内容、分页、登录状态或下载状态。

#### 场景:旧来源第一页迟到返回

- **当** 用户先请求 HComic 第 1 页，随后切换到 JM，且 JM 请求先完成
- **那么** 页面必须保持显示 JM 内容并高亮 JM
- **且** HComic 的迟到结果禁止覆盖当前可见内容

#### 场景:旧来源迟到结果写入隔离缓存

- **当** 已失去活动上下文的 HComic 请求成功返回
- **那么** 系统可以把结果写入 HComic 对应缓存
- **且** 禁止把该结果写入 JM 缓存

### 需求:侧边栏与内容区必须在双侧栏布局下保持可用

来源侧边栏必须使用固定窄宽度，收藏内容区必须使用 `min-width: 0` 吸收剩余空间。卡片网格在中等视口下必须减少列数，顶部操作区必须允许换行，禁止因全局侧边栏和来源侧边栏同时展开而产生水平溢出或不可操作的过窄卡片。

#### 场景:全局侧边栏展开且处于中等窗口

- **当** 全局侧边栏处于展开态且收藏夹页面宽度不足以舒适容纳五列卡片
- **那么** 收藏夹网格必须降为至多四列
- **且** 顶部刷新、批量操作和分页控件必须能够自然换行而不重叠

#### 场景:宽窗口显示收藏夹

- **当** 收藏内容区达到宽屏断点
- **那么** 卡片模式必须允许恢复为五列
- **且** 来源侧边栏宽度必须保持稳定

### 需求:来源按钮必须支持键盘和可见焦点操作

每个来源入口必须使用原生按钮或等价的可访问控件，必须可通过 Tab 聚焦并通过 Enter 或 Space 激活，且必须提供可见焦点样式。

#### 场景:仅使用键盘切换来源

- **当** 用户通过 Tab 聚焦 MoeImg 来源按钮并按 Enter
- **那么** 系统必须执行与鼠标点击相同的来源切换行为
- **且** 焦点位置必须保持可见

### 需求:来源侧边栏垂直位置必须独立于右侧内容高度

来源侧边栏的垂直位置必须独立于右侧主内容区的高度。当右侧主内容区因加载收藏数据、翻页或切换来源而高度发生变化时，左侧来源导航的初始垂直位置必须保持不变。侧边栏容器必须禁止被 flex 容器的默认 `align-items: stretch` 拉伸到与右侧等高，必须按自身内容高度收缩，使 sticky 导航的吸附基准不受右侧内容影响。

#### 场景:右侧加载出漫画数据后侧边栏位置不变

- **当** 收藏夹页面当前来源为任意来源（HComic、MoeImg、JM 或哔咔）且右侧主内容区从空状态变为显示漫画卡片
- **那么** 左侧来源导航的初始垂直位置必须与右侧为空时完全一致
- **且** 禁止因右侧内容变高而出现来源列表整体下移

#### 场景:右侧内容高度变化时侧边栏不跳动

- **当** 用户在收藏夹页面切换来源、翻页或刷新导致右侧主内容区高度增减
- **那么** 左侧来源导航的垂直位置必须保持稳定
- **且** 仅在用户主动滚动容器时按 `top-6` 吸附，禁止因内容高度变化自动位移

#### 场景:侧边栏滚动吸附行为保持不变

- **当** 用户向下滚动收藏夹内容
- **那么** 来源导航必须仍通过 sticky 布局吸附在滚动容器顶部下方 `top-6` 处
- **且** 修复不得改变既有的滚动吸附位置与可见性

### 需求:来源侧边栏顶部间距必须与设置页、工具箱页一致

收藏夹来源侧边栏在未滚动状态下的顶部位置必须与设置页、工具箱页的同类左侧导航保持一致，禁止出现收藏夹侧边栏紧贴内容区顶部而其他页面侧边栏留有间距的不一致。侧边栏容器必须通过显式顶部内边距保证来源导航顶部距内容区顶部至少 24px，该间距不得依赖 flex stretch 与 sticky 的副作用交互——当侧边栏容器因 `self-start` 收缩到内容高度而使 sticky 不再产生初始偏移时，必须由显式内边距补足。

#### 场景:未滚动时收藏夹侧边栏顶部与设置页对齐

- **当** 收藏夹页面已选择来源且未发生滚动
- **那么** 左侧来源导航顶部距内容区顶部必须为 24px
- **且** 必须与设置页、工具箱页侧边栏导航的初始顶部位置一致

#### 场景:来源导航仍低于主内容标题

- **当** 收藏夹页面渲染标题与来源侧边栏
- **那么** 来源导航顶部必须位于主内容标题基线之下
- **且** 与设置页侧边栏导航相对其标题的垂直关系保持一致

#### 场景:滚动吸附行为不受顶部内边距影响

- **当** 用户向下滚动收藏夹内容
- **那么** 来源导航必须仍通过 sticky 布局吸附在滚动容器顶部下方 `top-6` 处
- **且** 顶部内边距不得改变既有的滚动吸附位置与可见性
