# download-album-collapse

## 目的

下载管理界面多章专辑的章节折叠展示行为。
## 需求
### 需求:多章专辑默认折叠章节子行

下载管理界面在渲染多章专辑卡时，**必须**默认隐藏该专辑下的章节子行（独立的标题、章节级进度条、章节级控制按钮），只显示专辑标题、完成进度文案、专辑级进度条和专辑级控制按钮。折叠状态**必须**仅存在于前端会话内，**禁止**持久化到磁盘或配置。

#### 场景:首次加载多章专辑
- **当** 用户打开下载管理界面且存在一个无失败章节的多章专辑
- **那么** 该专辑卡只显示专辑级信息（标题、`N/M 章完成`、专辑级进度条、专辑级按钮），章节子行不可见

#### 场景:折叠态显示展开提示
- **当** 专辑处于折叠状态
- **那么** 专辑卡底部显示形如 `▶ 展开查看 N 章` 的提示，其中 N 为该专辑的章节数

#### 场景:刷新页面后状态重置
- **当** 用户刷新页面或重新打开下载管理界面
- **那么** 所有专辑的折叠状态丢失，按默认规则重新初始化

### 需求:用户可手动切换专辑折叠状态

用户**必须**能够通过点击切换专辑的折叠 / 展开状态。切换区域**必须**包括专辑卡头部（标题行）的非按钮区域，**禁止**因点击专辑级控制按钮（全部暂停 / 全部恢复 / 重试失败 / 全部取消 / 强制打包）而触发折叠切换。

#### 场景:点击头部空白处展开
- **当** 专辑处于折叠状态，用户点击专辑头部标题区域（非按钮部分）
- **那么** 专辑展开，章节子行变为可见，提示文案变为形如 `▼ 隐藏章节`

#### 场景:点击专辑级按钮不触发折叠
- **当** 用户点击专辑级控制按钮（如"全部暂停"、"强制打包"）
- **那么** 折叠状态保持不变，且对应按钮的专辑级操作正常执行

### 需求:专辑存在失败章节时自动展开

当某专辑出现失败章节（任一章节 `status === 'failed'`）时，系统**必须**自动展开该专辑，使失败章节对用户可见。该自动展开行为**必须**在失败从无到有产生时触发；当用户已显式折叠该专辑后再次出现失败时，**必须**再次自动展开以避免用户错过可重试的章节。

#### 场景:章节下载失败触发自动展开
- **当** 一个处于折叠状态的专辑的某章节状态变为 `failed`
- **那么** 该专辑自动展开，失败章节可见

#### 场景:失败专辑中失败章节可见性
- **当** 专辑包含至少一个失败章节
- **那么** 展开后该专辑的失败章节**必须**对用户可见，并显示章节级重试按钮

### 需求:展开态章节按失败优先排序

专辑展开渲染章节子行时，**必须**将 `status === 'failed'` 的章节排在所有非失败章节之前；失败章节之间、非失败章节之间**必须**保持原有的相对顺序（稳定排序）。

#### 场景:展开时失败章节置顶
- **当** 用户展开一个包含第 1、2、3 章的专辑，其中第 3 章失败、其余完成
- **那么** 渲染顺序为第 3 章（失败）、第 1 章、第 2 章

### 需求:用户显式操作优先于自动折叠

用户手动展开或折叠某专辑后，系统**禁止**因后续状态变化（如失败恢复为正常）自动反转该用户的折叠操作。失败从无到有产生时仍可自动展开（见"专辑存在失败章节时自动展开"需求）。

#### 场景:失败恢复后不自动折叠
- **当** 用户已手动展开某专辑，随后该专辑失败章节全部重试成功
- **那么** 专辑保持展开状态，不自动折叠

#### 场景:用户折叠后保持折叠
- **当** 用户手动折叠一个无失败的专辑，随后该专辑章节进度推进或完成
- **那么** 专辑保持折叠状态

### 需求:专辑命名弹窗默认名基于选中漫画的共有字段

当用户在搜索页或收藏页多选漫画并触发"下载为专辑"时，专辑命名弹窗（`AlbumNameDialog`）输入框的默认值**必须**基于当前选中漫画的标题计算得出，**禁止**使用与选中内容无关的写死文案作为唯一默认值。计算逻辑**必须**调用 `extractAlbumTitle(选中漫画标题列表)`；当该函数返回 `null`（提取失败或结果过短）时，**必须**回退到文案 `批量下载 - ${N}本漫画`，其中 N **必须**等于当前实际选中的漫画数量。默认值仅作为输入框预填建议，用户**必须**能够修改。

#### 场景:选中同作品多章节时预填作品名
- **当** 用户选中三本标题为 `[作者A] 某作品 第1话`、`[作者A] 某作品 第2话`、`[作者A] 某作品 第3话` 的漫画，点击"下载为专辑"
- **那么** 弹窗输入框预填 `[作者A] 某作品`，用户可在此基础上修改后确认

#### 场景:选中标题无共有部分时回退批量下载文案
- **当** 用户选中标题完全无共有的多本漫画（`extractAlbumTitle` 返回 `null`），点击"下载为专辑"
- **那么** 弹窗输入框预填 `批量下载 - ${N}本漫画`，N 为实际选中数量

#### 场景:回退文案的 N 反映真实选中数量
- **当** 用户选中 5 本漫画触发"下载为专辑"，且 `extractAlbumTitle` 返回 `null`
- **那么** 弹窗输入框预填 `批量下载 - 5本漫画`（**禁止**显示 `批量下载 - 0本漫画`）

### 需求:默认名在打开弹窗时计算而非预缓存

默认专辑名**必须**在用户点击"下载为专辑"按钮、即将打开命名弹窗时计算，**禁止**用 `useMemo` 基于选中状态预缓存。计算结果**必须**存入组件 state 供弹窗读取。这一时机保证：(1) 日志在"即将展示给用户"的诊断点输出；(2) `selectedComics()` 打开瞬间求值能拿到跨页缓存的最新数据。

#### 场景:点击按钮时计算并打开弹窗
- **当** 用户选中若干漫画后点击"下载为专辑"按钮
- **那么** 系统在打开弹窗的同一 handler 中调用 `pickAlbumDefaultName` 计算默认名、写入 state、再打开弹窗

#### 场景:勾选后异步翻页再打开仍取最新数据
- **当** 用户勾选漫画后触发异步翻页（导致 `selectedComics()` 缓存内容变化），随后点击"下载为专辑"
- **那么** 默认名基于打开瞬间的最新 `selectedComics()` 计算，**禁止**用过时的预缓存值

### 需求:默认名计算过程输出诊断日志

`pickAlbumDefaultName` 在计算默认名时**必须**通过 `console.info` 输出日志，日志**必须**包含 `inputCount`（输入标题数）、`sample`（前 3 个标题）、以及 `result`（提取成功时）或 `fallback`（回退文案）。日志**必须**使用 `[album-title]` 前缀，遵循项目既有的 `[模块前缀]` console 范式。日志级别**必须**为 `info`（**禁止**用 `debug`，因 Chrome DevTools 默认隐藏 verbose/debug 级别）。

#### 场景:提取成功时输出结果日志
- **当** `pickAlbumDefaultName` 成功从标题提取出共有字段
- **那么** console 输出 `[album-title] 提取共有字段` 及 `{ inputCount, sample, result }`

#### 场景:提取失败时输出回退日志
- **当** `extractAlbumTitle` 返回 null，`pickAlbumDefaultName` 回退到计数文案
- **那么** console 输出 `[album-title] 提取失败，回退计数文案` 及 `{ inputCount, sample, fallback }`

### 需求:专辑命名弹窗在每次打开时同步最新默认名


`AlbumNameDialog` 组件**必须**在其 `isOpen` prop 从 `false` 变为 `true` 的时刻，将内部输入框的值重置为当前传入的 `defaultName` prop。同步**必须**通过"渲染期间检测 prop 变化"模式实现（用 `wasOpen` state 记录前值，在渲染体中检测翻转并调用 `setName`），**禁止**用 `useEffect` + `setName`（会触发 ESLint `react-hooks/set-state-in-effect` 规则）。该重置**必须**仅在弹窗打开瞬间触发一次，**禁止**在弹窗保持打开期间因父组件重渲染或 `defaultName` 引用变化而覆盖用户已编辑的内容。这一行为对条件挂载（每次打开都是新组件实例）和常驻挂载（靠 `isOpen` 控制显隐）两种使用方式**必须**同时成立。

#### 场景:常驻挂载的弹窗多次打开显示最新默认名
- **当** `AlbumNameDialog` 以常驻挂载方式使用，用户第一次打开时选中 2 本（默认名为 `作品A`），关闭后选中 3 本再次打开（默认名应为 `作品B`）
- **那么** 第二次打开时输入框显示 `作品B`，**禁止**残留第一次的 `作品A`

#### 场景:用户编辑期间不被默认名覆盖
- **当** 弹窗已打开，用户在输入框里把默认的 `[作者A] 某作品` 改成了 `我的合集`，此时父组件因某种原因重渲染导致 `defaultName` 引用变化
- **那么** 输入框**必须**保持显示 `我的合集`，**禁止**被重置回 `[作者A] 某作品`

#### 场景:首次打开即预填准确的非零数量回退文案
- **当** 用户选中 4 本无共有标题的漫画，首次打开常驻挂载的 `AlbumNameDialog`
- **那么** 输入框首次显示即为 `批量下载 - 4本漫画`，**禁止**显示 `批量下载 - 0本漫画`
