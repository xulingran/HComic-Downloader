## 新增需求

### 需求:工具箱查缺补漏入口

工具箱页面必须提供「查缺补漏」入口，作为独立区块与「标签过滤」「推荐标签」「重复检测」并列。入口必须在 `ToolboxPage` 的 `SECTIONS` 导航数组中追加为第 4 项，点击后平滑滚动到对应区块。查缺补漏区块必须渲染独立组件 `MissingChapterDetector`，禁止与重复检测（`DuplicateDetector`）共享组件实例或组件级状态。

#### 场景:工具箱导航出现第 4 项
- **当** 用户进入工具箱页面
- **那么** 左侧导航必须显示 4 个条目，第 4 项为「查缺补漏」

#### 场景:点击导航滚动到区块
- **当** 用户点击导航中的「查缺补漏」
- **那么** 页面必须平滑滚动到查缺补漏区块，且该导航项短暂高亮

#### 场景:查缺补漏区块独立于重复检测
- **当** 用户在查缺补漏区块操作（切换来源、开始检测）
- **那么** 重复检测区块的状态与结果必须不受任何影响

### 需求:数据源限定为收藏夹

查缺补漏功能的唯一数据源必须是 `getFavourites` 分页拉取的收藏项列表。功能执行期间禁止读取 `download_history`、禁止探测硬盘文件、禁止调用任何与下载状态相关的 IPC 方法（`check_downloaded_status`、`get_album_progress` 等）。

#### 场景:不读取下载历史
- **当** 用户触发查缺补漏检测
- **那么** 检测过程必须仅调用 `getFavourites`，禁止调用 `check_downloaded_status` 或访问任何下载历史数据

#### 场景:文件缺失不影响判定
- **当** 某收藏项对应的下载文件已被删除，但仍存在于收藏夹
- **那么** 该收藏项仍必须正常参与聚类展示，不因其文件缺失而被排除

### 需求:同系列聚类复用相似度算法

查缺补漏必须复用 `findDuplicateGroups`（LCS≥0.6 + 并查集聚类）将收藏项按标题相似度归为同系列组。聚类必须对所有来源统一适用，禁止为不同来源使用不同的分组逻辑。所有聚类得到的组（成员数 ≥ 2）必须全部展示给用户，禁止过滤掉任何组。

#### 场景:所有来源统一聚类
- **当** 用户在 hcomic、moeimg、jmcomic、bika 任一来源执行检测
- **那么** 必须使用同一套 `findDuplicateGroups` 算法聚类，不因来源切换而改变分组逻辑

#### 场景:相似标题归为同组
- **当** 收藏夹含标题相似的若干项（LCS ≥ 0.6）
- **那么** 这些项必须被聚为同一个同系列组

#### 场景:所有组全部展示不过滤
- **当** 聚类得到 N 个组
- **那么** 这 N 个组必须全部展示给用户，禁止因任何启发式规则（如"无缺失"）隐藏组

#### 场景:孤立项不参与展示
- **当** 某收藏项的标题与任何其他收藏项相似度均低于 0.6
- **那么** 该项必须被排除（不构成组，无从判断是否同系列）

### 需求:每组提供搜索此系列入口

每个同系列组必须提供「搜索此系列」按钮，点击后触发一次搜索，搜索词为该组的系列名。系列名提取规则：调用 `extractAlbumTitle`（组内所有成员的原始标题）得到原始系列名，再经清洗剥离版本/翻译方括号标记（如 `[中国翻訳]`、`[DL版]`、`[汉化]`）与行尾序号标记（如 `LEVEL:N`、`第N话`、`#N`），保留作品名主体。清洗后长度 < 2 时，按钮必须禁用。点击按钮必须调用 `useDrawerStore` 的 `setPendingSearch(cleanedName, 'keyword')`，复用项目既有的 pendingSearch 机制（先例：`ComicInfoDrawer`）由 `App.tsx` 自动跳转搜索页并执行搜索。

#### 场景:点击搜索触发跳转
- **当** 用户点击某组的「搜索此系列」按钮
- **那么** 必须调用 `setPendingSearch` 传入清洗后的系列名与 `'keyword'` 模式

#### 场景:系列名剥离版本标记
- **当** 组内成员标题含共有的方括号标记（如 `[中国翻訳] [DL版]`）
- **那么** 搜索词必须剥离这些标记，仅保留作品名主体

#### 场景:系列名剥离序号标记
- **当** 组内成员标题含序号标记（如 `LEVEL:9`、`第3话`、`#5`）
- **那么** 搜索词必须剥离这些序号标记

#### 场景:系列名提取失败时禁用按钮
- **当** `extractAlbumTitle` 返回 null，或清洗后长度 < 2
- **那么** 「搜索此系列」按钮必须处于禁用态（disabled），且不触发搜索

### 需求:推测性质必须在 UI 显式标注

由于同系列判定本质是基于标题相似度的推测而非站点权威信息，结果区块顶部或显著位置必须展示一条免责声明。声明文案必须明确说明判定基于标题相似度推测，且是否漏收由用户自行核对。

#### 场景:区块含免责声明
- **当** 检测完成并展示结果
- **那么** 结果区块必须可见地展示声明，说明"同系列判定基于标题相似度推测，是否漏收请自行核对"

### 需求:检测进度与分页拉取

检测过程必须分两阶段向用户反馈进度：拉取收藏夹分页（显示"正在获取第 N/M 页..."）与计算相似度（显示"正在计算相似度..."）。分页拉取必须逐页串行调用 `getFavourites`，累计全部页的收藏项。单页拉取失败时必须跳过该页并累加失败页数计数，最终在结果区展示警告文案。

#### 场景:拉取阶段进度文案
- **当** 检测处于拉取阶段
- **那么** 按钮文案必须显示"正在获取第 N/M 页..."形式的进度

#### 场景:单页失败跳过
- **当** 某页 `getFavourites` 调用抛错
- **那么** 该页必须被跳过，失败页计数加 1，其余页继续拉取，结果区最终显示"警告：N 页数据获取失败，结果可能不完整"

### 需求:空态与未登录处理

当用户未登录当前来源（`getFavourites` 返回 `needsLogin: true`）时，必须显示明确的登录提示文案，禁止展示空结果或报错。当检测完成但无任何同系列组（收藏项不足或无相似标题）时，必须显示"未发现疑似同系列的收藏"空态文案。当用户尚未触发检测（初始状态）时，必须显示引导文案"选择来源并点击开始检测"。

#### 场景:未登录提示
- **当** `getFavourites` 返回 `needsLogin: true`
- **那么** 必须显示"请先登录当前来源"提示，禁止显示报错

#### 场景:无同系列组空态
- **当** 检测完成，收藏项 ≥ 2 但无任何相似度达标的组
- **那么** 必须显示"未发现疑似同系列的收藏"空态文案

#### 场景:初始引导态
- **当** 用户进入查缺补漏区块但未点击开始检测
- **那么** 必须显示"选择来源并点击开始检测"引导文案
