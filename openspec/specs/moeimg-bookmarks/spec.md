### 需求:用户可以查看 moeimg 收藏夹

系统必须允许已登录用户查看 moeimg 收藏夹中的漫画列表，支持分页浏览。

#### 场景:成功获取收藏夹第一页

- **当** 用户已登录 moeimg 且请求收藏夹第一页
- **那么** 系统解析 `/member/bookmarks` HTML，返回收藏夹漫画列表和分页信息

#### 场景:收藏夹为空

- **当** 用户已登录 moeimg 但收藏夹为空
- **那么** 系统返回空列表，needsLogin 为 false

#### 场景:未登录状态

- **当** 用户未登录 moeimg 或登录已过期且无凭据可自动重登
- **那么** 系统返回 needsLogin 为 true

#### 场景:分页浏览

- **当** 用户请求收藏夹第 N 页
- **那么** 系统解析 `/member/bookmarks?page=N`，返回对应页的漫画列表和正确的分页信息

#### 场景:HTML 解析提取字段

- **当** 解析收藏夹页面 HTML
- **那么** 系统从每个 `.u-fav-item` 提取：manga_id（从 `data-manga-id`）、标题（从 `.u-manga-title a`）、封面 URL（从 `.u-img-holder img src`）、详情链接（从 `.u-img-holder a href`）

### 需求:用户可以检查漫画的收藏状态

系统必须允许查询指定漫画是否已被收藏。

#### 场景:已收藏

- **当** 查询已收藏漫画的状态
- **那么** 系统调用 `/ajax/bookmark-status/{manga_id}`，返回 `status == 1`

#### 场景:未收藏

- **当** 查询未收藏漫画的状态
- **那么** 系统调用 `/ajax/bookmark-status/{manga_id}`，返回 `status == -1`

### 需求:用户可以添加漫画到收藏

系统必须允许用户将指定漫画添加到 moeimg 收藏夹。

#### 场景:成功添加

- **当** 用户收藏未收藏的漫画
- **那么** 系统调用 `/ajax/bookmark/{manga_id}`（toggle 模式），返回成功状态

#### 场景:已收藏的漫画

- **当** 用户尝试收藏已收藏的漫画
- **那么** 系统先检查状态，发现已收藏则直接返回成功（不重复调用 toggle）

### 需求:用户可以移除漫画收藏

系统必须允许用户将指定漫画从 moeimg 收藏夹移除。

#### 场景:成功移除

- **当** 用户移除已收藏的漫画
- **那么** 系统先检查状态确认已收藏，然后调用 `/ajax/bookmark/{manga_id}`（toggle 模式），返回成功状态

#### 场景:未收藏的漫画

- **当** 用户尝试移除未收藏的漫画
- **那么** 系统先检查状态，发现未收藏则直接返回成功（不调用 toggle）

### 需求:前端收藏夹页面支持 moeimg 来源

系统必须在收藏夹页面的来源选择器中包含 moeimg 选项。

#### 场景:切换到 moeimg 收藏夹

- **当** 用户在来源选择器中选择 moeimg
- **那么** 系统加载并显示 moeimg 收藏夹

#### 场景:moeimg 未登录时切换

- **当** 用户切换到 moeimg 收藏夹但未登录
- **那么** 系统显示登录提示
