# local-comic-reader 规范

## 目的

定义在应用内直接阅读本地漫画库资产的能力。支持 CBZ、ZIP、单层图片文件夹和多章节图片文件夹；图片通过受限资产标识与流式通道交付，复用既有阅读体验，持久化本地阅读进度，并安全处理阅读期间的资产变化与损坏。

## 需求

### 需求:本地阅读器支持全部漫画库资产格式

系统必须支持阅读 CBZ、ZIP、单层图片文件夹和多章节图片文件夹。图片必须按自然顺序排列；多章节资产必须先展示章节清单，并在切换章节时重置页资源缓存。

#### 场景:阅读 CBZ 漫画
- **当** 用户点击一个包含 `1.jpg`、`2.jpg`、`10.jpg` 的 CBZ 的“阅读”按钮
- **那么** 阅读器按 1、2、10 的自然顺序展示三页

#### 场景:切换本地专辑章节
- **当** 用户在多章节文件夹中从第一章切换到第二章
- **那么** 阅读器加载第二章页清单、清空第一章页缓存，并将当前页设为该章已保存进度或第一页

### 需求:渲染进程只能通过受限资产标识读取图片

本地图片交付接口必须只接受后端生成的资产 ID、章节 ID、页码和版本令牌，禁止接受渲染进程提供的任意绝对路径或压缩包条目名。每次读取必须重新确认资产真实路径位于当前漫画库根目录内且页码属于已索引清单。

#### 场景:提交伪造资产 ID
- **当** 渲染进程请求一个不存在或不属于当前漫画库的资产 ID
- **那么** 系统拒绝请求并返回不可枚举本地路径的错误

#### 场景:资产通过符号链接逃逸
- **当** 已索引路径后来被替换为指向漫画库根目录外的符号链接
- **那么** 页面读取被拒绝，系统禁止访问链接目标

### 需求:本地图片以流式方式交付

本地漫画页必须由 Electron 主进程或受限协议流式交付给 Chromium，禁止把图片字节编码为 base64 放入 JSON-RPC 响应。压缩包阅读禁止为打开一本漫画而一次性解压全部页面到磁盘或渲染进程内存。

#### 场景:打开大型 CBZ
- **当** 用户打开一个包含数百页的大型 CBZ
- **那么** 系统只按当前页和预加载窗口读取所需条目，图片字节不经过渲染进程 JavaScript 堆的 base64 转换

### 需求:本地阅读复用现有阅读体验

本地阅读必须复用现有滚动、单页/双页、缩放、页间距、预加载、失败重试和 reduced-motion 行为；新增本地数据源禁止改变远程预览阅读路径的既有行为。

#### 场景:在本地漫画中使用双页模式
- **当** 用户将阅读模式切换为双页并翻页
- **那么** 阅读器沿用现有双页布局和空白页规则，图片数据来自本地资产页接口

#### 场景:打开远程预览
- **当** 用户从搜索页打开远程漫画预览
- **那么** 阅读器继续通过既有预览 IPC 获取图片，不依赖漫画库索引

### 需求:持久化本地阅读进度

系统必须按资产 ID 和章节 ID 持久化当前页、总页数和最后阅读时间，并在用户切页时节流保存、关闭阅读器时立即保存。最近阅读排序必须使用该记录；资产被应用内重命名时进度必须保留。

#### 场景:继续阅读单本漫画
- **当** 用户在第 35 页关闭本地漫画，之后再次点击“继续阅读”
- **那么** 阅读器恢复到第 35 页，并更新最后阅读时间

#### 场景:外部修改导致页数缩短
- **当** 已保存进度为第 35 页但重新扫描后资产仅剩 20 页
- **那么** 恢复页被限制为第 20 页，禁止请求越界页面

### 需求:本地阅读必须处理资产变化和损坏

页请求必须携带索引版本令牌。若资产在阅读期间被修改、删除、替换或压缩包条目损坏，系统必须终止旧版本读取、清理相关句柄并向用户显示可恢复错误，不得崩溃或继续返回混合版本页面。

#### 场景:阅读期间外部替换文件
- **当** 用户阅读过程中外部程序替换同路径 CBZ，使版本令牌失效
- **那么** 后续页请求返回“资产已变化”，阅读器提示重新加载漫画库索引

#### 场景:单页图片损坏
- **当** 请求的压缩包条目无法解码为支持的图片
- **那么** 阅读器显示该页加载失败并允许重试或跳过，其他有效页面仍可阅读
