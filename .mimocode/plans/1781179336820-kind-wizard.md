# Plan: Bika 收藏夹排序从新到老

## 问题
Bika 收藏夹 API (`users/favourite?page={page}`) 默认按老→新排序，用户希望全局反转为新→旧。

## 修改文件
- `sources/bika/parser.py` — `__init__`、`favourites()`、`add_to_favourites()`、`remove_from_favourites()`

## 实现步骤

### 1. `__init__` (line 32-39): 添加缓存字段
```python
self._favourites_total_pages: int = 0
```

### 2. `favourites()` (line 444-469): 镜像页码 + 反转列表

核心逻辑：
- 若 `_favourites_total_pages` 已缓存 → 计算镜像页码 `api_page = total_pages - page + 1`，请求数据后反转 comics 列表
- 若未缓存（首次请求） → 先请求 page=1 获取总页数，然后立即请求镜像页（page N），反转列表返回
- 返回的 `PaginationInfo` 的 `current_page` 设为用户请求的 `page`

```python
def favourites(self, page: int = 1, raise_errors: bool = False):
    try:
        self._ensure_token()
    except ParserResponseError:
        return [], None, True

    try:
        # 首次请求：先获取总页数
        if self._favourites_total_pages == 0:
            result = self._request(Method.GET, "users/favourite?page=1")
            _, pagination = self._parse_comics_response(result)
            if pagination and pagination.total_pages > 0:
                self._favourites_total_pages = pagination.total_pages
            else:
                # 只有一页或无分页信息，直接返回（反转即可）
                comics, _ = self._parse_comics_response(result)
                comics.reverse()
                if pagination:
                    pagination.current_page = page
                return comics, pagination, False

        # 计算镜像页码
        total = self._favourites_total_pages
        api_page = max(1, min(total - page + 1, total))

        # 如果首次请求恰好就是需要的页，复用 result
        if api_page == 1 and self._favourites_total_pages > 0:
            # 已在上面请求了 page=1，这里需要判断是否复用
            pass  # 简化：统一重新请求

        result = self._request(Method.GET, f"users/favourite?page={api_page}")
        comics, pagination = self._parse_comics_response(result)
        comics.reverse()

        if pagination:
            pagination = dataclasses.replace(pagination, current_page=page)

        return comics, pagination, False
    except ParserResponseError as e:
        logger.error("Bika favourites failed: %s", e, exc_info=True)
        if raise_errors:
            raise
        return [], None, False
```

> **注意**：`PaginationInfo` 是 dataclass，可以用 `dataclasses.replace()` 创建新实例。也可直接赋值 `pagination.current_page = page`（它不是 frozen 的）。用直接赋值更简单。

### 3. `add_to_favourites()` 和 `remove_from_favourites()` 中清除缓存

在这两个方法成功后添加 `self._favourites_total_pages = 0`，确保下次请求重新获取总页数。
