## 关键问题
1. 如何在不破坏现有登录窗口公开契约的前提下返回仅供主进程使用的 DOM 快照？
2. 如何确保 Electron IPC 自定义错误数据不依赖跨 renderer 的 Error 属性序列化？

## 已做决策
| 决策 | 理由 |
|------|------|
| 基于现有未提交 JM 后台挑战恢复继续实现 | 这些改动属于用户现有工作且正是新变更的前置基础 |
| 主进程消费结构化挑战错误 | Electron renderer 不可靠保留自定义 Error 字段，且敏感恢复应留在主进程 |
| 一次 Python 重试后使用可信 DOM 快照兜底 | 防止 clearance 在不同客户端指纹间不可复用并避免无限循环 |
