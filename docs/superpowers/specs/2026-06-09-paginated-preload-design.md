# 列表分页预载设计

## 概述

本次设计为所有用户可手动翻页的列表页面增加分页预载能力，包括搜索结果、收藏夹和阅读历史。漫画阅读器中的图片页预载已经由现有 `usePreloadManager` 管理，本次保持现状不变，不调整其预载窗口和触发策略。

列表分页预载采用“当前页前后各两页”的策略。当前页成功加载或刷新后，后台预载 `N-2`、`N-1`、`N+1`、`N+2` 范围内的合法页码。用户翻到已预载页时，页面先立即显示缓存内容，同时后台刷新该页，刷新成功后替换显示和缓存。

## 目标

- 搜索、收藏夹、阅读历史等用户可翻页列表支持后台预载。
- 预载命中时页面立即切换，降低翻页等待感。
- 命中缓存后仍后台刷新当前页，兼顾速度和数据新鲜度。
- 预载失败不干扰当前页面，不进入用户可见错误态。
- 复用同一套分页预载调度逻辑，避免每个页面重复实现。

## 非目标

- 不修改漫画阅读器图片页预载策略。
- 不新增设置项或 UI 开关。
- 不为工具类内部批量扫描流程添加预载，例如重复检测工具内部遍历收藏夹页的逻辑。
- 不引入全局泛型分页缓存 store；各页面继续使用并扩展自己的缓存 store。

## 范围

纳入本次预载范围的用户可翻页入口：

| 页面 | 当前入口 | 预载上下文 |
|---|---|---|
| 搜索页 | `SearchPage` + `SearchBar` + `PaginationControls` | `source + mode + query + searchTags` |
| 收藏夹 | `FavouritesPage` + `PaginationControls` | `source` |
| 阅读历史 | `HistoryPage` + `PaginationControls` | 固定历史上下文 |

`TagDialog` 当前是一次性加载标签并在前端本地过滤，不提供用户分页控件；不纳入本次列表分页预载。漫画阅读器仍由现有图片预载机制覆盖。

## 架构

新增通用 hook，暂定名为 `usePaginatedPreloader`。它只负责分页预载调度，不包含搜索、收藏夹或历史记录的业务规则。

### `usePaginatedPreloader` 输入

| 参数 | 说明 |
|---|---|
| `currentPage` | 当前页面页码 |
| `totalPages` | 总页数 |
| `contextKey` | 当前分页上下文标识，搜索条件或来源变化时改变 |
| `enabled` | 是否允许预载，例如登录过期或当前无分页数据时关闭 |
| `hasPage(page)` | 判断指定页是否已有缓存 |
| `loadPage(page, reason)` | 后台加载指定页，`reason` 可为 `preload` |
| `onPreloadError` | 可选错误回调，默认只记录调试信息 |

### 调度规则

当前页为 `N` 时，候选页为 `N+1`、`N-1`、`N+2`、`N-2`。按距离当前页从近到远排序，优先准备最可能访问的页面。hook 会自动跳过：

- 小于 1 或大于 `totalPages` 的页码。
- 已经存在缓存的页码。
- 当前上下文中正在预载的页码。
- `enabled` 为 `false` 时的所有候选页。

同一上下文最多同时运行 2 个预载请求。剩余候选页在请求完成后继续调度，避免一次性发出过多后台请求。

### 上下文隔离

`contextKey` 改变时，旧上下文请求的返回结果不能写入新上下文缓存。实现上使用 generation/ref 标记忽略旧请求返回，不依赖底层 IPC 支持取消。

## 缓存模型

### 搜索缓存

当前 `useSearchCacheStore` 只缓存一个搜索结果页，需要扩展为按搜索上下文缓存多页。搜索上下文由 `source`、`mode`、`query` 和 `searchTags` 组成。

每个上下文下保存：

- `pages[page]`: 当前页的 `comics` 和 `pagination`。
- `currentContextKey`: 最近展示的搜索上下文。
- 当前展示页信息，用于返回搜索页时恢复上次页面。

搜索条件变化、来源变化、模式变化、标签变化或手动刷新时，页面切换到新的上下文或清理相关缓存，避免旧搜索结果误用到新条件。

### 收藏夹缓存

当前 `useFavouritesStore` 按来源保存一个页面快照，需要扩展为按 `source + page` 保存多页。

每页保存：

- `comics`
- `pagination`
- `currentPage`
- `downloadedStatus`

预载收藏夹页时也执行下载状态检查。下载状态检查失败不阻塞漫画列表缓存，失败页的 `downloadedStatus` 可为空映射或保持 `unknown`。

来源切换时读取对应来源的缓存。刷新当前来源时清理该来源缓存并重新加载第 1 页。

### 阅读历史缓存

当前 `useHistoryStore` 保存一个页面快照，需要扩展为多页缓存。

每页保存：

- `items`
- `pagination`
- `currentPage`

删除单条历史、清空历史或手动刷新后，历史缓存整体清空并重新加载当前目标页，避免已删除记录从预载缓存回流。

## 用户翻页数据流

当用户点击上一页、下一页或通过跳页对话框跳转时：

1. 页面根据目标页和当前上下文检查缓存。
2. 如果缓存命中，立即显示缓存内容并更新当前页码。
3. 命中缓存后仍后台请求目标页的新数据。
4. 刷新成功后，用新数据替换当前显示和缓存。
5. 刷新失败且已有缓存展示时，保留缓存展示，不回退为空白页。
6. 如果缓存未命中，则按现有主动加载流程显示加载态，请求失败时显示页面错误。

后台刷新不应使用全屏加载遮罩覆盖已显示的缓存内容。页面可以保留现有加载状态用于未命中加载；缓存命中刷新采用轻量内部状态或不显示显眼状态。

## 预载数据流

当前页主动加载或后台刷新成功后：

1. 页面更新当前显示和页面缓存。
2. `usePaginatedPreloader` 根据 `currentPage`、`totalPages` 和 `contextKey` 计算候选页。
3. hook 跳过非法页、已缓存页和正在请求页。
4. hook 按并发限制调用页面提供的 `loadPage(page, 'preload')`。
5. 预载成功只写入缓存，不改变当前页面展示。
6. 预载失败只记录调试信息，不显示 toast，不设置页面错误。

## 页面职责

### `SearchPage`

- 继续负责搜索条件、认证状态、筛选、批量下载和打开阅读器。
- 将搜索缓存改为多页缓存。
- 主动翻页时实现“缓存命中先显示，再后台刷新”。
- 当前搜索上下文成功加载后启用 `usePaginatedPreloader`。
- 登录过期、认证失败或当前无有效分页信息时禁用预载。

### `FavouritesPage`

- 继续负责来源切换、收藏夹加载、下载状态检查和批量下载。
- 将收藏夹缓存扩展为每个来源多页。
- 预载页也补充该页下载状态，但下载状态失败不阻塞预载缓存。
- 删除、刷新或来源切换时保持缓存边界清晰。

### `HistoryPage`

- 继续负责历史记录加载、删除、清空和打开阅读器。
- 将历史缓存扩展为多页。
- 删除单条记录、清空历史或手动刷新后清理历史缓存。
- 当前页加载成功后预载前后各两页。

### `PaginationControls`

`PaginationControls` 保持展示组件职责，不内置预载逻辑。预载由页面加载成功后统一触发，避免按钮组件承担业务数据请求职责。

## 错误处理

| 场景 | 行为 |
|---|---|
| 预载失败 | 静默失败，仅调试日志；未来用户打开该页时重新请求 |
| 缓存命中后的后台刷新失败 | 保留缓存展示，不进入全页错误态 |
| 无缓存的主动翻页失败 | 沿用现有页面错误展示 |
| 登录过期或认证失败 | 禁用预载，用户主动请求按现有逻辑处理 |
| 上下文切换后旧请求返回 | 忽略旧结果，不写入新上下文缓存 |
| 下载状态检查失败 | 收藏夹页仍缓存漫画列表，下载状态保持未知 |

## 测试计划

### 单元测试

- 页码计算覆盖第 1 页、第 2 页、中间页、倒数页和最后页。
- `usePaginatedPreloader` 不预载越界页。
- 已缓存页不会重复请求。
- 正在请求的页不会重复请求。
- `contextKey` 改变后，旧上下文请求返回不会污染新缓存。
- 并发限制最多同时运行 2 个预载请求。
- 预载失败不改变当前页面状态。

### Store 测试

- 搜索缓存按 `source + mode + query + searchTags + page` 隔离。
- 收藏夹缓存按 `source + page` 隔离。
- 历史缓存按页保存并能整体清理。
- 命中缓存后可立即恢复当前页数据。

### 页面集成测试

- 搜索页当前页加载成功后预载前后各两页。
- 搜索页翻到预载页时先显示缓存，再后台刷新。
- 收藏夹页预载页包含漫画列表，并尽量补充下载状态。
- 阅读历史页删除或清空后不会从预载缓存恢复旧记录。
- 漫画阅读器现有图片预载行为保持不变。

## 验收标准

- 搜索、收藏夹、阅读历史在当前页加载成功后都会后台预载合法范围内前后各两页。
- 用户翻到预载过的列表页时能立即看到缓存结果。
- 预载命中后会后台刷新目标页，刷新成功后更新展示。
- 预载失败不会显示错误、不会清空当前页面、不会阻止用户主动翻页。
- 搜索条件、来源或历史数据变化后不会使用错误上下文的缓存。
- 漫画阅读器图片页预载策略没有行为变化。
