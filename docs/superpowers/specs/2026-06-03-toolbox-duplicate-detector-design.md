# 工具箱 - 收藏夹重复漫画检测功能设计

## 概述

新增"工具箱"标签页，作为独立功能入口。首个工具为"收藏夹重复漫画检测"：前端逐页获取指定来源的全部收藏，通过标题相似度算法找出疑似重复的漫画分组，用户可点击打开详情抽屉进行人工确认。

## 架构

### 工具箱页面 (ToolboxPage)

新增顶级标签页，位于现有导航栏中。页面结构：

```
工具箱页面 (ToolboxPage)
├── 工具卡片列表
│   ├── 重复检测工具 ← 第一个工具
│   └── (未来更多工具卡片...)
```

每个工具是独立组件，封装自身逻辑和 UI。工具卡片展示名称、简要描述和入口按钮。点击后展开或进入该工具的操作面板。

### 组件职责

- **ToolboxPage**：工具箱主页面，承载各工具卡片
- **DuplicateDetector**：重复检测工具的主组件，管理来源选择、检测流程和结果展示
- **DuplicateGroup**：单个疑似重复分组的展示组件

## 数据获取流程

复用 `sync_favourite_tags` 的逐页遍历模式，在前端执行：

1. 用户选择来源后点击"开始检测"
2. 前端调用 `getFavourites(1, source)` 获取第一页，从返回值得到 `totalPages`
3. 循环调用 `getFavourites(page, source)` 从第 2 页到最后一页，将所有 `comics` 合并到一个数组
4. 每获取一页更新进度指示器（"正在获取第 X/Y 页..."）
5. 全部获取完毕后进入相似度计算阶段

容错策略：如果某一页请求失败，跳过该页继续，最终结果中标注"部分数据可能缺失"。

## 标题相似度算法

### 预处理（归一化标题）

- 去除常见后缀修饰词，如括号内容 `（全彩）`、`(Chinese)`、`[汉化组名]` 等
- 去除多余空格，全角转半角
- 保留核心标题文本

### 相似度计算

- 使用最长公共子序列比率（LCS ratio）作为主要指标：`LCS长度 / max(标题A长度, 标题B长度)`
- 阈值：比率 ≥ 0.6 的配对视为"疑似重复"
- 两两比较所有漫画标题，将满足阈值的漫画用并查集（union-find）归入同一分组（处理传递性相似关系）

### 结果排序

- 每组包含 2 本及以上标题相似的漫画
- 按组内漫画数量降序排列（最可能重复的排前面）

## 结果展示与交互

### 纵向分组列表

每个疑似重复组纵向排列，组内漫画也是纵向列表：

```
┌─ 疑似重复组 1（3 本）───────────────────┐
│  [封面]  完整标题A文字完整显示在这里       │
│  [封面]  完整标题B文字完整显示在这里       │
│  [封面]  完整标题C文字完整显示在这里       │
└──────────────────────────────────────────┘
┌─ 疑似重复组 2（2 本）───────────────────┐
│  [封面]  完整标题D文字完整显示在这里       │
│  [封面]  完整标题E文字完整显示在这里       │
└──────────────────────────────────────────┘
```

- 每组为可折叠卡片，标题显示"疑似重复组 N（X 本）"
- 组内每行：左侧封面缩略图 + 右侧完整标题文本（不截断）+ 相似度百分比
- 每行可点击，调用 `useDrawerStore.openDrawer(comic)` 打开详情抽屉
- 组与组之间有间距分隔

### 空状态

- 未检测时显示引导文案："选择来源并点击开始检测"
- 检测完成无重复时显示："未发现疑似重复的漫画"

### 性能考量

- 如果收藏数量较大（>500），相似度计算使用 `setTimeout` 分片执行避免阻塞 UI，配合进度条展示
