# 文件冲突检测功能设计

## 概述

在批量下载漫画前检测同名文件冲突，让用户选择覆盖、跳过或批量处理。

## 需求

- 批量下载时，在开始下载前统一检测所有冲突
- 逐个询问用户处理方式，提供"全部覆盖"/"全部跳过"快捷选项
- 单个选项：覆盖、跳过、覆盖全部、跳过全部

## 设计

### 核心逻辑

**冲突检测位置**

在 `execute_batch_download()` 方法中，任务加入队列之前进行冲突检测。流程：

```
用户点击批量下载
  → confirm_batch_download() 确认下载列表
  → detect_file_conflicts() 检测冲突
  → 如有冲突，显示 FileConflictDialog 让用户处理
  → 过滤掉用户选择跳过的漫画
  → add_tasks() 添加剩余任务到队列
```

**冲突检测方法**

在 `CBZBuilder` 中新增 `get_output_path(comic)` 方法，仅生成路径不创建文件。然后在 GUI 中遍历所有待下载漫画，用 `os.path.exists()` 检测文件是否已存在。

```
│                                                     │
│  ┌─────────────────────────────────────────────┐   │

│                                                     │
│                              [确定]    [取消]       │
└─────────────────────────────────────────────────────┘
```

**交互逻辑**

- 顶部单选按钮：设置默认操作（覆盖/跳过/覆盖全部/跳过全部）
- 列表中每项可单独勾选复选框覆盖默认操作
- 选择"覆盖全部"或"跳过全部"时，列表项全部选中并禁用
- 点击"确定"后根据选择过滤漫画列表

### 代码修改清单

**1. cbz_builder.py - 新增公开方法**

```python
def get_output_path(self, comic: ComicInfo) -> str:
    """获取漫画的输出路径（不创建文件）"""
    return self._generate_output_path(comic)
```

**2. gui.py - 新增冲突检测方法**

在 `HComicDownloaderGUI` 类中新增：
- `detect_file_conflicts(comics)` - 返回存在冲突的漫画列表及其索引
- 修改 `execute_batch_download()` - 在添加任务前调用冲突检测

**3. 新建 file_conflict_dialog.py**

独立的对话框模块，包含：
- `FileConflictDialog` 类 - 继承自 `tk.Toplevel`
- 返回值：`dict[index, bool]` 其中 bool 为 True 表示覆盖，False 表示跳过

4. 修改 `execute_batch_download()` 集成冲突检测
5. 测试各种场景（单冲突、多冲突、全部覆盖、全部跳过）
